# Гайд по переписке теории (IT Птица)

Для авторов, ревьюеров и AI при работе с `content/theory/`.

## Назначение

Импортированная из Buildin теория (~420 уроков) написана в стиле структурированного лонгрида. Нужно **свои формулировки и бренд IT Птица**, сохранив педагогику и связку **теория → квиз → практика**.

## Два режима

| Режим | Когда | Что делать |
|-------|--------|------------|
| **rewrite-light** | Все импортированные уроки (420) | Перефразировать прозу, убрать чужие бренды, не ломать структуру |
| **author-full** | Новые темы, тестовое ЦУ | Полный шаблон автора (ситуации, подконцепты) — **не** для массового импорта |

Этот документ описывает **rewrite-light**.

## Правила rewrite-light

### Сохранять

- Порядок и названия заголовков `##`, `###`
- Блоки кода, таблицы, списки
- `> **Примечание.**` и аналогичные callout
- Технические термины из квиза (селектор, каскад, flex, `querySelector`, …)
- `title` в YAML frontmatter, путь файла, `buildin_id`

### Менять

- Формулировки абзацев (синонимы, другой порядок слов)
- Упоминания Hexlet, Курсовик, codepen.io/hexlet
- Ссылки на guides.hexlet.io → MDN или нейтральные источники
- Примеры с брендом в тексте кода (напр. «Курсовик» → «IT Птица»)

### Не добавлять

- Блоки «Ситуация», «Зачем сейчас» на каждый подпункт
- Длинные мотивационные вступления
- Новые разделы, которых не было в оригинале

## Замены брендов

| Было | Стало |
|------|--------|
| Hexlet, hexlet.io | убрать или нейтрально |
| guides.hexlet.io/… | MDN / developer.mozilla.org |
| ru.hexlet.io/challenges/… | «практическое задание» или свой ресурс |
| codepen.io/hexlet/… | свой pen или оставить до эпика P2 (медиа) |
| Курсовик | IT Птица / учебная платформа / учебный проект |


## Definition of Done

- [ ] Текст перефразирован, нет дословных копий длинных абзацев
- [ ] `rg -i hexlet` по файлу — пусто
- [ ] Нет «Курсовик» (если был — заменён)
- [ ] `title`, путь, slug в `course.json` не менялись
- [ ] Квиз по уроку: термины из вопросов есть в тексте
- [ ] `source: platform`, `rewritten_at` заполнены
- [ ] `python3 scripts/check-theory-lesson.py <file>` — OK

## Примеры до / после

**Было (фрагмент `05-osnovy-css.md`):**

> Для визуального оформления WEB-страницы создан язык CSS. CSS переводится как *каскадные таблицы стилей*…

**Стало:**

> Чтобы управлять внешним видом страницы, используют язык **CSS** (*Cascading Style Sheets* — каскадные таблицы стилей). Именно он задаёт, как браузер отрисует HTML-элементы.
