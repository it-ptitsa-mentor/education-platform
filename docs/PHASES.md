**оригинальные тесты Hexlet**, вытянутые из IDE-контейнеров

## 82 нетестируемых задания — нужен режим «самопроверка»

После волн 19–21 останутся заглушки, которые **vitest закрыть не может структурно**:

| Блок | Заданий | Почему нет автопроверки |
|---|---:|---|
| Вёрстка контента + основы | 21 | визуальные, нужна snapshot-инфра (playwright) |
| CSS: Flex / Grid / макеты | 16 | то же |
| Основы командной строки | 16 | реальный терминал, нет web-IDE у Hexlet (`run.ide_url` отсутствует) |
| Введение в Git | 13 | терминал + git-состояние |
| Протокол HTTP | 11 | сервер/сокеты |
| HTTP API | 6 | то же |

**Продуктовое решение (задача в Todoist, Ready):** на этих заданиях студент не должен
видеть фейковый «PASS» от заглушки. Вместо RUN TESTS — режим «самопроверка»:
кнопка «Показать эталон / чек-лист», честная пометка «без автопроверки».
Реализация: классификатор уже отличает stub — прокинуть класс теста в каталог
(`packages/shared`) и ветвить UI (`ExerciseWorkspace`/`ExerciseRunner`).
