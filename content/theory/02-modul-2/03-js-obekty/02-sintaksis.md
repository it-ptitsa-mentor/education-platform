# Синтаксис

Объект в JavaScript — тип данных для связанного набора полей. С ним удобно оперировать данными как единым целым: и реальные предметы, и абстрактные сущности (например, геометрические фигуры) описываются одинаково.

```javascript
const user = { name: 'Vasya', married: true, age: 25 }

const rectangle = { width: 10, height: 5 }
```

Объект задаётся перечислением пар «ключ: значение» в фигурных скобках через запятую. Такие пары называют свойствами (property). Ключи уникальны: два разных значения под одним именем в одном объекте быть не может.

При большом числе свойств запись растягивают на несколько строк:

```javascript
const user = {
  name: 'Vasya',
  married: true,
  age: 25,
}
```

Завершающая запятая не обязательна, но линтер её рекомендует — проще добавлять и удалять поля без правки последней строки.

К свойствам обращаются через точку:

```javascript
user.name // 'Vasya'
rectangle.width // 10
```

Обращение к несуществующему свойству (случайно или намеренно) вернёт `undefined`, и выполнение продолжится. Из-за этого легко пропустить опечатку в имени — проверяйте написание, если данные «не те» или пустые:

```javascript
user.nme // undefined
user.name // 'Vasya'
```

Есть и альтернатива — квадратные скобки, как у массивов:

```javascript
// имена свойств хранятся внутри объекта в виде строк.
user['name'] // 'Vasya'
rectangle['width'] // 10
user['company'] // undefined
```

Зачем второй способ? В реальном коде имя свойства часто вычисляется в рантайме. Через точку подставить его нельзя, через скобки — можно:

```javascript
const user = { name: 'Vasya', married: true, age: 25 }

let propertyName = 'name'
user[propertyName] // 'Vasya'

propertyName = 'married'
user[propertyName] // true
```

Подробнее о таких сценариях — в одном из следующих уроков.
