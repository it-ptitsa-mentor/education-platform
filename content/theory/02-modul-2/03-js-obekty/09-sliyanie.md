# Слияние

Слияние (merge) — объединение объектов: данные одного переносят в другой.

Типичный сценарий — веб-форма: пользователь меняет профиль, приложение получает объект с полями, их нужно влить в объект пользователя в системе:

```
// Такой пользователь есть в системе
const user = { name: 'Tirion', email: 'support@email.io', age: 44 }

// Из формы пришли данные
const data = { name: 'Tirion 2', age: 33 }

// В результате должно получиться
// { name: 'Tirion 2', email: 'support@email.io', age: 33 };
```

Прямой перенос по полям:

```
user.name = data.name
user.age = data.age

// Где-то тут сохраняем пользователя в базе данных
```

Так удобно, пока полей мало и набор стабилен. Если полей много или состав меняется, код разрастается:

```
if (Object.hasOwn(data, 'name')) {
  user.name = data.name
}

// И так нужно перечислить все возможные свойства
```

Слиянием (часто говорят «мерж») всё сводится к одной строке:

```
Object.assign(user, data)
console.log(user)
// => { name: 'Tirion 2', email: 'support@email.io', age: 33 };
```

Метод [Object.assign()](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) берёт объект из первого аргумента и копирует в него свойства из остальных. У нас второй аргумент — один объект `data`.

Правила: поле только в первом объекте остаётся как было; поле из второго (и далее) записывается в первый — и создаёт новое, и перезаписывает существующее:

```
const obj1 = { a: 'a', b: 'b' }
const obj2 = { c: 'c', b: 'v' }
Object.assign(obj1, obj2)
console.log(obj1)
// => { a: 'a', b: 'v', c: 'c' }
```

Ограничение: `Object.assign()` делает только *поверхностное* слияние. Вложенные объекты не сливаются по полям — целиком заменяются:

```
const obj1 = { a: { a: 1 } }
const obj2 = { a: { b: 1 } }
Object.assign(obj1, obj2)
console.log(obj1)
// => { a: { b: 1 } }

// пример глубокого (рекурсивного) слияния
// используется метод merge() библиотеки lodash
import _ from 'lodash'
_.merge(obj1, obj2)
// => { a: { a: 1, b: 1 } }
```

Слияние — мощный инструмент, им нужно пользоваться аккуратно. В объекте бывают поля, которые нельзя перезаписывать «слепо» — например, баланс счёта. Без контроля состава второго объекта туда могут попасть лишние или злонамеренные ключи.

*Если говорить про веб-формы, то технически всегда можно послать больше данных, чем описано в форме.*
