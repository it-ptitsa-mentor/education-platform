# Глобальная установка пакетов

Часть пакетов Node.js — не библиотеки для импорта, а готовые программы. Например [cloc](https://github.com/AlDanial/cloc) считает строки кода в проекте на разных языках, не только в JavaScript.

Такие утилиты не привязаны к одному репозиторию. В *npm* для них есть **глобальная** установка:

```
# В зависимости от способа установки Node.js
# может понадобиться sudo в начале команды
npm install -g cloc
```

Флаг `-g` ставит пакет в общую директорию. Команду можно вызвать из любой папки. Путь к префиксу:

```
npm config ls -l

...
prefix = "/Users/tirion/.asdf/installs/nodejs/14.5.0/.npm"
...
```

После установки утилита доступна в shell под именем из документации. У *cloc* имя совпадает с пакетом:

```
# Обратите внимание на точку после команды cloc
# Она означает что нужно анализировать текущую директорию
cloc .

      49 text files.
      41 unique files.
      11 files ignored.

github.com/AlDanial/cloc v 1.98  T=0.02 s (2459.4 files/s, 74680.8 lines/s)
-------------------------------------------------------------------------------
Language                     files          blank        comment           code
-------------------------------------------------------------------------------
Markdown                        19            213              0            523
YAML                            21             35              0            471
CSS                              1              0              0              3
-------------------------------------------------------------------------------
SUM:                            41            248              0            997
-------------------------------------------------------------------------------
```

## Самостоятельная работа

1. Установите утилиту *cloc* глобально

1. В проекте *nodejs-package* запустите анализ количества строк

1. В проекте my*-js* также запустите cloc и сравните результаты анализа обоих проектов

1. В качестве эксперимента попробуйте установить [hexo.io](https://hexo.io/), фреймворк для создания блога. На главной странице проекта есть информация о том как его установить и запустить
