# Зависимости для разработки

У проекта обычно минимум два режима — **среды**. Когда вы работаете на своём компьютере, это среда разработки. Когда кодом пользуются пользователи — **продакшен** (production).

Среды стараются сделать похожими, чтобы не ловить баги «только у меня / только на сервере». Но требования разные. Для разработки часто нужны пакеты вроде тестового фреймворка; на продакшене они лишние — занимают место и замедляют доставку.

В JavaScript-проектах зависимости «только для разработки» помечают флагом при установке:

```
# jest – тестовый фреймворк,
# с помощью которого пишутся автоматизированные тесты кода
npm install --save-dev jest
```

Процедура почти как у обычных пакетов, но запись попадает в *devDependencies*, а не в *dependencies*.

```
"devDependencies": {
  "jest": "^26.4.1"
}
```

Использовать такие пакеты можно так же, как обычные; отличие — где их импортируют. Тесты обычно лежат в отдельной директории и запускаются только при разработке, поэтому импорты тестов не пересекаются с кодом приложения.

Всё это наглядно видно в учебном npm-пакете *nodejs-package* (см. задания курса).

## Флаг --production

Зависимости для разработки ставятся при обычном `npm install` — предполагается, что команду чаще вызывают локально. Для продакшена нужен флаг:

```
# Вот теперь зависимости из devDependencies устанавливаться не будут
npm install --production

# Продакшен режим можно задать и с помощью переменной окружения
NODE_ENV=production npm install
```

При сборке для деплоя (например, через Github Actions) тот же смысл у `npm ci`:

```
npm ci --production
```

С продакшеном вы познакомитесь позже, но про *devDependencies* важно знать сразу — они есть почти в каждом репозитории.

## Самостоятельная работа

1. Склонируйте учебный проект *nodejs-package* (см. задания курса)

1. Установите его зависимости

1. Из корневой директории проекта запустите код на выполнение `bin/nodejs-package.js 10`. На экран должно вывестись число

1. Запустите тесты `npm test`. Они должны завершиться успехом

1. Посмотрите какие зависимости есть в проекте, и в директориях *src* и *__tests__* как они импортируются

1. Изучите размер и содержимое *node_modules*, затем удалите ее, установите зависимости с флагом `--production`, и посмотрите новый размер и содержимое.
