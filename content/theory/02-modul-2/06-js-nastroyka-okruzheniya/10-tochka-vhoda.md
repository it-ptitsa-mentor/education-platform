# Точка входа

В разработке приложений есть понятие **точки входа**. Правильное выделение точек входа упрощает архитектуру и расширяет возможности проекта.

## Точка входа для приложения

Приложение — много модулей, но при запуске стартует один файл — точка входа:

```
// file: src/my-application/index.js

export default () => console.log('Welcome to my application!')

// file: index.js
import app from './src/my-application/index.js'

app()
```

Два файла:

- *src/my-application/index.js* — функция, выводящая сообщение, экспортирована наружу

- *index.js* — импорт и вызов

`. ├── index.js ├── package.json └── src └── my-application └── index.js`

Корневой *index.js* — точка входа. `node index.js` из корня проекта запускает приложение, вызывается `app()`:

```
node index.js
Welcome to my application!
```

Точек входа может быть несколько — каждая как отдельное приложение на общей логике.

В точке входа обычно вызывают одну функцию. Исключения бывают, но к единственному вызову стоит стремиться — обоснование ниже.

## Точка входа для библиотеки

У приложения точка входа — место старта. У **библиотеки** код при импорте не должен сам выполняться: библиотека отдаёт функции, а вызывает их потребитель.

`lodash` даёт набор утилит — импортируете что нужно и решаете, когда вызывать:

```
import { random } from 'lodash'

console.log(random(0, 5))
```

Мы указываем имя пакета, не путь к файлу. Node находит модуль благодаря точке входа npm-пакета.

В *package.json* поле `"main"` — путь к файлу, который экспортирует API библиотеки. Тогда потребителю не нужно знать внутреннюю структуру: при `import ... from 'имя-пакета'` подтягивается модуль из `"main"`.

## Точка входа библиотеки и приложения в одном проекте

Проект может быть и приложением, и библиотекой. Допустим, CLI спрашивает возраст и печатает год рождения.

Прямолинейный вариант — один файл:

```
// file: index.js
console.log('Сколько вам лет?')
const age = 18 // Каким-то способом получаем возраст пользователя, через терминал. Сейчас не важно как
const birthYear = 2005 // Подсчитываем результат
console.log(`Ваш год рождения: ${birthYear}`)
```

Детали ввода из терминала и текущей даты сейчас не важны — в примере стоят заглушки.

Работает, но возможности узкие. Небольшой рефакторинг:

```
// file: src/index.js
export default (age) => {
  const birthYear = 2005 // Подсчитываем результат

  return birthYear
}

// file: index.js
import getBirthYear from './src/index.js'

console.log('Сколько вам лет?')
const age = 18 // Каким-то способом получаем возраст пользователя, через терминал. Сейчас не важно как
const birthYear = getBirthYear(age)
console.log(`Ваш год рождения: ${birthYear}`)
```

Логику расчёта вынесли в модуль. В `"main"` можно указать `./src/index.js` — пакет импортируют как библиотеку. Переиспользование выросло.

Такие функции выделяют не только ради публикации пакета — так проще тестировать.

`console.log` и чтение ввода в функцию не кладут: это обязанности консольного приложения. Если бы функция только печатала, её нельзя было бы вызвать из веб-приложения.

По умолчанию `"main"` часто указывает на корневой *index.js*. Пример *package.json*:

```
{
  "name": "birth-year",
  "type": "module",
  "version": "1.0.0",
  "description": "",
  "main": "src/index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}
```

`"main"` указывает на *src/index.js*.

`. ├── index.js ├── package.json └── src └── index.js`

## Итог

Точка входа приложения — где начинается выполнение. У библиотеки — модуль с экспортом, прописанный в `"main"`. Выделение логики в функцию расширяет проект и облегчает тесты. Это база архитектуры для более сложных систем.

## Самостоятельная работа

1. В проекте my*-js* изучите какой файл импортируется из библиотеки *lodash* при импорте функций из этой библиотеки

1. Перенесите логику из *index.js* в *src/index.js*, оберните в функцию и экспортируйте ее

1. В файле *index.js* сделайте импорт функции из *src/index.js* и вызов этой функции

1. Укажите в `"main"` путь к модулю с экспортом функции

1. Добавьте все изменения на гитхаб
