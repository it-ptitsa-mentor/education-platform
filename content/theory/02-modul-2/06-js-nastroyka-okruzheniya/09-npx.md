# NPX

Есть класс пакетов, которые ставят в проект, но запускают как консольные программы, а не импортируют в код. Они правят стиль, собирают бандлы, проверяют код под старые версии Node или готовят его для браузера.

Один из них — *prettier*: автоформатирование по общим правилам, чтобы команда не спорила о отступах. Попробовать можно в [песочнице на сайте](https://prettier.io/playground/).

До:

```
const fn = (a, b) => {
  const c = a + b; return c
}
```

После:

```
const fn = (a, b) => {
  const c = a + b
  return c
}
```

*prettier* — инструмент разработки, ставят как *dev* зависимость:

```
npm install --save-dev prettier
```

Дальше запускают как утилиту. Имя исполняемого файла обычно совпадает с именем пакета:

```
# Флаг --write означает что мы разрешаем утилите перезаписать файлы проекта
prettier --write .
```

Без глобальной установки получите *command not found: prettier*. Локальный пакет лежит в *node_modules* проекта, а shell ищет команды в системных каталогах, не в репозитории.

Исполняемые файлы локальных пакетов — в *node_modules/.bin*:

```
node_modules/.bin/prettier --write .
```

Неудобно: длинный путь и привязка к структуре каталогов.

Вместе с Node.js идёт *npx* — запуск локальных бинарников без полного пути:

```
npx prettier --write .
```

*npx* смотрит в *node_modules/.bin* и подставляет найденный файл. Если пакета нет — предложит установить.

В реальных JS-проектах утилит много: Babel, Webpack, Eslint, Gulp. Для их запуска обычно нужен *npx*.

## Глобальная установка vs Локальная

*prettier* можно поставить и глобально, и локально — в обоих случаях он работает. Лучше локально: зависимости в репозитории получают все разработчики; при глобальной установке каждый ставит сам, и версии расходятся — форматирование и CI начинают спорить.

## Самостоятельная работа

1. Добавьте *prettier* в проект my*-js*

1. Запустите его через `npx`, посмотрите какие файлы были изменены

1. Подключите *prettier* к своему редактору. В сети довольно много подобных инструкций для всех редакторов

1. Добавьте все изменения на гитхаб
