# Что такое JS?

Что такое язык программирования?

Это набор правил, по которым записывают исходный код. В отличие от естественных языков, правила здесь жёсткие: малейшая опечатка — и программа не запустится. Или запустится, но поведёт себя не так, как ожидалось. Например, в JavaScript нельзя написать `const name;`. Эти правила зафиксированы не только в головах разработчиков, но и в спецификации — документе, описывающем, как язык должен работать.

Синтаксически корректный текст — лишь половина дела. Компьютер не исполняет исходник напрямую: для него понятен только машинный код. Превратить текст в исполняемую программу можно двумя путями. **Компиляция** — заранее собрать готовый бинарник (в Windows у таких программ часто расширение *exe*). **Интерпретация** — переводить код в машинные инструкции по ходу выполнения.

JavaScript — интерпретируемый язык. Для запуска нужен интерпретатор. На сервере им чаще всего выступает Node.js, в браузере — сам браузер. Во встраиваемых устройствах и других средах могут быть свои движки.

*На практике интерпретацию выполняет встроенный «движок» — и в Node.js, и в браузерах.*

```
// node запускает интерпретатор, который берет исходный код index.js,
// анализирует его (это называется парсинг),
// а затем построчно исполняет
node index.js
```

Глобально языки делят на интерпретируемые и компилируемые; у каждой группы свои плюсы и минусы. Здесь сосредоточимся на особенностях JavaScript.

Исторически JavaScript создавали под популярный тогда браузер: язык добавлял интерактивность на страницы — сначала в основном визуальные эффекты. Со временем на нём стали строить всё более сложную логику. Поворотным моментом стал [Gmail](https://gmail.com/) с активным использованием AJAX. Роль JavaScript на сайтах росла, и каждый крупный вендор браузера делал свой интерпретатор.

Позже JavaScript вышел за пределы браузера: появился серверный Node.js, затем использование во встраиваемых системах. Сегодня на JavaScript можно писать и для бытовой техники.

## ECMAScript

Много разных движков расширило применение языка, но создало проблемы совместимости: один и тот же код мог вести себя по-разному. Для единообразия появился стандарт **ECMAScript** — большой документ о поведении языка. Когда что-то в JavaScript кажется неочевидным, ответ обычно есть в спецификации.

ECMAScript развивается: каждая версия добавляет возможности — новый синтаксис или функции стандартной библиотеки. Стандарты в основном обратно совместимы, но не всегда без сюрпризов: поведение со временем часто становится строже.

Наличие возможности в стандарте не гарантирует её во всех движках: спецификация и реализации живут своей жизнью. Иногда фича сначала появляется в одном движке, набирает популярность и только потом попадает в стандарт. Код под новый стандарт на старом интерпретаторе часто падает с синтаксической ошибкой.

На сайтах вроде [compat-table](https://compat-table.github.io/compat-table/es6/) и [caniuse](https://caniuse.com/es6) видно, какие возможности поддерживают разные среды. Разработчики смотрят на них, когда решают, можно ли безопасно использовать новинки.

## Самостоятельная работа

1. [Создайте на гитхабе репозиторий](https://www.youtube.com/watch?v=ku3Mg_XCsAo) с именем my*-js* и файлом README

1. Склонируйте его в домашнюю директорию на своем компьютере

1. Добавьте в *README.md* текст "Мой первый проект на JavaScript"

1. Добавьте все изменения на гитхаб
