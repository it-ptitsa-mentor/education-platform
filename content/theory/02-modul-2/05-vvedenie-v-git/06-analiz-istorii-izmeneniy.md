Программирование — не только написание нового кода, но и разбор уже существующего. Иногда код понятен без комментариев — это хороший знак.

Но не всегда: остаются вопросы — почему так, кто правил и когда.

Ответ даёт история изменений. Понятные коммиты с одной законченной правкой превращают историю в инструмент анализа — поэтому важно понимать идеи Git и придерживаться практик.

У Git много команд и опций для извлечения информации из истории — познакомимся с основными.

## Команда `git log`

Базовый обзор — `git log`: список коммитов по дате, сверху самые новые:

```
git log

# Ниже неполный вывод истории проекта

commit 5120bea3e5528c29f8d1da43731cbe895892eb6d
Author: tirion <tirion@got.com>
Date:   Thu Sep 17 18:04:19 2020 -0400

    add new content

commit e6f625cf8433c8b1f1aaed58cd2b437ec8a60f27
Author: tirion <tirion@got.com>
Date:   Thu Sep 17 16:14:09 2020 -0400

    add INFO.md

commit 273f81cf2117044f1973ea80ce1067a94bea3f80
Author: tirion <tirion@got.com>
Date:   Thu Sep 17 16:08:39 2020 -0400

    remove NEW.md

# Этот вывод показывается через пейджер
```

Видно, кто, когда и с каким сообщением коммитил. При аккуратных сообщениях уже многое ясно; оформление коммитов разберём отдельно.

Полезен флаг `-p` — сразу показывает diff каждого коммита:

```
git log -p
# Тут все коммиты с полным дифом
# Промотать вперед — кнопка f, промотать назад — b
# Выйти из режима просмотра — q
```

## Команда `git show`

У каждого коммита есть уникальный **идентификатор** (хеш). По нему можно посмотреть все изменения внутри одного коммита:

```
git show 5120bea3e5528c29f8d1da43731cbe895892eb6d
# Тут выводится диф между этим коммитом и предыдущим

diff --git a/INFO.md b/INFO.md
index d5225f8..40f51f1 100644
--- a/INFO.md
+++ b/INFO.md
@@ -1 +1,2 @@
 git is awesome!
+new line
diff --git a/README.md b/README.md
index ffe7ece..00fd294 100644
--- a/README.md
+++ b/README.md
@@ -1 +1 @@
-Hello, Git!
+Hello, Git! How are you?
# Здесь мы видим, что было изменено этим коммитом
```

Полные хеши длинные; Git позволяет указывать только начало — часто хватает первых семи символов:

```
git show 5120bea
```

Многие команды сами выводят укороченный хеш. Первые семь символов обычно уникальны в рамках репозитория.

## Команда `git blame`

Если коммит неизвестен, но нужно узнать, кто последним трогал строку в файле, подойдёт `git blame <путь до файла>`. Команда выводит файл построчно с автором и коммитом:

```
git blame INFO.md

e6f625cf (tirion 2020-09-17 16:14:09 -0400 1) git is awesome!
5120bea3 (tirion 2020-09-17 18:04:19 -0400 2) new line
```

Правка строки не равна её первоначальному написанию: автор мог поправить опечатку, а строку создал другой разработчик. С хешем из вывода можно открыть конкретный коммит.

## Команда `git grep`

`git grep` ищет строку во всех файлах проекта — удобно из терминала. Он учитывает `.gitignore`, не лезет в `.git` и умеет искать по истории:

```
git grep line

INFO.md:new line

# Флаг `i` позволяет искать без учета регистра
git grep -i git

README.md:Hello, Git! How are you?

# Поиск в конкретном коммите
git grep Git 5120bea

# Поиск по всей истории
# Возвращаем список хешей коммитов `rev-list`
git grep Git $(git rev-list --all)
```
