# Введение

<!-- IMG (из Buildin, перезалить отдельно) -->
Независимо от языка и направления разработки, итоговый код — это текст в множестве файлов. Файлы постоянно создаются, удаляются и правятся — и вместе с этим возникают типичные вопросы:

- Как не потерять исходники?

- Как вспомнить, что менялось недавно и где?

- Как защититься от случайных правок и удалений?

- Как откатить изменения, если они оказались неверными?

- Как нескольким разработчикам работать над одним кодом одновременно?

Представьте проект из сотни файлов и десятков тысяч строк. Вы делаете задачу, трогаете 15 файлов и 300 строк — и внезапно задача отменяется. Нужно вернуть код к состоянию «до начала работы».

Другой случай: в процессе одной задачи срочно нужно исправить баг в текущей версии проекта. Незавершённую работу выкладывать нельзя — значит, правки вносят в код, который был актуален до старта первой задачи.

Совместная разработка добавляет сложности. Два человека правят одни и те же файлы — как не затереть чужие изменения?

Для таких задач (и многих других) используют [Git](https://git-scm.com/) — систему контроля версий. Это инструмент, который фиксирует изменения файлов и помогает работать над кодом в одиночку и в команде.

В этом уроке начнём знакомство с Git.

## Что такое Git

Сегодня Git — стандартный инструмент, с которого начинается большинство проектов. Этот курс тоже вёлся с его помощью.

Базовые возможности Git:

- Возврат к любой прошлой версии кода

- Просмотр истории изменений

- Восстановление данных

- Совместная работа без риска потерять или перезаписать чужие правки

Git встроен в редакторы кода. Экосистема инструментов строится вокруг Git и хостингов вроде [GitHub](https://github.com/) или [GitLab](https://about.gitlab.com/).

Обычно код хранят на таких сервисах: это и общий доступ для команды, и резервная копия на случай поломки локальной машины.

Git — большая система с множеством команд и сценариев. Даже опытные разработчики регулярно заглядывают в документацию. Кроме официальной справки, полезна книга [Pro Git](https://git-scm.com/book/ru/v2).

При этом начать пользоваться Git несложно: есть «ядро» команд и идей, которого хватает в большинстве случаев.

Именно с этим ядром мы и будем работать в курсе. Важно помнить: Git понимают на практике — через ошибки и повторение. Одной только теории мало.

В курсе установим Git, разберём основные команды, создадим первый репозиторий и отработаем типичные операции: добавление, изменение и удаление файлов, просмотр истории. Также познакомимся с хостингами и с тем, как устроено открытое ПО.

Цель курса — дать фундамент и показать направления; дальше помогают только эксперименты.

Часто Git активно осваивают уже на работе — и процесс бывает болезненным: на учебных проектах сложно воспроизвести реальные командные ситуации.

Выигрывают те, кто во время обучения участвует в открытых проектах — это один из лучших способов по-настоящему освоить Git.

Перед стартом рекомендуем [установить и настроить Git](https://git-scm.com/book/ru/v2/%D0%9D%D0%B0%D1%87%D0%B0%D0%BB%D0%BE-%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D1%8B-%D0%A3%D1%81%D1%82%D0%B0%D0%BD%D0%BE%D0%B2%D0%BA%D0%B0-Git) по официальной инструкции.
