# Открытые проекты

Как и большинство программистов, мы каждый день пользуемся ПО с открытым исходным кодом. В уроке — что это такое и чем open source полезен разработчику.

**Проекты с открытым исходным кодом** (**Open Source**, *OSS*) — программы, код которых доступен публично. Любой может его посмотреть; часто — и повлиять на развитие.

Сюда входят:

- Прикладное ПО и сервисы — VS Code, Git

- Множество библиотек в наших проектах

- Практически все современные языки программирования

Масштабы впечатляют:

- Linux (> 12 000 000 строк кода)

- Chromium (> 16 000 000 строк кода)

Такая скорость разработки возможна благодаря open source.

Откуда берутся такие проекты и почему они популярны? Часто код — побочный продукт другой работы. При разработке Linux понадобилась удобная система контроля версий — так появился Git.

Почему код открыли? Удачные проекты притягивают контрибьюторов: багрепорты, патчи, иногда постоянная разработка — часто без оплаты. Люди растут как специалисты, проще найти работу. Коллеги без участия в open source обычно реже в таких историях.

Если сравнить с прошлым, число участников open source выросло на порядки.

Раньше мешали сложный вход и жёсткие правила приёма кода — на конференциях по часу рассказывали, как принимать людей и патчи.

Сейчас присоединиться можно за минуты, почти без барьеров. Главные причины — Git и GitHub.

Ключевой механизм GitHub — **запрос на включение изменений**, **pull request**. Он сильно упростил участие в чужих проектах.

Допустим, в библиотеке нашли ошибку в коде или документации. Шаги до принятия патча мейнтейнерами:

## Клонирование

Сначала — копия репозитория в вашем аккаунте. На странице репозитория — кнопка *Fork*:

<!-- IMG (из Buildin, перезалить отдельно) -->
В аккаунте появится репозиторий с тем же именем. GitHub помечает его как форк оригинала.

Дальше как обычно: клонируете на компьютер, вносите правки. Хорошая практика — отдельная ветка от *main*.

## Запрос на включение изменений кода

После push интерфейс GitHub меняется. На странице форка появится *Pull request*:

<!-- IMG (из Buildin, перезалить отдельно) -->
По клику — форма с названием и описанием PR.

После отправки PR появится в оригинальном репозитории на вкладке *Pull requests*. Дальше мейнтейнеры могут:

- Принять PR

- Отклонить

- Задать вопросы и попросить доработать

## Исправления прямо на GitHub

Для мелочей — опечатка в тексте — PR можно сделать в браузере.

Откройте файл, нажмите редактирование, сохраните — GitHub предложит создать pull request с изменением.
