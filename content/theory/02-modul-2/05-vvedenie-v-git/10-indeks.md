# Индекс

**Индекс** в Git — промежуточная область между рабочей директорией и репозиторием. В коммит попадают только изменения, добавленные в индекс. В этом уроке — зачем он нужен и как с ним работать.

Индекс не случаен. Даже над одной задачей разработчик натыкается на разное в коде:

- Ошибки

- Плохое оформление

- Участки, которые потом придётся переделать под новые требования

Так и делают — правят по ходу. В рабочей директории копятся правки: часть относится к задаче, часть — побочные исправления. В чём проблема?

Если закоммитить всё одним коммитом, получаем несколько неприятных эффектов.

Во-первых, история становится шумной. В одном коммите оказываются несвязанные правки — отвлекают при ревью:

```
# Обычно в таких коммитах встречается условие И в описании
# Это показатель того, что в рамках одного коммита сделано несколько изменений
git commit -m 'add new feature and fix something'
```

Во-вторых, откат такого коммита уберёт и те правки, которые всё равно нужны.

Здесь помогает **индекс**. С ним можно не бояться, что в коммит попадёт лишнее.

Стандартный цикл — добавить или изменить файлы и закоммитить:

```
git add somefile
git commit -m 'add somefile'
```

Если нужно закоммитить один-два уже отслеживаемых файла, можно короче:

```
echo 'new data' >> INFO.md
# Не нужно явно вызывать команду `git add`
git commit INFO.md -m 'update INFO.md'
```

Иногда наоборот — много файлов, и хочется добавить всё сразу. Поможет точка:

```
# Добавляет абсолютно все изменения рабочей директории в индекс
git add .
```

Команда опасна: легко закоммитить лишнее, особенно без `git diff --staged` перед коммитом.

Ещё сильнее — коммит с автоматическим добавлением всего в индекс:

```
# Флаг -a автоматически добавляет все изменения рабочей директории в индекс
git commit -am 'do something'
```

Часто правки затрагивают одни и те же файлы, но логически должны уйти в разные коммиты.

И это тоже возможно. Команда `git add -i` показывает фрагменты изменений и спрашивает, что с ними делать:

[Terminal session recording](https://asciinema.org/a/TCGpKR6sojvHhW1bo15MMc6g9/iframe?cols=130&rows=25)

Так можно точно выбрать, что войдёт в коммит. Уверенное владение Git часто начинается с этого.
