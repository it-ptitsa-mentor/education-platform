# Проверка существования значения

При работе с массивами часто возникает «выход за границу» — обращение к несуществующему индексу:

```
const animals = ['cats', 'dogs', 'birds']
// Элемента с индексом 5 не существует
animals[5] // undefined
```

В разных языках поведение при таком обращении различается: где-то ошибка, где-то нет, а в Си иногда читаются случайные байты соседней памяти — с катастрофическими последствиями.

В JavaScript подход мягче: обращение за пределы даёт `undefined`, без исключения — это считается нормой:

```
const animals = ['cats', 'dogs', 'birds']

// Выход за границы массива
animals[5] // undefined
animals[4] // undefined
animals[3] // undefined

// Ура, мы попали в границы массива :)
animals[2] // 'birds'
```

В большинстве случаев выход за границу — признак логической ошибки. Программа может продолжать работать и иногда даже выдавать верный результат. Простая проверка: индекс должен быть строго меньше длины:

```
// Важно что <, а не <=.
// потому что такого индекса нет items[items.length]
if (index < items.length) {
  items[index] // все отлично!
}
```

С опытом такие ситуации замечаются быстрее, но и опытные разработчики периодически ошибаются с индексами.
