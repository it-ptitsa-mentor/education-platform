# Массивы в памяти компьютера

На высокоуровневом JavaScript можно решать повседневные задачи, не зная устройства массивов в памяти. Но такое понимание снимает «магию» и помогает заглядывать глубже.

## Массивы в Cи

Нагляднее всего «настоящие» массивы видны в **Cи**: язык простой и близок к железу, почти не скрывает деталей. У примитивов вроде числа или строки интуитивно ясно: под значение выделяется фиксированный объём памяти по типу. А как выделяется память под массив? Что массив *есть* в памяти? На уровне хранения отдельного типа «массив» нет — это сплошной блок, размер которого считают так: *число элементов × размер одного элемента*. Отсюда два вывода:

- Размер массива — фиксированная величина. Те динамические массивы (изменяющие свой размер во время работы), с которыми мы имеем дело во многих языках, реализованы уже внутри языка, а не на уровне железа.

- Все элементы массива имеют один тип и занимают одно и то же количество памяти. Благодаря этому появляется возможность простым умножением (по формуле, описанной выше) получить адрес той ячейки, в которой лежит нужный нам элемент. Именно это происходит под капотом, при обращении к элементу массива под определенным индексом.

Индекс — смещение от начала блока памяти массива. Адрес элемента: *индекс × размер одного элемента (для типа на данной архитектуре)*. Пример на Си:

```
// Инициализация массива из пяти элементов типа int
// Предположим, что int занимает 2 байта
// Общее количество памяти, выделенное под массив: int * 5 = 2 * 5 = 10 байт
int numbers[] = {19, 10, 8, 17, 9};
numbers[3]; // 17
```

Если `int` занимает 2 байта, адрес элемента с индексом `3`: *начальный адрес + 3 × 2*. Начальный адрес — куда положили массив при выделении. Пример для `numbers`:

```
// Первый элемент
// Начальный адрес + 2 * 0 = начальный адрес
numbers[0]; // 19

// Начальный адрес + 2 * 1 = начальный адрес + 2
// То есть сместились на 2 байта
numbers[1]; // 10

// Начальный адрес + 2 * 2 = начальный адрес + 4
// То есть сместились на 4 байта
numbers[2]; // 8

// Последний элемент
// Начальный адрес + 2 * 4 = начальный адрес + 8
// То есть сместились на 8 байт
// И сам элемент занимает 2 байта. В сумме как раз 10
numbers[4]; // 9
```

Поэтому индексы с нуля: 0 — нулевое **смещение**.

Не все данные одного размера. Как хранить массив строк? Длины разные, памяти нужно разное количество. В Cи строки часто кладут как массив массивов (строка в C — массив символов). Вложенные массивы должны иметь один размер — физическое ограничение. Выбирают длину с запасом под самые длинные строки:

```
// Массив из трех элементов, внутри которого массивы по 10 элементов
// Это значит, что здесь можно хранить 3 строки длиной не больше 10 символов
char strings[3][10] = {
   "spike",
   "tom",
   "jerry"
};

strings[0]; // spike
```

### Безопасность

В высокоуровневых языках выход за границы массива обычно ловят; в **C** обращение за пределами не всегда даёт ошибку (иногда segfault, но не всегда). По индексу за границей читают то, что лежит по вычисленному адресу — содержимое непредсказуемо (интерпретируется как тип массива: для `int` вернётся число). За этим охотятся при взломе ПО.

## Массивы в динамических языках

В JavaScript и других динамических языках устройство массивов сложнее: типы определяются в рантайме, и массив не может работать как в C — заранее неизвестно, что окажется внутри.

Такие массивы хранят не сами значения, а ссылки (адреса) на них. Тогда не важен размер данных: в ячейке лежит адрес фиксированной длины. Гибко, но медленнее.

К тому же массивы динамические: длина может расти и уменьшаться. Если ссылкам тесно, рантайм выделяет больший блок (часто вдвое) и переносит ссылки. Удобно для разработки, но снова цена — производительность.
