# Стек

Алгоритмы не существуют в вакууме: важно не только построить верное решение, но и выбрать подходящую структуру для данных.

**Структура данных** — конкретный способ хранить и упорядочивать информацию. Для разных задач удобны разные организации. Как минимум одну вы уже знаете хорошо — массив. Логически это набор элементов с индексированным доступом; физически в памяти устройство сложнее — массивы бывают разными и внутри языка реализуются [по-разному](https://ru.wikipedia.org/wiki/%D0%9C%D0%B0%D1%81%D1%81%D0%B8%D0%B2_(%D1%82%D0%B8%D0%BF_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85)).

Кроме массивов есть списки, хеш-таблицы, деревья, графы, стек, очередь и другие. Подходящая структура заметно упрощает код и убирает запутанную логику.

Часть из них разберём в курсах и проектах учебной платформы; остальное придётся добирать из книг. В любом случае алгоритмы и структуры данных (без фанатизма) — фундамент, на который опирается остальная разработка.

Полезно различать три понятия:

- Структура данных

- Конкретный тип данных (или просто "тип данных")

- Абстрактный тип данных (АТД)

Со структурой данных всё ясно — определение выше. С типом данных тоже: *числа в JavaScript — это тип данных*. «Тип» всегда привязан к языку и может называться как угодно. Если бы создатели JS назвали числа String, никто бы не запретил — даже если имя выглядело бы абсурдно.

Абстрактные типы данных — теоретическая конструкция. АТД задаётся набором допустимых операций и ничего не говорит о хранении: он существует на бумаге и в голове. В языках же есть конкретные типы, которые эти АТД реализуют. АТД часто путают со «структурой данных», и названия у них нередко совпадают. Углубляться в терминологию не обязательно — достаточно общей картины.

В этом уроке разберём один из простейших и важных АТД — стек (stack, «стопка»).

## Стек

Стек — упорядоченная коллекция, где добавление и удаление всегда происходит с одного конца, обычно называемого вершиной.

<!-- IMG (из Buildin, перезалить отдельно) -->
Аналоги из жизни: пирамида колец. Кольца кладут друг на друга и снимают в обратном порядке — последнее снятое было положено последним.

<!-- IMG (из Buildin, перезалить отдельно) -->
Почти любая стопка ведёт себя как стек. Без грубой силы с ней работают двумя способами: положить элемент сверху или снять с верхушки. Отсюда название LIFO — Last In First Out, «последним зашёл, первым вышел».

Стек неразрывно связан с исполнением программ. Функции вызывают функции, те — следующие, и так до самой глубины. Когда глубочайшая завершается, начинается обратный ход: сначала выход из вложенных, потом из вызывающих, до самого верха. Вызов — это push в стек, возврат — pop. Так устроено и на аппаратном уровне; при ошибке выводят *Stack Trace* — трассировку стека.

Ещё пример — кнопка «назад» в браузере: история как стек, каждый переход добавляет запись, «назад» снимает последнюю.

Операции со стеком:

- Добавить в стек (push)

- Взять из стека (pop)

- Вернуть элемент с вершины стека без удаления (peekBack)

- Проверить на пустоту (isEmpty)

- Вернуть размер (size)

Первые две — базовые, остальные нужны по ситуации. В JavaScript стек удобно моделировать массивом: `push()`, `pop()` и `length`.

```
const stack = []

stack.push(3)
console.log(stack) // => [ 3 ]
stack.push('Winterfall')
console.log(stack) // => [ 3, 'Winterfall' ]
stack.push(true)
console.log(stack) // => [ 3, 'Winterfall', true ]

const element1 = stack.pop()
console.log(element1) // => true
console.log(stack) // => [ 3, 'Winterfall' ]

const element2 = stack.pop()
console.log(element2) // => Winterfall
console.log(stack) // => [ 3 ]
```

`pop()` и `push()` меняют исходный массив. `pop()` ещё и возвращает снятый элемент.

Разберём задачу, где стек даёт простое решение — её часто дают на собеседованиях, чтобы проверить знание базовых структур.

Задача:

Нужна функция, проверяющая баланс парных скобок: у каждой открывающей есть закрывающая — `()`, `((()()))`. Несбалансированные примеры: `(`, `(()`, `)(`. Одного подсчёта количества мало, важен порядок.

*Есть решение и без стека, но стек здесь нагляден и уместен.*

Алгоритм со стеком:

1. Если перед нами открывающий элемент, то заносим его в стек

1. Если закрывающий, то достаем из стека элемент (очевидно, последний добавленный). Если стек пустой, то есть в нем нет открывающих скобок, значит выражение не соответствует требуемому формату.

1. Если мы дошли до конца строки и стек пустой, то все хорошо. Если в стеке остались элементы, то проверка не прошла. Такое может быть, если в начале строки были открывающие элементы, но в конце не было закрывающих.

Построчно:

```
const checkIsBalanced = (expression) => {
  // Инициализация стека
  const stack = []
  // Проходим по каждому символу в строке
  for (const symbol of expression) {
    // Смотрим, открывающий или закрывающий
    // Если символ открывающий
    if (symbol === '(') {
      // Добавляем его в стек
      stack.push(symbol)
    // Если символ закрывающий
    }
    else if (symbol === ')') {
      // Достаем из стека последний элемент
      // Если стек пуст, значит для закрывающего не нашлось открывающего
      // Значит баланса нет, возвращаем false
      if (!stack.pop()) {
        return false
      }
    }
  }

  // Проверяем, что стек пуст
  // Если в стеке остались элементы, то не все открывающие скобки закрыты,
  // а значит баланса нет
  return stack.length === 0
}

export default checkIsBalanced
```

Для строки `(()` ключевые шаги:

1. Первая скобка `(` заносится в стек, так как она открывающая

1. Следующая скобка `(` также заносится в стек по той же самой причине

1. Последняя `)` является закрывающей, из стека извлекается последняя добавленная скобка

1. В стеке остался один элемент, значит баланса нет

## Семантика

Может возникнуть соблазн применять эти методы в быту — например, взять последний элемент массива. `pop()` формально это делает, но так делать не стоит:

1. Побочный эффект данной операции — изменение исходного массива. Даже если массив потом не используется, такой код вносит потенциальные проблемы и заставляет его переписывать в будущем.

1. Нарушается семантика. Инструменты нужно использовать по назначению, иначе рождается код, который декларирует одно, но в реальности делает другое. Любой опытный программист, который видит `pop()` сразу считает, что массив в данной части программы используется как стек. Если это не так, то понадобятся дополнительные умственные усилия для понимания происходящего.

Для извлечения последнего элемента можно использовать метод массива [at()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/at)

```
const items = ['first', 'second', 'third']
items.at(-1) // 'third'
```
