# Сортировка массивов

Сортировка — базовая алгоритмическая тема на собеседованиях. В продакшене обычно вызывают готовый `sort()`:

```
const numbers = [8, 3, 10]
// sort изменяет массив!
numbers.sort((a, b) => a - b) // сортировка по возрастанию
console.log(numbers) // => [3, 8, 10]

// В обратную сторону можно через reverse() выполненный после sort()
// Тоже изменяет массив
numbers.reverse()
console.log(numbers) // => [10, 8, 3]
```

Зачем тогда спрашивают с нуля? Обычно проверяют:

1. Знаете ли вы, что алгоритмы существуют

1. Умеете ли писать код сами

1. Как устроено алгоритмическое мышление

Алгоритмы влияют на то, как вы думаете. Не нужно знать всё — но ключевые стоит понимать и по возможности реализовать.

Роберт Мартин в «Идеальном программисте» описывает [Ката](https://ru.wikipedia.org/wiki/%D0%9A%D0%B0%D1%82%D0%B0) из боевых искусств:

> Принцип изучения боевого искусства на основе ката состоит в том, что повторяя ката многие тысячи раз, практик боевого искусства приучает свое тело к определенного рода движениям, выводя их на бессознательный уровень. Таким образом, попадая в боевую ситуацию, тело работает "само" на основе рефлексов, вложенных многократным повторением ката. Также считается, что ката обладают медитативным воздействием.

## Сортировка

Алгоритмов сортировки много. Для обучения популярен [пузырьковый sort](https://ru.wikipedia.org/wiki/%D0%A1%D0%BE%D1%80%D1%82%D0%B8%D1%80%D0%BE%D0%B2%D0%BA%D0%B0_%D0%BF%D1%83%D0%B7%D1%8B%D1%80%D1%8C%D0%BA%D0%BE%D0%BC) (bubble sort).

Идея: повторные проходы; соседние элементы сравнивают и меняют местами при неверном порядке. Проходы повторяют, пока за проход не было обменов. На каждом проходе «всплывает» максимум к концу.

```
// Функция изменяет входящий массив coll
const bubbleSort = (coll) => {
  let stepsCount = coll.length - 1
  // Объявляем переменную swapped, значение которой показывает,
  // был ли совершен обмен элементов во время перебора массива
  let swapped
  // do..while цикл. Работает почти идентично while
  // Разница в проверке. Тут она делается не до выполнения тела, а после.
  // Такой цикл полезен там, где надо выполнить тело хотя бы раз в любом случае.
  do {
    swapped = false
    // Перебираем массив и меняем местами элементы, если предыдущий
    // больше, чем следующий
    for (let i = 0; i < stepsCount; i += 1) {
      if (coll[i] > coll[i + 1]) {
        // temp – временная константа для хранения текущего элемента
        const temp = coll[i]
        coll[i] = coll[i + 1]
        coll[i + 1] = temp
        // Если сработал if и была совершена перестановка,
        // присваиваем swapped значение true
        swapped = true
      }
    }
    // Уменьшаем счетчик на 1, т.к. самый большой элемент уже находится
    // в конце массива
    stepsCount -= 1
  } while (swapped) // продолжаем, пока swapped === true

  return coll
}

console.log(bubbleSort([3, 2, 10, -2, 0])) // => [ -2, 0, 2, 3, 10 ]
```

Два уровня:

1. Внутренний `for` — попарные обмены

1. Внешний `do...while` — остановка; в худшем случае до `coll.length` итераций

Пузырёк прост и нагляден — полезно уметь написать по памяти. Попробуйте без подглядывания в теорию.
