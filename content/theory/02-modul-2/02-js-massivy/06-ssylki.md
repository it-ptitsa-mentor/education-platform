# Ссылки

В JavaScript переменные и константы хранят данные двух родов: примитивные и ссылочные. К примитивам относятся числа, строки, булевы значения и остальные примитивные типы. К ссылочным — объекты. Объекты как класс сущностей разбираются в следующем курсе, но массив внутри тоже объект:

```
typeof [] // 'object'
```

Чем ссылочные типы отличаются от примитивов и зачем это знать?

Для практики разница видна при изменении данных, передаче в функции и возврате из них. Массив в `const` можно менять — как раз из-за ссылочной природы: константа держит ссылку на массив, а не копию содержимого; ссылка не меняется, содержимое — может. С примитивами так не выйдет.

Убедиться, что массивы — ссылки, можно так: две константы указывают на один массив, и изменение через одну видно в другой:

```
const items = [1, 2]
// Ссылаются на один и тот же массив
const items2 = items
items2.push(3)

console.log(items2) // => [1, 2, 3]
console.log(items) // => [1, 2, 3]
items2 === items // true
```

Сравнение массивов — по ссылке. Структурно одинаковые массивы могут быть не равны:

```
[1, 2, 3] === [1, 2, 3] // false
```

Если передать массив в функцию, которая его меняет, изменится и исходный — в функцию уходит ссылка:

```
const addElement = coll => coll.push('wow')

const items1 = ['one']
addElement(items1)
console.log(items1) // => [ 'one', 'wow' ]
addElement(items1)
console.log(items1) // => [ 'one', 'wow', 'wow' ]

const changeElement = (coll, index) => {
  coll[index] = 'changed'
}

const items2 = ['one', 'two', 'three']
changeElement(items2, 1) // => [ 'one', 'changed', 'three' ]
changeElement(items2, 2) // => [ 'one', 'changed', 'changed' ]
console.log(items2) // => [ 'one', 'changed', 'changed' ]
```

## Проектирование функций

При работе с массивами в функциях есть два подхода: мутировать переданный массив или собрать новый и вернуть его. В большинстве случаев лучше второй — безопаснее, предсказуемее, нет скрытых побочных эффектов.

Мутация может удивить. Функция `last()`, которая «достаёт» последний элемент, могла бы быть такой:

```
// Метод .pop извлекает последний элемент из массива
// Он изменяет массив, удаляя оттуда этот элемент
const last = coll => coll.pop()
```

Вы хотели лишь посмотреть последний элемент — а он ещё и удалился. Для такой задачи это неожиданно и ломает ожидания от хорошего API (см. принцип CQS в курсе по функциям). Уместнее:

```
// Метод .at() возвращает элемент массива по указанному индексу
// Он не меняет сам массив
// Индекс -1 означает первый элемент с конца
const last = coll => coll.at(-1)
```

Когда же мутировать уместно? Чаще всего ради производительности — поэтому встроенные `reverse()` и `sort()` меняют массив на месте:

```
const items = [3, 8, 1]

// Нет присвоения результата, массив изменяется напрямую
items.sort()
console.log(items) // => [1, 3, 8]

items.reverse()
console.log(items) // => [8, 3, 1]
```

В документации обычно явно пишут, мутирует ли метод исходный массив. Например, [concat()](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Array/concat) возвращает новый массив, в отличие от `sort()`.

Мутация сложнее в отладке, но в ряде языков её используют ради эффективности: копирование очень большого массива дорого. В типичной веб-разработке это редко проблема, но знать полезно.
