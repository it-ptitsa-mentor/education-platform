# Деструктуризация

Деструктуризация (destructuring) — синтаксис, который «раскладывает» элементы массива (и не только) в отдельные константы или переменные. Это необязательная, но очень удобная возможность языка. Разберём на примерах.

Допустим, есть массив из двух чисел, и в коде нужны оба значения. Простейший путь — обращаться по индексу `point[0]` и `point[1]`.

```
const point = [3, 5]

console.log(`${point[0]}:${point[1]}`) // => 3:5
```

Индексы мало что говорят о смысле — придётся вчитываться. Лучше сначала присвоить значения переменным с говорящими именами:

```
const x = point[0]
const y = point[1]

console.log(`${x}:${y}`) // => 3:5
```

Код читается проще, хотя строк стало больше. Деструктуризация делает то же короче:

```
const [x, y] = point
// Слева массив повторяет структуру правого массива
// но вместо значений используются идентификаторы
// они заполняются значениями, стоящими на тех же позициях в правом массиве
// [x, y] = [3, 5]
// x = 3, y = 5

console.log(`${x}:${y}`) // => 3:5
```

Короче и понятнее (особенно когда привыкнете). Можно вытащить из массива почти любую часть — с начала, с конца, с пропусками:

```
// Извлекаем первый элемент
// Остальные игнорируются
const [x] = point

// Извлекаем второй элемент
// Для этого просто не указываем первый
const [, y] = point

// и даже так
const [, secondElement, , fourthElement, fifthElement] = [1, 2, 3, 4, 5, 6]

console.log(secondElement) // => 2
console.log(fourthElement) // => 4
console.log(fifthElement) // => 5
```

Из `[1, 2, 3, 4, 5, 6]` взяты второй, четвёртый и пятый элементы в `secondElement`, `fourthElement`, `fifthElement`. На местах первого и третьего — пропуски: эти значения не нужны.

Если переменной не соответствует элемент, попадёт `undefined`:

```
const [firstElement, secondElement, thirdElement] = [1, 2]

console.log(firstElement) // => 1
console.log(secondElement) // => 2
console.log(thirdElement) // => undefined
```

В массиве только два элемента, поэтому `thirdElement` — `undefined`. Можно задать значение по умолчанию:

```
const [firstElement, secondElement, thirdElement = 3] = [1, 2]

console.log(firstElement) // => 1
console.log(secondElement) // => 2
console.log(thirdElement) // => 3
```

Третьего элемента нет — в `thirdElement` записалось значение по умолчанию `3`.

Деструктуризация работает на любой глубине вложенности:

```
const [one, [two, three]] = [1, [2, 3]]
```

Комбинаций много — чем больше экспериментируете, тем больше находите удобных приёмов.

## Деструктуризация в циклах

Разложение массива можно использовать не только отдельной инструкцией, но и в циклах:

```
const points = [
  [4, 3],
  [0, -3],
]

for (const [x, y] of points) {
  console.log([x, y])
}

// => [ 4, 3 ]
// => [ 0, -3 ]
```

Каждый элемент `points` — массив. Без деструктуризации:

```
for (const item of points) {
  console.log(item)
}
```

Внутри `for` переменная `item` - это массив, поэтому вместо нее можно подставить деструктуризацию `[x, y]`.

Такое разложение допустимо везде, где ожидается массив: параметры функций, возвращаемые значения, циклы и другие места, с которыми вы столкнётесь в коде.

```
const swapValues = ([a, b]) => [b, a]

swapValues([1, 2]) // [2, 1]
```

## Деструктуризация строк

В JavaScript строки ведут себя подобно массивам и их также можно деструктурировать.

```
const [first, second, third] = 'two'
console.log(first) // => 't'
console.log(second) // => 'w'
console.log(third) // => 'o'
```
