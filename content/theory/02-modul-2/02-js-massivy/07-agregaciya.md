# Агрегация

Частый сценарий для циклов с массивами — **агрегация**: вычисления по всему набору данных — максимум, среднее, сумма и т.п. Нового синтаксиса это не требует, но меняет ход алгоритма, поэтому выделим отдельно. Начнём с поиска максимума:

```
const calculateMax = (coll) => {
  // Если коллекция пустая, то у нее не может быть максимального значения
  // В подобных ситуациях принято возвращать null
  // Это классический пример использования идиомы guard expression
  if (coll.length === 0) {
    return null
  }

  // Сравнение элементов нужно начать с какого-то первого элемента
  let max = coll[0] // Принимаем за максимальное значение первый элемент
  // Обход начинаем со второго элемента
  for (let i = 1; i < coll.length; i += 1) {
    const currentElement = coll[i]
    // Если текущий элемент больше максимального,
    // то он становится максимальным
    if (currentElement > max) {
      max = currentElement
    }
  }

  // Не забываем вернуть максимальное число
  return max
}

console.log(calculateMax([])) // => null
console.log(calculateMax([3, 2, -10, 38, 0])) // => 38
```

Здесь агрегация — **вычисление**, где сравниваются все элементы, чтобы получить один итог.

Заметьте: стартовый `max` — первый элемент, а не `0`. Иначе при всех отрицательных числах ответ будет неверным.

## Нейтральный элемент

Теперь сумма:

```
const calculateSum = (coll) => {
  // Начальное значение суммы
  // Используем переменную, так как в процессе работы сумма будет меняться
  let sum = 0
  for (let i = 0; i < coll.length; i += 1) {
    // Поочередно складываем все элементы
    sum += coll[i]
  }

  return sum
}

// Сумма элементов всегда возвращает какое-то число
// Если массив пустой, то сумма его элементов равна нулю
console.log(calculateSum([])) // => 0

console.log(calculateSum([3, 2, -10, 38, 0])) // => 33
// Процесс вычислений
let sum = 0
sum = sum + 3 // 3
sum = sum + 2 // 5
sum = sum + -10 // -5
sum = sum + 38 // 33
sum = sum + 0 // 33
```

У суммы есть нюансы.

Сумма пустого массива? В математике и по смыслу — `0`: нет элементов — ноль яблок. Функции ведут себя так же.

Второй момент — начальное значение `sum`. Повторяющаяся операция должна откуда-то стартовать; без начала получится ошибка:

```
  // Начальное значение не задано
  // Поэтому JavaScript автоматически считает, что начальное значение равно undefined
  let sum

  // Первая итерация цикла
  sum = sum + 2 // ?
```

В итоге `NaN` — попытка сложить `2` и `undefined`. Нужно начальное значение; `0` подходит для суммы. Если взять `1`, результат сдвинется на единицу.

В математике есть [**нейтральный элемент бинарной операции**](https://ru.wikipedia.org/wiki/%D0%9D%D0%B5%D0%B9%D1%82%D1%80%D0%B0%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D1%8D%D0%BB%D0%B5%D0%BC%D0%B5%D0%BD%D1%82) — тот, кто не меняет результат (`x + 0 = x`). Сумму `3 + 2 + 8` можно считать как `0 + 3 + 2 + 8` — так устроен наш код.

Нейтральный элемент важен для любой агрегации: для сложения это `0`, для умножения — `1`, для конкатенации строк — `''` (`'' + 'one'` → `'one'`).

Агрегация не обязана сводить коллекцию к одному примитиву — результатом может быть и массив (например, список уникальных слов в тексте).
