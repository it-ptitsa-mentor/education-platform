# Цикл for...of

`for` — низкоуровневый цикл: нужны счётчик, шаг и условие выхода. Удобнее было бы обходить элементы напрямую. В JavaScript для этого есть `for...of`.

```
const userNames = ['petya', 'vasya', 'evgeny']

// name на каждой итерации свой собственный (локальный), поэтому используется const
for (const name of userNames) {
  console.log(name)
}
// => "petya"
// => "vasya"
// => "evgeny"
```

С `for...of` код короче и яснее: цикл сам знает, как идти по элементам и где остановиться. Вместо `userNames[i]` на каждой итерации появляется `const name` с значением текущего элемента; область видимости — тело цикла.

Для агрегации это тоже удобно:

```
const calculateSum = (coll) => {
  let sum = 0
  for (const value of coll) {
    sum += value
  }

  return sum
}
```

`for...of` — не только про массивы. Полная картина связана с объектами, итераторами и т.д. (позже). Если упрощённо: некоторые значения ведут себя как коллекции — строка перебирается посимвольно:

```
const greeting = 'Hello'
// В этот момент со строкой происходит магия, которая разбирается в курсе ООП
for (const symbol of greeting) {
  console.log(symbol)
}
// => "H"
// => "e"
// => "l"
// => "l"
// => "o"
```

Строку с массивом не путайте: индексный доступ похож, но это не массив.

## Применимость

В большинстве задач с перебором предпочтительнее `for...of`. Иногда нужен ручной контроль — тогда снова `for`. Например, шаг через один элемент:

```
for (let i = 0; i < items.length; i += 2) {
  // какой-то код
}
```

Или обход с конца — `for...of` не подойдёт:

```
for (let i = items.length - 1; i >= 0; i -= 1) {
  // какой-то код
}
```

Бывает, что перебирают не массив, а диапазон чисел — тогда массива нет:

```
for (let i = 5; i < 10; i += 1) {
  // какой-то код
}
```

Или при обходе нужно менять исходный массив по индексу:

```
for (let i = 0; i < items.length; i += 1) {
  items[i] = /* что-то делаем */
}
```

*Если заглядывать в будущее и в то, как пишется реальный код на JavaScript, то там появляются функции высшего порядка. То есть на практике циклы, можно сказать, не нужны за редким исключением. Однако, невозможно перепрыгнуть работу с циклами, так как это база. А функции высшего порядка требуют понимания таких тем, которые за один присест не изучаются.*
