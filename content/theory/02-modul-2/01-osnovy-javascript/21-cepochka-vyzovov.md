# Цепочка вызовов

<!-- IMG (из Buildin, перезалить отдельно) -->
У чисел есть метод, переводящий значение в строку:

```
const peopleCount = 5
peopleCount.toString() // 5
```

Попробуйте сами: сработает ли код ниже и что именно выведется в консоль?

```
const name = 'Tirion'
console.log(name.length.toString())
```

Цепочка из нескольких точек подряд — новый синтаксис, но каждая операция в примере уже знакома. Здесь просто склеены известные приёмы языка. Так часто бывает в коде: можно пробовать комбинировать знакомые части, даже не зная точного правила — и нередко всё работает.

Проще всего разобрать цепочку, разложив её на шаги:

```
const name = 'Tirion'
const len = name.length
console.log(len.toString())
```

Оба варианта дают один и тот же результат. Операции можно выполнять по шагам через промежуточные константы или записать одной цепочкой свойств и методов. В цепочке вычисления идут слева направо.

Еще один пример для закрепления:

```
const name = 'Tirion'
console.log(name.toUpperCase().toLowerCase())
```

Такой код нужно читать внимательно: `.toLowerCase()` вызывается у результата метода слева, а `toUpperCase()` возвращает строку. Частая ошибка — забыть скобки вызова:

```
const name = 'Tirion'
// Этот код отработает неверно!
console.log(name.toUpperCase.toLowerCase())
```

По той же логике можно строить очень длинные (здесь бессмысленные) цепочки:

```
// Чему равен результат такого вызова?
console.log(name.toUpperCase().toLowerCase().length.toString().length)
```

С функциями так не получится: их обычно вкладывают f(f(f())), и читать это труднее. В других языках композицию оформляют через *композицию функций* или *пайплайн-оператор*; в JavaScript пайплайн тоже обсуждают: [https://github.com/tc39/proposal-pipeline-operator](https://github.com/tc39/proposal-pipeline-operator)
