# Арифметические операции

В «железе» всё сводится к числам; в прикладном коде тоже много арифметики. Для начала хватит школьной математики.

<!-- IMG (из Buildin, перезалить отдельно) -->
Сложение записывается так же, как в математике — `3 + 4`. Пример:

```javascript
// Не забываем точку с запятой в конце,
// так как каждая строчка в коде – инструкция
3 + 4
```

Выражение `3 + 4;` вычислится, но без `console.log` вы ничего не увидите — результат никуда не попадёт. Нужно **использовать** сумму, например вывести:

```javascript
// Сначала вычисляется сумма,
// затем она передается в функцию печати
console.log(3 + 4) // => '7'
```

Между оператором и числами ставьте пробелы — так читается легче: `3 + 4`, а не `3+4`.

Кроме сложения, доступны следующие операции:

- `*` — умножение

- `/` — деление

- `-` — вычитание

- `%` — остаток от деления

- `**` — возведение в степень

Теперь далее выведем на экран результат деления, а потом результат возведения в степень:

```javascript
console.log(8 / 2) // => 4
console.log(3 ** 2) // => 9
```

Первая инструкция выведет на экран `4` (потому что 8 / 2 это 4), а вторая инструкция выведет на экран `9` (потому что 32 это 9).

## Операторы

Терминология: `+` — **оператор**, числа вокруг — **операнды**. Операторы чаще всего — символы, как в математике.

```javascript
console.log(8 + 2)
```

Здесь `+` — оператор, `8` и `2` — операнды.

В случае сложения у нас есть два операнда: один слева, другой справа от знака `+`. Операции, которые требуют наличия двух операндов, называются **бинарными**. Если пропустить хотя бы один операнд, например, `3 + ;`, то программа завершится с синтаксической ошибкой.

Бывают унарные и тернарные операции; один и тот же символ может означать разное в разном контексте.

```javascript
console.log(-3) // => -3
```

Унарный минус перед `3` даёт `-3`.

Путаница возможна: `-3` и литерал, и унарная операция — так устроены языки.

## Коммутативная операция

Из школы: сумма не меняется от перестановки слагаемых — **коммутативность**.

Операция коммутативна, если `a op b` равно `b op a`. Сложение — да: *3 + 2 = 2 + 3*.

Вычитание не коммутативно: *2 - 3 ≠ 3 - 2*. В коде — так же.

В реальном коде порядок операндов чаще всего важен — следите за ним.

## Композиция операций

Составное выражение `3 * 5 - 2` пишется как в математике:

```javascript
console.log(3 * 5 - 2) // => 13
```

Сначала `*` и `/`, потом `+` и `-` — как в математике. Порядок можно менять скобками (ниже). Ещё пример:

```javascript
console.log(2 * 4 * 5 * 10)
```

Операции можно цепочкой комбинировать. Разберём по шагам `2 * 4 * 5 * 10`:

1. Сначала вычисляется `2 * 4` и получается выражение `8 * 5 * 10`.

1. Затем `8 * 5`. В итоге имеем `40 * 10`.

1. В конце концов происходит последнее умножение, и получается результат `400`.

Интерпретатор сворачивает выражение слева направо с учётом приоритета: `*`/`/` раньше `+`/`-`.

## Приоритет операций

Посчитайте в уме: `2 + 2 * 2`.

Ответ `6`, не `8`: сначала умножение. **Приоритет** задаёт порядок; `**` выше `*`/`/`, те выше `+`/`-`. Например, `2 ** 3 * 2` → `16`.

Нужен другой порядок — используйте скобки: `(2 + 2) * 2`.

Скобки вкладываются без ограничений. Примеры:

```javascript
console.log(3 ** (4 - 2)) // => 9
console.log(7 * 3 + (4 / 2) - (8 + (2 - 1))) // => 14
```

Для читаемости скобки можно добавить без смены смысла:

Было:

```javascript
console.log(8 / 2 + 5 - -3 / 2) // => 10.5
```

Стало:

```javascript
console.log(((8 / 2) + 5) - (-3 / 2)) // => 10.5
```

Код в первую очередь для людей; машине важна только корректность.
