# Определение функций

<!-- IMG (из Buildin, перезалить отдельно) -->
Свои функции упрощают написание и сопровождение кода: сложную последовательность шагов можно спрятать за одним вызовом. Отправка письма на сайте, к примеру, тянет за собой сеть, API почты и формат сообщения — снаружи это выглядит как простой вызов:

```
// Гипотетический пример
// Место откуда берется функция
import { send } from 'mailer'

const email = 'support@mail.io'
const title = 'Помогите'
const body = 'Я написал историю успеха, как я могу получить скидку?'

// Один маленький вызов — и много логики внутри
send(email, title, body)
```

Внутри такого вызова — подключение к серверу, сборка запроса из заголовка и текста и отправка с корректным закрытием соединения.

Создадим первую функцию — она выведет приветствие *Hello, World!*:

```
// Определение функции
// Определение не вызывает и не выполняет функцию
// Мы лишь говорим, что теперь такая функция существует
const showGreeting = () => {
  // Внутри тела отступ 2 пробела для удобства чтения
  const text = 'Hello, World!'
  console.log(text)
}

// Вызов функции
showGreeting() // => Hello, World!
```

Данные описывают «что», функции — «что сделать». Имена функций обычно глаголы: «отправить», «нарисовать», «открыть».

Код между `{` и `}` — **тело функции**. Туда можно писать любые инструкции, как в мини-программе. Тело выполняется при каждом вызове, и вызовы не мешают друг другу. Тело может быть пустым:

```
// Минимальное определение функции
const noop = () => {
  // Тут мог бы быть код, но его нет
}

noop()
```

Определение функции по форме похоже на объявление константы. На деле здесь две части: описание функции и присвоение её имени:

1. Определение: `() => { }`

1. Присваивание `const nameOfFunction = ...`

Можно описать функцию без имени — но тогда к ней не обратиться:

```
() => {
  // Работающий, но бесполезный код
}
```

«Создать функцию» на практике называют и «реализовать», и «определить», и «implement» — смысл один.
