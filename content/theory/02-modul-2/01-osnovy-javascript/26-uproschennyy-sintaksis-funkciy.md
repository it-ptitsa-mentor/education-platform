# Упрощенный синтаксис функций

В JavaScript полная запись функции выглядит многословнее, чем во многих функциональных языках:

```
const square = (x) => {
  return x ** 2
}
```

Много скобок и явный `return`. С ES6 есть **краткая форма** — без `{`/`}` и без `return`, когда тело — одно выражение:

```
// Требуется немного времени на привыкание к этой форме,
// но потом вы не сможете без нее жить
const double = x => x ** 2
```

От полной формы отличается двумя вещами: нет `{}` и нет `return` — результат единственного выражения возвращается сам.

Семантика та же, меняется только синтаксис. Два аргумента:

Полная версия:

```
const sum = (a, b) => {
  return a + b
}
```

Сокращенная версия:

```
const sum = (a, b) => a + b
```

Без `{}` — краткая форма. Ошибка: `const sum = (a, b) => { a + b }` — со скобками нужен явный `return`, иначе функция вернёт `undefined`.
