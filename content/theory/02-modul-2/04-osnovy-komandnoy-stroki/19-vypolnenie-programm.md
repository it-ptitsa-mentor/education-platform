# Выполнение программ

Запуск программы в командной строке включает цепочку поиска и исполнения. Понимание этого механизма помогает разбираться с типичными проблемами после установки ПО.

Программа — **набор файлов на диске**, один из них должен быть **исполняемым**. Тот самый `x` из урока про права: без него запуск завершится ошибкой.

В *nix всё держится на **соглашениях**. Когда вы вводите, например, `ls`, оболочка ищет исполняемый файл с таким именем в каталогах из переменной `PATH`:

```
echo $PATH

/home/hex/.local/bin:/home/hex/bin/:/home/hex/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

[Terminal session recording](https://asciinema.org/a/01M4S4F4DnfFA7GUIscaygEfl/iframe?cols=120&rows=10&preload=1)

Каталоги разделены двоеточием. В списке есть и общесистемные пути, и пользовательские. У вас `PATH` будет свой.

Оболочка идёт слева направо и ищет исполняемый `ls`. Если не находит — ошибка:

```
wrongname

bash: wrongname: command not found
```

[Terminal session recording](https://asciinema.org/a/mQnMTPx8omF0aByyb1f9HXaDM/iframe?cols=120&rows=10&preload=1)

Порядок важен: при одинаковых именах в разных каталогах сработает тот, что левее в `PATH`.

Где лежит исполняемый файл программы, покажут `type`, `which` и `whereis`:

```
type cp

cp is /bin/cp

whereis cp

cp: /bin/cp /usr/share/man/man1/cp.1

which cp

/bin/cp

ls -al /bin | grep cp

-rwxr-xr-x 1 root root  141528 Jan 18  2018 cp
```

[Terminal session recording](https://asciinema.org/a/ixtEcoBaPwduH58Icdj9JWXjb/iframe?cols=120&rows=14&preload=1)

Файл `cp` принадлежит `root`, но исполнять его могут все.

Пути к программам часто заканчиваются на *bin*. Такие исполняемые файлы называют **бинарниками**. Пакетные менеджеры при установке кладут их в каталог из `PATH`; для новых программ это обычно */usr/local/bin*.

Иногда нужно расширить `PATH` — часто после установки через менеджеры языков. Делайте это осторожно и только если без этого никак.

Если программа не в каталогах `PATH` и туда попадать не должна, а исполняемый файл лежит в текущей папке, одного имени недостаточно — так сделано из соображений безопасности: злоумышленник мог подложить одноимённый вредоносный файл. Запуск по полному пути обязателен; для текущего каталога — через `./`:

```
pwd

/home/hex

cd test/
ls -al

total 16
drwxr-xr-x 2 hex hex 4096 Sep 10 15:24 .
drwxr-xr-x 1 hex hex 4096 Sep 10 15:24 ..
-rwxr-xr-x 1 hex hex   60 Sep 10 15:24 executable

executable

bash: executable: command not found

./executable

Hello from executable!
```

[Terminal session recording](https://asciinema.org/a/DC4LS6Spp5YFLjg0mhf9MGJnm/iframe?cols=120&rows=15&preload=1)

По шагам:

1. Из домашней директории переходим в директорию *test*.

1. Находим в директории *test* исполняемый файл *executable*, который при запуске выводит сообщение `Hello from executable!`.

1. Пытаемся запустить по имени файл *executable* и получаем ошибку.

1. Пробуем запустить через точку и получаем успешный запуск.

## Изменение переменной `PATH`

В общем случае мы не рекомендуем менять переменную `PATH` самостоятельно. При правильной установке большинство программ самостоятельно добавляют свои исполняемые файлы в нужные места. В некоторых случаях они просят об этом пользователя — дают подсказки, какие файлы и куда нужно добавить после установки.

Чтобы вручную изменить переменную `PATH`, нужно поправить один из конфигурационных файлов Bash. Эти файлы лежат в домашней директории пользователя и могут называться так:

- *.bashrc*.

- *.bash_profile*.

- *.profile*.

В зависимости от настроек терминала, Bash загружает либо одни файлы, либо другие. Если в вашей домашней директории есть файл *.bashrc*, то попробуйте использовать его. Если нет, пробуйте остальные файлы в том порядке, в котором они приведены выше. Добавьте в этот файл следующую строку:

```
export PATH=$PATH:/path/to/directory
```

В этой строке вам нужно заменить */path/to/directory* на путь до директории с исполняемыми файлами. После этих манипуляций не забудьте перезапустить терминал.

Если вы используете оболочку Zsh (например, на MacOS), то вместо *.bashrc* используется файл *.zshrc*. Чтобы изменить значение переменной `PATH` в Zsh, откройте файл *~/.zshrc* и добавьте ту же самую строку
