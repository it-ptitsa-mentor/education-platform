# Переменные окружения

`cd` без аргументов переводит вас в домашний каталог. Откуда команда знает путь?

Не из прямого запроса к ОС и не из чтения конфигов при каждом вызове — она смотрит в **переменную окружения**. Об этом урок.

## Что такое переменные окружения

По смыслу они близки к переменным в коде: живут в рамках сессии оболочки и исчезнут после закрытия терминала. Значения подгружаются при старте сессии (и могут появляться и иначе).

Список — команда `env` (*environment*). Ниже фрагмент с одного сервера:

```
env

TERM=xterm-256color
SHELL=/bin/bash
LC_ALL=en_US.UTF-8
USER=kirill.m
YANDEX_VERSION=v2711
PATH=/home/kirill.m/bin:/home/kirill.m/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin
PWD=/home/kirill.m
LANG=en_US.UTF-8
SHLVL=1
HOME=/home/kirill.m
LOGNAME=kirill.m
```

Формат простой: слева от `=` имя, справа значение. У вас набор и значения будут другими: `YANDEX``_VERSION`, скорее всего, не будет, а `HOME` — будет, с вашим путём.

Зачем они нужны: настройка системы и программ. Удобны тем, что программе не важно, **как** значение попало в окружение — важно, что оно есть.

`cd` знает только: домашний путь лежит в `HOME`. Откуда он там взялся — не её дело.

Без переменных окружения пришлось бы либо каждый раз передавать путь явно (`cd --home-dir /home/kirill.m`), либо договориться о общем файле настроек, который `cd` читает при запуске.

Bash при старте выставляет базовый набор переменных для утилит и системы. Одна из них — `HOME`:

```
# Для вывода переменной используется echo,
# а перед самой переменной ставится знак $
echo $HOME

/home/kirill.m

# Вокруг знака `=` не должно быть пробелов
HOME=/tmp
echo $HOME

/tmp

# Возвращаем обратно
HOME=/home/kirill.m
echo $HOME

/home/kirill.m
```

Эксперимент: зададим `HOME` и вызовем `cd` без аргументов — она должна увести в «дом»:

```
# Достаточно выполнить присваивание слева от запускаемой команды
HOME=/tmp cd

/tmp

pwd

/tmp
```

`cd` сходила по пути из `HOME`. Повторим `cd` **без** присваивания слева:

```
cd
pwd

/home/kirill.m

echo $HOME

/home/kirill.m
```

[Terminal session recording](https://asciinema.org/a/UEKyxEZcTUQmHAVJIbH6yhL95/iframe?cols=120&rows=14&preload=1)

Вернулись в настоящий домашний каталог. Запись `HOME=/tmp cd` меняла переменную только для **одного** запуска `cd`. Есть **локальная** и **глобальная** установка.

`HOME=/tmp` перед командой не трогает сессию — только дочерний процесс. Так можно с любой командой.

Глобально для всей сессии — через `export`:

```
echo $HOME

/home/kirill.m

export HOME=/tmp
cd
pwd

/tmp

echo $HOME

/tmp
```

[Terminal session recording](https://asciinema.org/a/sK3iiDyoyPPfBSo2AwSxAtyCm/iframe?cols=120&rows=13&preload=1)

Изменение действует в **текущей** вкладке терминала; соседняя вкладка не затронута.

Особенно важна переменная `PATH` — из‑за неё часто ломаются сценарии; ей посвящён отдельный урок дальше по курсу.

### Самостоятельная работа

Изучите переменные окружения в вашей локальной среде, набрав команду `env`.
