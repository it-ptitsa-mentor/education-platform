# Редактирование файлов

В терминале тексты правят в основном через Nano, Vim или Emacs. Здесь разберём два первых.

Чаще всего в системе есть Vim — базовые навыки с ним пригодятся почти всегда. Редакторы стартуют внутри терминала; у Vim и Emacs есть и отдельное оконное приложение.

## Nano

Nano — простой редактор в духе Блокнота в Windows, но с подсветкой синтаксиса и чуть большим набором возможностей:

```
nano .profile
```

<!-- IMG (из Buildin, перезалить отдельно) -->
Внизу появляется панель с подсказками по горячим клавишам. Символ `^` означает `Ctrl`.

Например, `Ctrl + x` завершает работу. При выходе спросят, сохранять ли изменения:

- Сохранить — `y` (*yes*)

- Не сохранять — `n` (*no*)

Про Nano больше сказать мало: для правки одного файла его хватает, но для большого проекта с десятками файлов он неудобен.

## Vim

Другой уровень — Vim (*Vi Improved*), свободный редактор на базе классического vi (*visual editor*).

У него непривычная модель работы:

```
vim .profile
```

<!-- IMG (из Buildin, перезалить отдельно) -->
Главное отличие — **режимы**. В обычном редакторе файл открыли и сразу печатаете. Vim стартует в **COMMAND** (командный режим): нажатие клавиши — это команда, а не символ.

Без знания команд легко «сломать» экран. Текст набирают в режиме **INSERT**: войти — клавиша `i`, выйти — `Esc` или `Ctrl + [`.

Четыре основных режима:

- Командный — по умолчанию: навигация, правки комбинациями клавиш, поиск и удаление

- Вставки — обычный ввод с клавиатуры

- Визуальный — выделение фрагментов текста

- Командной строки — внутренняя строка команд редактора (`:`)

Vim славится высоким порогом входа — над этим даже шутят:

<!-- IMG (из Buildin, перезалить отдельно) -->
В шутке есть доля правды. С опытом всё чаще встречаются разработчики, которые целиком работают в Vim. Сильно помогают **расширения**.

«Голый» Vim выглядит скромно, но с плагинами превращается в мощную среду:

<!-- IMG (из Buildin, перезалить отдельно) -->
Почему так и что в Vim особенного — в [официальной документации Vim](https://www.vim.org/docs.php).

Говорят, из Vim сложнее всего выйти — запомним заранее:

- Убедитесь, что вы в командном режиме: `Esc` или `Ctrl + [`

- Наберите `:` — режим командной строки

- Введите `q!` и `Enter`

Так выход без сохранения. `:wq` — сохранить и выйти, `:w` — только сохранить.
