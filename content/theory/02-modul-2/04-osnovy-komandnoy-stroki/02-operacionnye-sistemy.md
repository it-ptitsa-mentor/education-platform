# Операционные системы

В этом уроке разберём историю появления командной строки.

## Как появилась командная строка

Всё началось в 1970-х. Тогда не было ни привычной Windows, ни персональных компьютеров. **Мейнфреймы** — огромные машины на целый этаж — и были «компьютерами»:

<!-- IMG (из Buildin, перезалить отдельно) -->
Примерно в те же годы появилась **Unix** — ОС на языке Си для мейнфреймов. Главная её черта — многопользовательский режим: один мейнфрейм обслуживал много пользователей одновременно, потому что компьютеры были редкостью.

К мейнфрейму подключались через **терминалы** — устройства с двумя функциями:

- Ввод (клавиатура)

- Вывод (монитор)

Обработка шла на сервере. Сам терминал выглядел так:

<!-- IMG (из Buildin, перезалить отдельно) -->
Unix оказалась удачной, но была платной и **закрытой** — исходный код был недоступен.

Со временем на базе Unix появлялись новые системы без обратной совместимости: под каждую ОС приходилось писать отдельную версию программы.

Это усложняло жизнь всем, поэтому приняли стандарт **POSIX** (*Portable Operating System Interface for UNIX*). Сейчас это семейство стандартов, описывающих ключевые компоненты ОС и способы взаимодействия с ними из программ на Си.

## Операционная система Linux

Параллельно развивались сети: сначала прообразы интернета, затем сам интернет. Компьютеры уменьшались, серверов становилось больше. В 1991 году студент Линус Торвальдс начал писать ОС, совместимую с POSIX.

Через интернет Линус собрал сообщество разработчиков, которое быстро создало рабочую **Linux** и выложило исходники в открытый доступ. С тех пор Linux на серверах набирал популярность.

Сам Linux — только ядро; без сопутствующих программ им не обойтись.

К тому моменту уже существовал проект **GNU** Ричарда Столлмана. В нём были свободные программы — базовый инструментарий разработчика на Си:

- Текстовый редактор **Emacs**

- Компилятор **gcc**

- Командная оболочка **bash**

- Библиотека **libc**

Эти программы писались под Unix-подобные системы и использовали POSIX-вызовы. Поэтому Linux сразу поставлялся с утилитами — архиваторами, bash и др., с которой мы начнём работу в курсе.

Для тех, кто знает только Windows, сюрприз: у Linux нет «родной» графической оболочки. Поэтому Linux может выглядеть по-разному — на него ставят разные GUI:

Сейчас Linux в чистом виде почти не используют. Пользователи работают с **дистрибутивами**, в которые входят:

- Ядро Linux

- GNU-программы

- Утилиты установки и настройки

- Регулярные обновления ядра и окружения

Дистрибутивов много; среди популярных — бесплатная Ubuntu:

Также известны варианты Ubuntu:

- Ubuntu Server — для серверов

- Xubuntu — для слабых машин

В отличие от Unix-подобных систем, Windows изначально рассчитывала на одного пользователя и не стремилась к POSIX. Она заняла рынок ПК, но на серверах почти не встречается, хотя есть Windows Server.

Windows редко ставят на серверы: она платная и несовместима с POSIX. Есть `cmd.exe`, но её команды и возможности отличаются от Linux; опыт с ней слабо помогает в Linux-окружении, с которым всё равно придётся работать.

Рекомендуем осваивать Linux и командную строку через полное погружение: установить Ubuntu основной системой или [активировать](https://learn.microsoft.com/ru-ru/windows/wsl/install) её внутри Windows.

На macOS ничего дополнительно ставить не нужно — система на базе Unix и даже более POSIX-совместима, чем Linux. Там отличная командная строка и инструментарий.

Далее в курсе примеры командной строки — только для POSIX-совместимых ОС.

### Самостоятельная работа

Если у вас Windows, настройте окружение [по инструкции Microsoft по WSL](https://learn.microsoft.com/ru-ru/windows/wsl/install).

При создании пароля Ubuntu в терминале символы не отображаются — даже `*****`. Это нормально: так устроена безопасность Linux-терминала.
