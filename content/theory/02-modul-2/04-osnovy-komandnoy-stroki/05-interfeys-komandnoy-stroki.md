# Интерфейс командной строки

Представьте новую программу с меню и кнопками — вы нажимаете и получаете результат. Это **графический интерфейс** (*GUI*, *Graphical User Interface*).

GUI — не единственный вариант. Здесь разберём **интерфейс командной строки** (*CLI*, *Command Line Interface*). Он кажется непривычным: видно только имя программы и ввод текста.

## Аргументы и опции

Чем чаще вы в терминале, тем больше утилит встретите. Многие станут ежедневными — например, `ls` для списка файлов и каталогов.

Базовый вызов — имя программы и `Enter`:

```
ls

Desktop Documents Downloads Library Movies Music Pictures Public
```

Скрытые файлы в *nix начинаются с точки: `.profile`. Их покажет `ls -a`:

```
ls -a

.  .CFUserTextEncoding Desktop   Downloads Movies Pictures
.. .localized          Documents Library   Music  Public
```

Содержимое каталога Public — `ls` с аргументом:

```
ls Public

Drop Box
```

Сложные утилиты настраиваются длинными строками:

```
ffmpeg -i input.mp4 -vcodec libx264 -crf 30 output.mp4
```

Детали таких команд сейчас не важны — важны общие закономерности CLI.

Хорошая новость: закономерности есть. Плохая: не все программы им следуют строго.

Почти любую команду дополняют двумя способами:

Способ 1 — **аргументы**. Пример: `ls Music`, где `Music` — аргумент

Способ 2 — **опции** (флаги). Пример: `ls -a` с опцией `-a`

## Опции

Опции начинаются с одного или двух дефисов. Частая опция списка файлов — `-l` (подробный вывод):

```
ls -l

total 0
drwx------+  3 Guest  _guest   96 Nov 21  2017 Desktop
drwx------+  3 Guest  _guest   96 Nov 21  2017 Documents
drwx------+  3 Guest  _guest   96 Nov 21  2017 Downloads
drwx------+ 26 Guest  _guest  832 Nov 21  2017 Library
drwx------+  3 Guest  _guest   96 Nov 21  2017 Movies
drwx------+  3 Guest  _guest   96 Nov 21  2017 Music
drwx------+  3 Guest  _guest   96 Nov 21  2017 Pictures
drwxr-xr-x+  4 Guest  _guest  128 Nov 21  2017 Public
```

Опции можно комбинировать. Все файлы, включая скрытые, с подробностями — `ls -a -l` или короче:

- `ls -al`

- `ls -la`

Не забывайте дефис перед опцией. Без него `ls la` воспримет `la` как аргумент (имя каталога).

Опции и аргументы часто сочетаются — зависит от программы. У `ls` можно: `ls -la Music` — подробный список каталога `Music`:

```
ls -la Music

total 0
drwx------+  4 Guest  _guest   128 Nov 21  2017 .
drwxr-xr-x+ 89 Guest  _guest  2848 Aug 24 14:06 ..
-rw-r--r--   1 Guest  _guest     0 Nov 21  2017 .localized
drwxr-xr-x   9 Guest  _guest   288 Aug 26 17:25 iTunes
```

Обычно опции слева от аргументов; в сложных утилитах бывает иначе — их пока не разбираем.

Запись вроде `-tupa` неоднозначна:

- одна опция `tupa`

- четыре опции `t`, `u`, `p`, `a` в цепочке

Смотрите документацию: `man` (*manual*) — `man <имя команды>` открывает справку.

## Варианты опций

У многих утилит одна опция в короткой и длинной форме. В PHP — `-v` и `--version`:

```
php -v

PHP 7.2.7 (cli) (built: Jun 22 2018 06:27:50) ( NTS )
Copyright (c) 1997-2018 The PHP Group
Zend Engine v3.2.0, Copyright (c) 1998-2018 Zend Technologies

php --version

PHP 7.2.7 (cli) (built: Jun 22 2018 06:27:50) ( NTS )
Copyright (c) 1997-2018 The PHP Group
Zend Engine v3.2.0, Copyright (c) 1998-2018 Zend Technologies
```

[Terminal session recording](https://asciinema.org/a/pMP3RNIkrIyBMmVxe6rqUWueB/iframe?cols=160&rows=20&preload=1)

Короткие и длинные опции уместны в разных ситуациях:

- В интерактивном терминале важна скорость — удобны короткие

- В скриптах важна читаемость — лучше длинные, сразу видно смысл

Обычно длинные опции с `--`, но некоторые программы используют один дефис.

Рассмотренные опции без параметров. Бывают опции со **значением**.

В macOS утилита `say` произносит переданный текст. С опцией `-o` можно записать звук в файл:

```
# Вместо -o можно написать --output-file
say -o hi.aac 'Hello, World.'
```

Путь к файлу — **значение опции**, чаще через пробел после флага.

Спецсимволы и пробелы в значении оборачивают в кавычки:

```
say -o 'hi.aac' 'Hello, World.'
```

Иногда допустим `=` вместо пробела:

```
# Команда say такое не позволяет, но зато видно принцип
say -o=hi.aac 'Hello, World.'
```

`say` может читать входной файл: тогда текст-аргумент игнорируется:

```
# Здесь мы указали входной файл, который нужно прочитать — hello_world.txt
# Еще здесь указано, каким голосом надо читать и в какой файл записывать прочитанное
say -v Alex -o hi -f hello_world.txt
```

В разделе **SYNOPSIS** документации `say` — все варианты вызова:

```
say [-v voice] [-r rate] [-o outfile [audio format options] | -n name:port | -a device] [-f file | string ...]
```

Похожая нотация у большинства утилит:

- `[]` — необязательный фрагмент; `-v` и остальные опции в примере не обязательны

- `|` — «или»: в `[-f file | string ...]` либо файл, либо строка аргументом, не оба сразу

Есть и другие обозначения: значения по умолчанию, выбор из списка, отрицание.

Разобрали базу; заучивать не обязательно — опции приходят с практикой. Общие принципы и man/explainshell — дальше эксперименты.
