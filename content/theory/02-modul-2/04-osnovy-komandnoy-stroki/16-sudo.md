# Sudo

В командной строке часто нужно повысить привилегии и выполнить команду от имени суперпользователя `root`. С частью таких случаев вы уже сталкивались, с другими — впереди:

- Установка новых программ

- Навигация по чужим директориям

- Изменение прав доступа

- Изменение содержимого в файлах, не принадлежащих текущему пользователю

- Создание, редактирование и удаление файлов, когда не хватает прав текущего пользователя

- Запуск программ, требующих повышенных привилегий

В этом уроке — как временно стать другим пользователем. После входа в систему можно вызвать `su` (*substitute user* / *switch user*). Раньше это был основной способ, сейчас он устарел и не рекомендуется. Подробнее — в статье «[Su или sudo?](https://habr.com/ru/articles/44783/)» из дополнительных материалов.

В современных системах привилегии обычно повышают через `sudo` (*substitute user and do* — «подменить пользователя и выполнить»).

`sudo` ставят слева от любой команды. По умолчанию она пытается выполнить её от `root`:

```
# Нет прав на выполнение
touch /etc/myfile

touch: cannot touch '/etc/myfile': Permission denied

# С `sudo` все работает
sudo touch /etc/myfile

# Видно, что владелец файла — это суперпользователь `root`
stat /etc/myfile

  File: '/etc/myfile'
  Size: 0           Blocks: 0          IO Block: 4096   regular empty file
Device: ca01h/51713d    Inode: 2761        Links: 1
Access: (0644/-rw-r--r--)  Uid: (    0/    root)   Gid: (    0/    root)

# Нет прав на удаление
rm /etc/myfile

rm: remove write-protected regular empty file '/etc/myfile'? y
rm: cannot remove '/etc/myfile': Permission denied

# Опять помогла утилита `sudo`
sudo rm /etc/myfile
```

В зависимости от настроек `sudo` запросит пароль или откажет, если прав на неё нет. В Ubuntu обычно спрашивают пароль и запоминают его на пять минут — в этот период `sudo` можно вызывать без повторного ввода.

Чтобы выполнить команду не от `root`, а от другого пользователя, добавьте флаг `-u`:

```
sudo -u nobody mkdir /tmp/test
# Директория создана от имени пользователя nobody
stat /tmp/test

  File: '/tmp/test'
  Size: 4096        Blocks: 8          IO Block: 4096   directory
Device: ca01h/51713d    Inode: 4577        Links: 2
Access: (0755/drwxr-xr-x)  Uid: (65534/  nobody)   Gid: (65534/ nogroup)
```

Если нужно сделать серию действий от другого пользователя, запустите вложенную оболочку — новую сессию внутри текущей:

```
# Команда `sudo` запросит пароль текущего пользователя
sudo -i

# Проверяем, что сессия запущена от суперпользователя
id

uid=0(root) gid=0(root) groups=0(root)
```

После работы не забудьте вернуться: наберите `exit`.

## Подводные камни

Знание `sudo` часто обманывает новичков: при странной ошибке они сразу добавляют `sudo` и запускают команду от суперпользователя. Иногда это «срабатывает», но порождает новые проблемы.

Если под `sudo` создать файлы и каталоги, владельцем станет `root`. Любая работа с ними без `sudo` даст «Permission denied» — даже если программа обращается к ним косвенно, читая конфиги и служебные файлы.

Правильное решение зависит от ситуации: где-то нужен именно `sudo`, где-то — смена прав (следующий урок), иногда — переустановка части системы.

Ориентир может быть таким:

- Всё в домашних каталогах пользователя должно принадлежать ему

- Всё вне домашнего каталога, что требует особых прав, чаще запускают с `sudo`, но бывают исключения

На практике:

```
ls -la

# Все содержимое домашней директории принадлежит одному пользователю
# Сама директория пользователя принадлежит ему же
# Родительская директория принадлежит суперпользователю
drwxr-xr-x+ 117 mokevnin  staff    3744 Feb 19 15:55 .
drwxr-xr-x    5 root      admin     160 Oct 12 19:15 ..
-r--------    1 mokevnin  staff       7 Nov 21  2017 .CFUserTextEncoding
-rw-r--r--@   1 mokevnin  staff   22532 Feb  8 00:04 .DS_Store
```
