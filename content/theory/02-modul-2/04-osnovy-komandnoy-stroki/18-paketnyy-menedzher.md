# Пакетный менеджер

Привычный для пользователя Windows путь — искать программы в интернете и ставить вручную. В *nix так делают реже: для установки ПО используют **пакетные менеджеры**, с которыми познакомимся здесь.

В одних ОС менеджер уже есть, в других его ставят отдельно. На macOS, например, популярен [brew](https://brew.sh/).

Дальше примеры на **apt** из Ubuntu. Установка PHP выглядит так:

```
# Требует sudo, потому что мы устанавливаем в системную директорию
sudo apt install php
# Здесь будет много текста и вопрос, устанавливать ли? Если да, то  PHP установится

php -v

PHP 7.4.3 (cli) (built: Feb 20 2020 08:51:50) ( NTS )
```

Менеджер состоит из нескольких частей. С пользователем напрямую общается утилита `apt`: `apt install` и имя пакета.

Чаще всего нужен `root`, потому что пакеты попадают в системные каталоги, куда обычный пользователь писать не может:

```
apt install php

E: Could not open lock file /var/lib/dpkg/lock - open (13: Permission denied)
E: Unable to lock the administration directory (/var/lib/dpkg/), are you root?
```

Но не всегда — на macOS `sudo` для установки через brew обычно не нужен.

Установленные пакеты удаляют командой `apt remove`:

```
apt remove php
# Тут тоже много вывода и вопрос, удалять ли?
```

С другой стороны — **каталог пакетов**. При установке `apt` ищет там пакет с указанным именем. Если не находит, сообщит об этом:

```
# Пакета wrongname не существует
sudo apt install wrongname

Reading package lists... Done
Building dependency tree
Reading state information... Done
E: Unable to locate package wrongname
```

Чтобы не тянуть каталог при каждой операции, apt хранит локальный **индекс** — список доступных пакетов на диске.

Иногда пакет в каталоге есть, а `apt` его «не видит». Тогда обновите индекс вручную:

```
sudo apt update
# Теперь можно устанавливать
sudo apt install php
```

Здесь обычно возникают три вопроса:

1. Как узнать, какой пакет ставить?
  Самый прямой путь — сайт программы и раздел «Скачать» (*Download*): там часто дают команды для пакетного менеджера. Так сделано, например, у [Node.js](https://nodejs.org/en/download/package-manager/).

  Можно поискать *php install ubuntu* — найдутся те же инструкции. Со временем имена пакетов запоминаются, и ставить можно без подсказок.

1. Чем пакет отличается от программы?
  Если коротко, пакет — обёртка вокруг программы. Сама программа о менеджере пакетов не знает. Чтобы ставить её через менеджер, разработчик заранее собирает пакет.

  В Ubuntu пакеты в формате *deb*; для него есть подробная документация по сборке.

1. Как программы попадают в каталог?
  Зависит от политики сопровождающих каталога. В строгие репозитории попадает с трудом — только проверенный софт, обновления медленные. В других всё проще и быстрее.

  В любом случае нужна процедура публикации. **От наполнения каталогов во многом отличаются дистрибутивы Linux**. Каталоги огромны — десятки и сотни тысяч программ и библиотек.

Пакетные менеджеры есть не только в ОС. Любая экосистема с распространением библиотек и программ обычно имеет свой: у языков программирования менеджеры часто встроены:

- В Node.js — пакетный менеджер npm

- в PHP — Composer

- в Ruby — RubyGems

- в Python — pip

### Самостоятельная работа

Утилита `tree` показывает дерево каталогов и файлов, но в стандартный набор не входит — её нужно установить.

Воспользуйтесь пакетным менеджером:

- Установите пакетный менеджер для вашей операционной системы, если он еще не установлен

- Используя пакетный менеджер, установите программу `tree`

- Запустите программу и убедитесь, что все работает корректно

Документацию по утилите можно посмотреть, выполнив в терминале команду `man tree`.
