# Файловая структура

Файловая система *nix сильно отличается от Windows — разберём её отдельно.

Базовые понятия:

- Структура — **дерево**

- В **узлах** — **директории** (каталоги), файлы особого типа

- В **листьях** — обычные **файлы**

## Директории

В *nix не говорят «папка» — используют **директорию** или **каталог**; по смыслу это то же самое.

В Windows несколько деревьев — по одному на диск. В *nix одно дерево с корнем `/`; диски и устройства монтируются как каталоги и файлы внутри него.

Метаданные файла или каталога — команда `stat` (*file system status*):

```
# Не обращайте внимание на непонятные для вас данные
# Об их значении поговорим позже
stat ~/.profile
```

В Windows и macOS имена часто **регистронезависимы**: `file.txt` и `File.txt` — один объект.

В *nix регистр важен: `index.html`, `Index.html`, `INDEX.HTML` — разные файлы. Следите за регистром в путях.

Говорят: в *nix «всё — файл». На низком уровне так и есть. **Директория** — специальный файл со списком имён. Устройство после подключения тоже файл или каталог.

Для разработчика это удобно: запись на принтер и вывод на экран с точки зрения кода похожи на запись в файл. На пользовательском уровне каталоги и файлы различаются и имеют свои команды.

В *nix есть стандарт **FHS** — согласованный набор каталогов с ролями, например:

- `/etc` — конфигурация программ в текстовых файлах (реестра как в Windows нет)

- `/home` — домашние каталоги пользователей; у суперпользователя `root` дом обычно в `/root`

Загляните в [статью FHS в Википедии](https://ru.wikipedia.org/wiki/FHS) — что означает каждый типичный каталог.

## Скрытые файлы

Не в каждый каталог можно зайти, не каждый файл прочитать или изменить, не каждую программу запустить. В *nix права завязаны на пользователей и группы — об этом позже. Пока достаточно знать, что ограничения есть; их видно в `ls -l`.

В *nix нет отдельного понятия «расширение»: точка — часть имени. Имена вроде `hello.mp3` привычны; целиком строка — имя файла. Бывает и `index.html.haml`.

Скрытые файлы в Windows и *nix устроены по-разному:

- Windows: скрытость — свойство файла в настройках

- *nix: имя, начинающееся с `.`, считается скрытым; достаточно переименовать

Все файлы, включая скрытые — `ls -a`:

```
ls -a

.  ..  .bash_logout  .bashrc  .config  .gitconfig  .profile
```

Две особые записи:

- `.` — текущий каталог

- `..` — родительский

Поэтому работает `cd ..`.

Кроме обычных файлов в *nix есть и другие типы:

- **Hard Link** — дополнительное имя того же файла

- **Symbolic link** — символическая ссылка, аналог ярлыка в Windows; при удалении цели ссылка «ломается»

- **Socket** — файл для обмена между процессами; с сокетами часто сталкиваются в разработке

Это главные типы на старте; есть и другие, но сейчас не углубляемся.
