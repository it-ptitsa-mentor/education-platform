# Права доступа

Разработчики нередко видят **ошибку доступа**:

```
touch /etc/myfile

touch: cannot touch '/etc/myfile': Permission denied
```

Она значит, что у текущего пользователя нет прав создавать файлы в `/etc`. Причина — права доступа, которые разберём в этом уроке.

Помимо владельца и группы, у каждого файла есть права:

- **r** — чтение (*read*)

- **w** — запись (*write*)

- **x** — исполнение (*execute*)

Права задаются для трёх категорий субъектов:

- Владельца (User)

- Пользователей той же группы (Group)

- Всех остальных (Other) — кто не попал в первые две

Пример:

```
# Пример строчки из вывода команды ls -la
-rw-r--r-- 1 kirill.m kirill.m 3771 Aug 31  2015 .profile
```

Слева — строка прав (*permissions*). Разобьём её на части:

- `-` — это файл (не каталог)

- `rw-` — права владельца: чтение (`r`) и запись (`w`), исполнение запрещено (прочерк)

- `r--` — права группы `kirill.m`: только чтение, запись и исполнение недоступны

- `r--` — права остальных: здесь совпадают с группой, файл только для чтения

Кто может удалить этот файл? Нужны владелец, группа и права **каталога**, где лежит `.profile`. Сам файл не задаёт право на своё удаление — смотрят на родительскую директорию.

Удаление возможно, если можно писать в эту директорию:

```
ls -la /home/ | grep kirill

drwxr-xr-x  5 kirill.m          kirill.m          4096 Aug 29 11:34 kirill.m
```

У домашнего каталога другие права:

- В начале `d` — это директория

- У владельца `rwx`, у остальных `r-x`: писать внутрь может только владелец

Что значит `x` для каталога? Право заходить в него и обращаться к вложенным файлам и папкам — при условии, что у самих объектов есть нужные `r`, `w` или `x`. Исполняемый на чтение файл в каталоге с `x` можно прочитать; без `x` доступ к нему пропадёт.

Чтение каталога — это просмотр списка имён в нём.

Если нужен не простой список, а вывод как у `ls -l` с метаданными (владелец, группа, дата, права), понадобится и `x` — оболочке придётся читать данные о каждом элементе. Без `r` содержимое каталога всё равно не увидеть.

В документации иногда встречается числовая запись вроде `755`. Это три цифры — по одной на владельца, группу и остальных, в том же порядке; каждая кодирует набор `rwx`. `0` — нет прав ни на что:

|  |
|  |
|  |
|  |
|  |
|  |
|  |
|  |

Примеры перевода:

- `drwxr-xr-x` соответствует `755`

- `-rw-r--r--` соответствует `644`

Пользователь `root` вне этой схемы: для него не важно, какие права выставлены — суперпользователь может всё.

Сменить владельца файла или каталога помогает `chown` (*change owner*). В простейшем виде указывают пользователя и путь:

```
chown kirill .profile
```

Права меняют через `chmod` (*change mode*): передают новые права и путь. Запись бывает буквенной или числовой; `+` и `-` добавляют или снимают права:

```
chmod +r file.txt
chmod 754 file.txt
```
