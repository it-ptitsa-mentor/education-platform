# Определения

Файловая структура — пример дерева, знакомого каждому пользователю компьютера. Она состоит из директорий и файлов разных типов.

```javascript
nodejs-package # корневая директория
├── Makefile # файл
├── README.md # файл
├── __tests__ # директория
│   └── half.test.js # файл
├── babel.config.js # файл
└── node_modules # директория
    └── @babel # директория
        └── cli # директория
            └── LICENSE # файл
```

Так её называют из‑за формы: все элементы файловой системы выстраиваются в иерархию. На верхнем уровне — корневая директория (или диск в Windows), ниже — файлы и каталоги, которые сами могут содержать файлы и каталоги.

Главная черта древовидной структуры — **рекурсивность**. Дерево состоит из поддеревьев, каждое из которых снова состоит из поддеревьев, и так далее. Отсюда и основные приёмы работы с деревьями в коде: почти все они так или иначе опираются на рекурсию.

Дерево состоит из **узлов** (вершин или нод — по‑английски **node**) и **рёбер** между ними. Рёбра в реальности не существуют: они нужны, чтобы визуализировать связь и при необходимости описать её. Узлы делятся на **внутренние** (у которых есть потомки) и **листовые** (без потомков). В файловой системе листья — это файлы, внутренние узлы — директории.

У каждой вершины есть **родитель** (предок). Исключение — **корневой узел**: у него нет родителя, с него начинается дерево. Число потомков у внутренней вершины в общем случае любое. В деревьях также говорят о **глубине** (depth) — сколько шагов по рёбрам от корня до текущей вершины. Вершины на одной глубине с общим родителем называют братскими или сестринскими.

## Реализация

Способов описать дерево в коде бесконечно много. Самый простой — вложенные массивы:

```javascript
[['index.html', 'main.js'], 'index.js', ['favicon.ico', 'app.css']];
//                    * корень – сам массив
//         /          |         \
//       *         index.js       *
//  /         |               |        \
// index.html main.js   favicon.ico app.css

// Ещё пара примеров деревьев с произвольными данными:
[]; // пустое дерево
[3, 2, [3, 8], [[8], 3]];
[1, null, [[3]], [5, 'string', [undefined, [3], { key: 'value' }]]]
```

Здесь корень — сам массив, а элементы — дети. Элемент, не являющийся массивом, считается листовым узлом; массив — внутренним. Внутренний узел, в свою очередь, состоит из детей.

Любое дерево можно разделить на две части:

1. Данные, которые в нём хранятся

1. Структуру связей между данными

```javascript
[['index.html', 'main.js'], 'index.js', ['favicon.ico', 'app.css']]
```

Что здесь данные, а что структура? Данные — листовые узлы (имена файлов). Внутренние массивы — только структура: они задают расположение данных, но сами данных не несут. Для полноценной файловой системы такой формат неудобен — например, нельзя задать имя директории.

Расширим представление: каждый элемент — массив, где первый элемент — значение в узле, второй (если есть) — массив детей. Нет второго элемента — узел листовой.

```javascript
['app', [ // Корень
  ['dist', [ // Внутренний узел
    ['index.html'], // Лист
    ['main.js'], // Лист
  ]],
  ['index.js'], // Лист
  ['assets', [ // Внутренний узел
    ['favicon.ico'], // Лист
    ['app.css'], // Лист
  ]],
]]

//                   app
//         /          |         \
//       dist      index.js   assets
//  /         |               |        \
// index.html main.js   favicon.ico app.css
```

Вариант многословнее, но данные можно хранить в любом узле, не только в листе — и не обязательно строкой. Объекты вместо строк позволяют добавить любые поля.

Самый гибкий и удобный вариант — объекты: каждый узел — объект, список детей — массив.

```javascript
// Обратите внимание на разделение структуры и данных
// Здесь оно значительно более очевидное
{
  value: 5,
  children: [
    { value: 10 },
    { value: 100 },
    { value: 'nested', children: [/* ... */] }
  ]
}
```

В каком‑то смысле и массив, и объект сами по себе уже деревья: любая рекурсивная структура, элементами которой может быть она же, трактуется как дерево. В массиве может лежать массив, в объекте — объект.
