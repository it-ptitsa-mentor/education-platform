# Виртуальная файловая система

В этом курсе мы соберём виртуальную (не настоящую) файловую систему и реализуем типичные операции: подсчёт свободного места, поиск файлов и каталогов и т.п. Практической пользы у такой ФС нет — на ней отрабатываются навыки работы с древовидными структурами. Обработка любого дерева по сути одна и та же: файловая система, каталоги товаров, адреса, родственные связи — всё это дерево, и подход к коду будет одинаковым. Освоив одно дерево, те же приёмы переносятся на другие.

Вот как создаётся дерево виртуальной файловой системы:

```javascript
import * as fsTrees from '@hexlet/immutable-fs-trees'

// mkdir вторым параметром принимает список детей,
// которые могут быть либо директориями, созданными mkdir,
// либо файлами, созданными mkfile
const tree = fsTrees.mkdir('etc', [
  fsTrees.mkfile('bashrc'),
  fsTrees.mkdir('consul', [
    fsTrees.mkfile('config.json'),
  ]),
])
```

Первым аргументом в `mkdir()` и `mkfile()` задаётся имя директории или файла. Вторым `mkdir()` принимает вложенные файлы и каталоги. Обе функции последним параметром принимают метаданные `meta` — о них чуть ниже.

В итоге получается такая структура:

```javascript
etc
├── bashrc
└── consul
    └── config.json
```

Вкладывая `mkdir` и `mkfile` друг в друга, можно описать любую иерархию. Корень — директория; в листьях — файлы или пустые каталоги.

Структура виртуальная: на диске ничего не создаётся. Вся информация живёт в переменной `tree`. Если её вывести в консоль, увидим внутреннее представление:

```javascript
{
  name: 'etc',
  type: 'directory',
  meta: {},
  children: [
    {
      name: 'bashrc',
      type: 'file',
      meta: {},
    },
    {
      name: 'consul',
      type: 'directory',
      meta: {},
      children: [
        {
          name: 'config.json',
          type: 'file',
          meta: {},
        }
      ],
    },
  ],
};
```

Так устроено файловое дерево внутри: два типа узлов — директории и файлы.

Представление директории:

```javascript
{
  name: /* ... */,
  type: 'directory',
  meta: {}, // Свойства директории
  children: [/* ... */], // Здесь хранятся дети
}
```

Представление файла:

```javascript
{
  name: /* ... */,
  type: 'file',
  meta: {}, // Свойства файла
}
```

У файлов и директорий общее поле `name`. По `type` узнаём тип узла при обходе. В `meta` лежат произвольные данные — размер, дата создания и т.д. Метаданные задаются при создании:

```javascript
fsTrees.mkfile('.bashrc', { size: 75 })
fsTrees.mkdir('hexlet', [/* дети */], { owner: 'nobody' })
```

Они нужны функциям, которые анализируют дерево — например, считают занятое место.
