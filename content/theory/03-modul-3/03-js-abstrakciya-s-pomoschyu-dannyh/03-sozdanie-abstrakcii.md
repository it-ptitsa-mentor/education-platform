# Создание абстракции

Декартова система координат — не единственный способ описать фигуру на плоскости. Есть и полярная система.

> Полярная система координат — двухмерная система, в которой точка задаётся полярным углом и полярным радиусом. Она удобна, когда связи между точками проще выразить через радиус и угол; в декартовой (прямоугольной) системе для того же нужны тригонометрические преобразования. (c) Wikipedia

Представьте графический редактор (условный аналог Photoshop). Библиотека примитивов построена на декартовых координатах. Потом выясняется, что полярная система упростит и ускорит код. Какова цена перехода? Придётся переписать почти всё.

```
const point = { x: 2, y: 3 }
const symmetricalPoint = { x: -point.x, y: point.y }
```

Библиотека не скрывает внутреннюю структуру. Любой код, работающий с точками и отрезками, знает, как они устроены — и при создании примитивов, и при чтении их частей. Спрятать реализацию можно с помощью функций:

```
const point = makePoint(3, 4)
const symmetricalPoint = makePoint(-getX(point), getY(point))
```

Здесь три функции: `makePoint()`, `getX()` и `getY()`. `makePoint()` — **конструктор**: создаёт примитив. `getX()` и `getY()` — **селекторы** (от select — «извлекать», «выбирать»). Небольшое изменение даёт большой эффект: в прикладном коде нет прямой работы со структурой.

```
// То есть мы работаем не так
const point = [1, 4] // мы знаем, что это массив
console.log(point[1]) // прямое обращение к массиву

// А так
const point = makePoint(3, 4) // мы не знаем как устроена точка
console.log(getY(point)) // мы получаем доступ к частям только через селекторы
```

По коду нельзя понять, из чего «внутри» состоит точка (для этого нужна отладочная печать). Так строится **абстракция данных**: внутренняя реализация скрыта от внешнего кода.

Один из способов реализации:

```
const makePoint = (x, y) => ({ x, y })

const getX = point => point.x
const getY = point => point.y
```

Теперь реализацию можно менять без полного переписывания клиентского кода (отдельные места всё же могут потребовать правок). Хотя `makePoint()` принимает декартовые `x` и `y`, внутри точка может храниться в полярных координатах — при конструировании выполняется преобразование:

```
const makePoint = (x, y) => {
  // конвертация
  return {
    angle: Math.atan2(y, x),
    radius: Math.sqrt(x ** 2 + y ** 2),
  }
}
```

Один раз перейдя на абстракцию данных, обратно к «голым» структурам не возвращаются: пользуйтесь своими функциями или API библиотеки.
