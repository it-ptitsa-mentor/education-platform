# Уровневое проектирование

Разберём ещё одну простую систему — рациональные числа и операции над ними. Рациональное число записывается дробью `a/b`, где `a` — числитель, `b` — знаменатель; `b` не должен быть нулём.

В JavaScript рациональных типов нет, поэтому абстракцию создаём сами. Нужны конструктор и селекторы:

```
const num = makeRational(1, 2) // создали рациональное число "одна вторая"
const numer = getNumer(num) // 1
const denom = getDenom(num) // 2
```

Три функции задают рациональное число: конструктор собирает значение, селекторы извлекают части. Чем именно в языке является `num` — неважно: функция, массив, объект. Внутри можно даже использовать строки:

```
const makeRational = (numer, denom) => `${numer}/${denom}`

const getNumer = rational => rational.split('/')[0]

const getDenom = rational => rational.split('/')[1]

console.log(makeRational(10, 3)) // => 10/3
```

Представить число — мало. Абстракция полезна, когда с ней можно работать: сложение, вычитание, умножение. Умножение дробей — перемножить числители и знаменатели:

```
3/4 * 4/5 = (3 * 4)/(4 * 5) = 12/20
```

При реализации, если внутри число — `{ numer: 2, denom: 3 }`, наивный вариант такой:

```
const mul = (rational1, rational2) => {
  return {
    numer: rational1['numer'] * rational2['numer'],
    denom: rational1['denom'] * rational2['denom'],
  }
}
```

Снаружи всё выглядит нормально: на входе и выходе — «рациональные числа». Внутри же абстракции нет: код знает устройство объекта. Любая смена представления потребует править все такие операции. Это нарушает принцип **одного уровня абстракции** (single layer abstraction).

В сложных системах применяют **уровневое проектирование**: структура строится слоями. Каждый уровень комбинирует части, которые на нём считаются элементарными; эти части становятся кирпичиками следующего уровня.

> Уровневое проектирование пронизывает всю технику построения сложных систем. Например, при проектировании компьютеров резисторы и транзисторы сочетаются (и описываются при помощи языка аналоговых схем), и из них строятся и-, или- элементы и им подобные, служащие основой языка цифровых схем. Из этих элементов строятся процессоры, шины и системы памяти, которые в свою очередь служат элементами в построении компьютеров при помощи языков, подходящих для описания компьютерной архитектуры. Компьютеры, сочетаясь, дают распределённые системы, которые описываются при помощи языков описания сетевых взаимодействий, и так далее. (c) SICP

```
const mul = (rational1, rational2) => {
  return makeRational(
    getNumer(rational1) * getNumer(rational2),
    getDenom(rational1) * getDenom(rational2),
  )
}
```

Базовый уровень — встроенные типы языка: числа и объекты. На них — уровень рациональных чисел: `makeRational`, `getDenom`, `getNumer`. Выше — арифметика: сложение, вычитание, умножение.

Речь о реализации уровней. Сложение опирается только на конструктор и селекторы и не знает внутреннего устройства дроби. При этом в одной функции могут соседствовать вызовы разных уровней — это нормально:

```
const f = (rational1, rational2) => {
  const rational3 = sum(rational1, rational2)
  const denom = getDenom(rational3)
  const numer = getNumer(rational3)
  console.log(`Denom: ${denom}`)
  console.log(`Numer: ${numer}`)
}
```
