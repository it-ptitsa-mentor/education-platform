# Интерфейсы

В IT термин **интерфейс** близок к бытовому смыслу. Пользовательский интерфейс — элементы управления сайтом, банкоматом, телефоном. У пульта телевизора интерфейс — кнопки; у автомобиля — рычаги и кнопки. Интерфейс задаёт способ взаимодействия с системой.

Хороший интерфейс сделать сложнее, чем кажется. Неудобные решения встречаются повсюду — от дверей до лифтов. Чем больше состояний у системы, тем труднее спроектировать управление. Даже у кнопки питания телевизора (вкл/выкл) можно сделать две кнопки или одну с разным поведением в зависимости от состояния.

В программировании **интерфейс** — набор функций (сигнатуры: имена, параметры, возвращаемые значения), не привязанный к конкретной реализации. Это совпадает с [абстрактным типом данных](https://ru.wikipedia.org/wiki/%D0%90%D0%B1%D1%81%D1%82%D1%80%D0%B0%D0%BA%D1%82%D0%BD%D1%8B%D0%B9_%D1%82%D0%B8%D0%BF_%D0%B4%D0%B0%D0%BD%D0%BD%D1%8B%D1%85). Для точек интерфейс — функции из практики и теории этого блока.

Как связаны **абстракция** и **интерфейс**? Абстракция описывает данные, с которыми работаем — например, «пользователь» в веб-приложении. Интерфейс — функции для работы с этими данными.

Функции бывают интерфейсные и вспомогательные: последние не для внешнего кода, а для внутренней логики абстракции.

```
// Функции makeUser, getAge, isAdult — интерфейс абстракции User.
// Они используются внешним (пользовательским, вызывающим) кодом.
const makeUser = (name, birthday) => ({ name, birthday })

const getAge = user => calculateAge(user.birthday)

const isAdult = user => getAge(user) >= 18

// Эта функция не является частью интерфейса абстракции User.
// Она является "внутренней".
const calculateAge = (birthday) => {
  const milliSecondsInYear = 31556926 * 1000

  // Date.now() – возвращает количество миллисекунд,
  // прошедших с 1 января 1970 года 00:00:00 по UTC
  // Date.parse(str) – разбирает строковое представление даты
  // и также возвращает количество миллисекунд
  return Math.trunc((Date.now() - Date.parse(birthday)) / milliSecondsInYear)
}
```

В крупных абстракциях (в том числе в библиотеках) внутренних функций обычно больше, чем публичных — иногда сотни реализации против одной-двух точек входа. Качество абстракции во многом определяется удобством её интерфейса.
