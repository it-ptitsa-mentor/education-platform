# Теория: Особенности работы this со стрелочными функциями

Стрелочные функции отличаются от обычных не только синтаксисом. Главное — работа с контекстом: у обычной функции `this` зависит от способа вызова, у стрелочной — от места, где функция объявлена.

Несколько примеров. На уровне модуля контекстом вызова будет сам модуль.

*Примечание: примеры ниже приведены для Node.js. В браузере **`this`** по умолчанию будет содержать глобальный объект **`Window`**.*

```
const f1 = () => { // стрелочная функция
  console.log(this)
}

f1() // undefined

function f2() { // обычная функция
  console.log(this)
}

f2() // undefined
```

Поведение совпадает: контекст — модуль, а в ES-модулях `this` не задан. Добавим обе функции в объект:

```
const obj = {
  f1, f2,
}

obj.f1() // undefined
obj.f2() // { f1: [Function: f1], f2: [Function: f2] }
```

Обычная функция привязалась к объекту вызова. У стрелочной своего `this` нет — она берёт лексическое окружение (функцию, внутри которой объявлена). Контекст задаёт функция верхнего уровня, а не объект. Это нельзя переопределить через `call` или `bind`:

```
f1.call({ name: 'world' }) // undefined
f1.bind({ name: 'world' })() // undefined
```

Объявим стрелочную функцию внутри объекта:

```
const company = {
  f1: () => { // стрелочная функция
    console.log(this)
  },
  f2() { // обычная функция
    console.log(this)
  },
}

company.f1() // undefined
company.f2() // { f1: [Function: f1], f2: [Function: f2] }
```

Картина та же: стрелочная объявлена в модуле, значит `this` — контекст модуля, а не `company`.

Теперь стрелочная внутри другой функции:

```
const printer = {
  items: [1],
  print() { // важно что внешняя функция имеет контекст
    // Стрелочная функция определяется внутри функции print,
    // но вызывается внутри метода forEach
    this.items.forEach(() => console.log(this.items))
  },
}

printer.print() // [1]
```

Кажется, у стрелочной появился `this`, но он заимствован у `print`. Внутри `forEach` колбек вызывается как обычная функция, не как метод. Обычная функция в такой ситуации получила бы свой `this`; стрелочная — `this` из места объявления, то есть из `print`.

Тот же код с обычной функцией падает:

```
const printer = {
  items: [1],
  print() {
    this.items.forEach(function () { console.log(this.items) })
  },
}

printer.print() // Error!
```

Колбек вызван как функция, не как метод — контекст глобальный (часто `undefined`), и `this.items` даёт ошибку.

Где это нужно? В стрелочных колбеках внутри методов, когда нужен `this` внешнего метода — типичный случай с `forEach`, `map` и т.п., как в примере выше.
