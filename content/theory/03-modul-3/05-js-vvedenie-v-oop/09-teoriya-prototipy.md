# Теория: Прототипы

<!-- IMG (из Buildin, перезалить отдельно) -->
*Здесь — краткое введение в прототипы. Одним уроком тему не закрыть, даже отдельного курса мало. Прототипы приходят с практикой, чередующейся с теорией. Если не всё ясно с первого раза — это нормально.*

Прототипы сильнее всего определяют поведение объектов в JavaScript. В прикладном коде их редко трогают напрямую (чаще в библиотеках), но без понимания прототипов трудно отлаживать код и разбираться с классами дальше по курсу. В уроке — только основы; углублённо тему разбирают дополнительные материалы.

У каждого объекта в JavaScript есть связанный прототип — обычный объект в служебном поле `[[prototype]]` (напрямую к нему не обратиться). Получить прототип можно так:

```
const date = new Date()
// Эта функция извлекает прототип объекта из самого объекта
const proto = Object.getPrototypeOf(date) // Date {}

// В прототипе хранится не конструктор
// Что там хранится – узнаем дальше
proto === Date // false

const numbers = [1, 2]
Object.getPrototypeOf(numbers) // [] – отображение отличается, но это массив

// Прототипы есть и у конструкторов, которые мы определяем сами
function Company(name) {
  this.name = name
}

const company = new Company()
Object.getPrototypeOf(company) // Company {}
```

## Для чего нужны прототипы

Обратимся к свойству объекта, которого в самом объекте нет — обычно получим `undefined`. Но есть важный нюанс.

Если свойства нет в объекте, движок смотрит его прототип. Нашли свойство в прототипе — вернётся его значение. Так можно читать свойство, которого нет у экземпляра, но есть у прототипа.

На практике цепочка длиннее: не найдено в прототипе — поиск идёт по прототипу прототипа и так до `null`. На этом строится наследование — тема за рамками урока.

Прототипы есть даже у простых объектов:

```
Object.getPrototypeOf({}) // {} — это и есть Object
```

Поэтому у пустого объекта уже есть унаследованные свойства и методы:

```
const obj = {} // То же самое можно сделать так: const obj = new Object();
// Это функция-конструктор, из которой был получен текущий объект, в нашем случае — Object
obj.constructor // [Function: Object]
// У obj нет своего собственного свойства constructor, оно пришло из прототипа
Object.hasOwn(obj, 'constructor') // false
Object.hasOwn(obj, 'name') // false
obj.name = 'collection'
// Имя есть в самом объекте, потому что мы его только что добавили
Object.hasOwn(obj, 'name') // true
```

К прототипу можно обратиться и через свойство `prototype` у конструктора, который создаёт экземпляры:

```
function Company(name) {
  this.name = name
}

// Одно и то же, полученное разными способами
// Company.prototype === Object.getPrototypeOf(new Company())
```

Откуда берётся прототип? Это объект в свойстве `prototype` функции-конструктора, а не сам конструктор. Проверить работу прототипов легко, изменив их:

```
// Добавляем свойство getName (делаем его методом)
Company.prototype.getName = function getName() {
  // this по-прежнему зависит от контекста, в котором вызывается
  return this.name
}

const company = new Company('Рога и копыта')
// Свойство доступно!
console.log(company.getName()) // => Рога и копыта
```

Ничто не мешает переопределить `getName` только в одном экземпляре — остальные по-прежнему возьмут метод из прототипа:

```
const company1 = new Company('Рога и копыта')
const company2 = new Company('Google')
company2.getName = function getName() {
  return 'Alphabet'
}

// Этот вызов возьмет свойство из самого объекта
company2.getName() // Alphabet
// Этот вызов возьмет значение свойства из прототипа
company1.getName() // Рога и копыта
```

Правильный способ строить свои абстракции в JavaScript — конструктор и прототип с нужными свойствами и методами.

Что даёт механизм?

Проще всего — расширять встроенные типы и библиотеки без доступа к исходникам. Прототипы гибки: в рантайме можно менять почти что угодно. Например, добавить или подменить метод у встроенных объектов:

```
const numbers1 = [1, 3]

// Как только выполнится этот код, все массивы,
// включая уже созданные, обзаведутся методом last
Array.prototype.last = function last() {
  // Такое обращение сработает, ведь this — это ссылка на сам объект,
  // который в нашем случае является массивом
  return this[this.length - 1]
}

numbers1.last() // 3

const numbers2 = [10, 0, -2]
numbers2.last() // -2

// Пример замены
// Это запрещенный прием, никогда так не делайте в реальном коде
Array.prototype.map = function map() {
  return 'Ehu!'
}

numbers1.map() // "Ehu!"
```

Мощный механизм и опасный — можно сломать ожидаемое поведение:

```
// Очень злой код, который ломает работу метода push
Array.prototype.push = function push(value) {
  return this.unshift(value)
}

const numbers = [1, 2]
numbers.push(3)
console.log(numbers) // => [3, 1, 2] !!!
```

При разумном использовании прототипы полезны: в [тестировании](https://github.com/nock/nock) ими подменяют нежелательные HTTP-вызовы; в полифилах добавляют в старые движки методы и свойства из новых версий языка.

Теперь понятно, почему в документации методы записывают как `Number.prototype.toLocaleString()`.
