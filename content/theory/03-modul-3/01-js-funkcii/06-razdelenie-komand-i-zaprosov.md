# Разделение команд и запросов

Command-query Separation (CQS) — принцип, сформулированный Бертраном Мейером, автором языка Eiffel. По CQS каждая функция либо **команда** (выполняет действие), либо **запрос** (возвращает данные), но не то и другое сразу. Команды связаны с побочными эффектами; чистые функции возможны только у запросов.

## Команда

```
// Возвращает true или false как результат своего выполнения
save(user)
```

По CQS `save()` — команда. Она может сообщить об успехе (`true`/`false`, иногда `undefined`, как у `console.log()`). Возврат осмысленных данных из команды считается нарушением принципа — хотя бывают исключения: открытие файла на запись возвращает дескриптор, без которого дальнейшая работа невозможна.

```
const file = fopen('/etc/hosts', 'r')
```

Разделение команд и запросов перекликается с уроком о чистых функциях. Команды по определению недетерминированы и с эффектами: повторный вызов либо падает, либо повторяет действие (детерминировать их можно, но часто это маскирует ошибки). Вынести «чтение» из команды в отдельный запрос — способ изолировать чистый код от кода с эффектами.

## Запрос

```
// Возвращает true или false
isAdmin(user)
```

`isAdmin()` — предикат, типичный запрос: вопрос «пользователь — администратор?». По CQS такая функция не должна менять состояние (дату проверки, роль и т.п.) — это противоречит и принципу, и здравому смыслу. Здесь `true`/`false` — ответ на вопрос, а не признак успешности выполнения.

Пример запроса, который портит исходные данные:

```
const users = [
  { name: 'Stan', children: ['John', 'Mary'] },
  { name: 'Donald', children: ['James'] },
  { name: 'Lily', children: [] },
]

// takeChildren() возвращает массив детей всех пользователей
takeChildren(users) // ['John', 'Mary', 'James']
// На самом деле внутри она меняет массив users и возвращает его наружу
console.log(users) // => ['John', 'Mary', 'James']
```

Повторный `takeChildren(users)` скорее всего упадёт — структура массива уже другая. Для запроса такое поведение неестественно. Альтернативная формулировка CQS: «задавая вопрос, не меняй ответ».

К запросам относят и чистые вычисления:

```
// Функция Math.max() возвращает максимальное число из переданных
const getMaxNumber = Math.max(1, 30, 4) // 30
```

Без эффектов, детерминированно — можно вызывать сколько угодно.

Запрос не должен менять данные — это правило стоит соблюдать всегда. Интуитивно никто не ждёт, что `isAdmin()` или поиск максимума что-то сломает; на практике такой код всё же встречается, и теперь вы знаете, как его чинить.

Когда команду и запрос не разделить? Допустим, нужно сформировать имя пользователя, если его нет, и записать в БД:

```
// Проверяем есть ли
let username = getName()
if (!username) {
  // Если имени нет, то генерируем и запоминаем
  username = generateName()
  setName(username)
}
```

Если спрятать это в одну функцию:

```
const username = getUserName()
```

Имя вводит в заблуждение: похоже на запрос, а внутри — и генерация, и запись. Без чтения реализации не догадаться.

В таких случаях называйте функцию как команду — скрытых смыслов меньше. CQS всё равно нарушен, но намерение явное:

```
const username = setUserName()

// Или даже так
const username = setUserNameIfEmpty()
```
