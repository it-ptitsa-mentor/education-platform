# Итеративный процесс

Рекурсию можно организовать по-разному. Вариант из прошлого урока — **рекурсивный процесс**. Этот урок опирается на него; лучше сначала разобрать рекурсию и сделать упражнение.

Другой вариант — **итеративный процесс**. Оба описывают рекурсию в коде; различается способ вычисления.

В рекурсивном процессе каждый новый «ящик» `factorial()` ждёт возврата от следующего; умножения откладываются до базового случая. Для `factorial(100)` в памяти копится много отложенных операций. **Откладывание** — суть рекурсивного процесса: прервать на полпути — полезного ответа нет. Это как откладывать всю работу на пятницу после обеда.

Всё, что программа помнит в момент времени (вычисления, константы, функции), — **состояние**. С состоянием связано много типичных проблем.

**Итеративный процесс** распределяет работу по шагам: умножать по ходу, а не копить в стеке.

Факториал итеративно:

```
const factorial = (n) => {
  if (n === 0) {
    return 1
  }

  const iter = (counter, acc) => {
    if (counter === 1) {
      return acc
    }
    return iter(counter - 1, counter * acc)
  }

  return iter(n, 1)
}
```

Код уже не похож на формулу и на рекурсивный вариант: рекурсивный процесс читается ближе к идее, итеративный обычно эффективнее, но сложнее.

Внутри — вложенная `iter(counter, acc)`:

- `counter` — от *n* до 1

- `acc` — накопленное произведение

При `counter === 1` возвращается `acc`. Снаружи: `return iter(n, 1)`.

При `factorial(3)` сначала `iter(3, 1)`. 3 ≠ 1 → `iter(2, 3)` (уже одно полезное умножение). Затем `iter(1, 6)` → возврат 6.

Пошагово:

```
iter(3, 1) // iter(3 - 1, 3 * 1);
iter(2, 3) // iter(2 - 1, 2 * 3);
iter(1, 6) // counter === 1, return 6
6
```

Состояние всегда из двух чисел — размер фиксирован.

Шаблон итеративного процесса:

1. Начальное состояние (`iter(n, 1)`)

1. Терминальный случай (`counter === 1`)

1. Переход (`iter(counter - 1, counter * acc)`)

1. Повтор до терминала

Кратко:

1. **Рекурсия** — определение через само себя

1. **Рекурсивный процесс** — отложенные вычисления

1. **Итеративный процесс** — фиксированное число значений состояния

В `iter()` нет `else` — после `return acc` второй `return` не выполняется:

```
const iter = (counter, acc) => {
  if (counter === 1) {
    return acc
  }
  return iter(counter - 1, counter * acc)
}
```

```
const someFunction = (a) => {
  return 100
  return a
  return 4123123
}
```

Всегда вернётся `100` — после первого `return` экземпляр функции завершён.

### Резюме

Рекурсивный `factorial` из прошлого урока:

```
factorial(3) // ничего не происходит, вычисляем множители
3 * factorial(2) // ничего не происходит, вычисляем множители
3 * 2 * factorial(1) // ничего не происходит, вычисляем множители
3 * 2 * 1 // наконец начинаем умножение
3 * 2
6
```

Это **рекурсивный процесс** с **откладыванием**.

**Состояние** — всё, что помнит программа. **Итеративный процесс** держит состояние ограниченным.

```
const factorial = (n) => {
  if (n === 0) {
    return 1
  }

  const iter = (counter, acc) => {
    if (counter === 1) {
      return acc
    }
    return iter(counter - 1, counter * acc)
  }

  return iter(n, 1)
}
```

Идея:

1. Считаем от n до 1

1. Берем число из предыдущего шага

1. Умножаем это число на текущее число

1. Передаем его в новый шаг

1. Когда counter достигает 1, число передается из предыдущего шага в ответ

Старт с `iter(n, 1)`, потому что шаг 2 требует значение с прошлого шага, а *n × 1 = n*:

```
return iter(n, 1)
```

Для *3!*:

```
iter(3, 1) // iter(3 - 1, 3 * 1);
iter(2, 3) // iter(2 - 1, 2 * 3);
iter(1, 6) // counter === 1, return 6
6
```
