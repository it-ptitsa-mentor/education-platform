# Парадигмы программирования

В программировании часто говорят о **парадигме**.

> Паради́гма программи́рования — совокупность идей и понятий, задающих стиль написания программ (подход к программированию). Это способ концептуализации: как организованы вычисления и как структурирована работа, которую выполняет компьютер. (Wikipedia)

Парадигма — не просто другой алгоритм: при смене парадигмы структура кода меняется сильно и нужны идеи шире синтаксиса (например, автоматное программирование опирается на теорию автоматов). Большинство современных языков **мультипарадигменны**; стили то исключают друг друга, то дополняют. К этому моменту мы уже писали в двух парадигмах: императивной и декларативной.

## Императивная парадигма

Императивный стиль — последовательность инструкций и активное использование переменных. На практике он доминирует; почти весь код до этого курса был императивным.

```
// Поиск максимального числа
const numbers = [10, 20, 52, 105, 56, 89, 96]

let max = numbers[0]
for (const number of numbers) {
  if (number > max) {
    max = number
  }
}

console.log(max) // => 105
```

Присваивание, переменные, циклы. Парадигма близка к модели компьютера: шаг за шагом, память для промежуточных результатов. Императивная программа отвечает на вопрос **КАК**.

JavaScript (как Java, Ruby, Python, C#, Perl, PHP, Go) — императивный язык в том смысле, что императивный стиль — основной, но другие парадигмы тоже доступны.

## Декларативная парадигма

Декларативному (часто говорят **функциональному**) стилю противопоставляют императивный: программа похожа на спецификацию, а не на список команд. Вопрос **ЧТО**, а не **КАК**. Математика по сути декларативна.

```
// Поиск максимального числа
const numbers = [10, 20, 52, 105, 56, 89, 96]

const max = numbers.reduce(
  (acc, number) => number > acc ? number : acc,
  numbers[0],
)

console.log(max) // => 105
```

На практике декларативный код избегает **присваивания** после инициализации. `const myVar = expression` ближе к «пусть A — множество чисел»: имя не меняется по ходу рассуждения.

*В математике доказательства — цепочки утверждений. Если бы утверждения менялись, выводы ломались бы задним числом.*

То же с `acc` и `number` в колбеке: каждый вызов независим. В функциональных языках это **связывание**, не присваивание; повторное связывание — ошибка (пример на Erlang):

```
1> A = 4.
4
2> A = 'hey'.
** exception error: no match of right hand side value hey
```

Без присваивания нет и циклов — вместо них рекурсия. Важна и работа с функциями как с данными (функции высшего порядка); они часто заменяют рекурсию — мы уже прошли `map`, `filter`, `reduce`.

Чисто декларативные языки: Haskell, Erlang, Elixir, OCaml, F# — без присваивания и циклов. Scala, Clojure и LISP-семейство толкают к декларативности, но императивный код возможен. Императивные языки, наоборот, позволяют писать декларативно; у современного JavaScript эта поддержка сильная.
