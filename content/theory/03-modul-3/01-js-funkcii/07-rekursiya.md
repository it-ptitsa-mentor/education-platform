# Рекурсия

Рекурсия — вызов функции из самой себя. В математике многие определения устроены так же, и языки программирования перенимают этот приём. В JavaScript в теле функции можно ссылаться на только что объявленную функцию:

```
const f = () => {
  f()
}
```

Такой вызов не остановится сам — бесконечная рекурсия. Осмыслим идею на бытовом примере: три книги на полке — сколько способов их расставить?

Получается шесть перестановок из трёх книг. Из четырёх — 24. Из 13 — почти столько же, сколько людей на планете. 25 книг? Вариантов больше, чем атомов в организме человека.

Всего *n!* перестановок *n* книг: перемножить целые от 1 до *n*. 3! = *1 × 2 × 3*. Напишем факториал:

```
const factorial = (n) => {
  return 1 * 2 * 3 * 4 //
}
```

Не сходится: *n* заранее неизвестно. Из математики: если *n* = 0, факториал 1; иначе *n × (n−1)!*.

```
const factorial = (n) => {
  if (n === 0) {
    return 1
  }
  else {
    return n * factorial(n - 1)
  }
}

const answer = factorial(3)
```

Странно на первый взгляд: функция зовёт саму себя. Дело в том, что определение функции — не «ящик», а **описание** ящика. Ящик появляется при вызове и исчезает после возврата. Новый вызов — новый ящик.

Проследим `factorial(3)`. 3 ≠ 0, нужно умножение, но второй множитель — результат `factorial(2)`.

Новый ящик, 2 ≠ 0, снова нужен `factorial(1)`.

Ещё ящик, 1 ≠ 0, вызов `factorial(0)`.

`factorial(0)` сразу возвращает 1.

1 × 1 → 1, × 2 → 2, × 3 → 6 в `answer`. Готово.

Рекурсия описывает объект через себя. Нужны два условия:

1. **Базовый** (терминальный) случай — где остановиться. Здесь *n* = 0.

1. **Шаг углубления** — здесь `n * factorial(n - 1)`.

Линтер может ругаться на `no-else-return`. Упростим:

```
const factorial = (n) => {
  if (n === 0) {
    return 1
  }
  return n * factorial(n - 1)
}

const answer = factorial(3)
```

Пошагово без «заглядывания в ящики»:

```
factorial(3);
3 * factorial(2);
3 * 2 * factorial(1);
3 * 2 * 1 * factorial(0);
3 * 2 * 1 * 1;
3 * 2 * 1
3 * 2;
6;
```

Умножения откладываются до базового случая, затем идут снизу вверх.

Рекурсия широко используется, особенно в функциональном стиле — не только для формул. Данные часто иерархичны: HTML-элементы внутри элементов; браузер рекурсивно обходит дерево, чтобы отрисовать страницу.

С рекурсией вы ещё не раз столкнётесь в этом и следующих курсах — это мощный инструмент.
