# Чистые функции

У функций есть свойства, которые помогают решать, как дробить код и когда выносить логику в отдельные функции.

## Детерминированность

[Math.random()](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Math/random) даёт случайное число от *0* до *1*:

```
Math.random() // 0.9337432365797949
Math.random() // 0.5550694016887598
```

При одних и тех же входах (включая «нет аргументов») результат разный — функция **недетерминированная**. То же с системным временем: [Date.now()](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Global_Objects/Date/now) каждый раз новое:

```
// Возвращает текущее время в миллисекундах
Date.now() // 1571909874844
Date.now() // 1571909876648
```

`getAge(2000)` сегодня и через год даст разные ответы при том же аргументе — значит, недетерминированна.

**Детерминированные** функции при одинаковых входах всегда дают один выход — как в математике.

`console.log()` детерминированна в смысле возврата: всегда `undefined`, а не то, что печатается. Печать — побочный эффект.

```
console.log('Boom – Big Bang')
```

Два действия: вывод в консоль и возврат `undefined` — какое бы сообщение ни передали.

Недетерминированность появляется и при чтении внешнего состояния — глобальных переменных, окружения и т.д.:

```
const getCurrentShell = () => process.env.SHELL

getCurrentShell() // /bin/bash
```

`SHELL` в разных средах и моментах времени может отличаться.

Полная детерминированность не всегда возможна (случайность, дата нужны в проде), но полезно максимизировать детерминированный код: меньше глобалов, больше зависимости только от аргументов.

Детерминированность — не только про код: подбрасывание монеты — недетерминированный процесс.

## Побочные эффекты (side effects)

Второй признак — **побочные эффекты**: любое взаимодействие с внешним миром — файлы, сеть, консоль.

```
const someFunction = () => {
  // Функция fetch выполняет HTTP-запрос
  // HTTP-запрос — это побочный эффект
  fetch('https://ru.boom.io/courses')
}
```

Эффектом считают и изменение внешних переменных, и мутацию аргументов, переданных по ссылке:

```
const someFunction = (obj) => {
  // Какая-то логика
  // Побочный эффект. Изменение входного аргумента.
  obj.key = 'value'
}
```

Чистое сложение без эффектов:

```
const sum = (num1, num2) => num1 + num2
```

Эффекты усложняют логику и тесты; у файловых операций сотни способов упасть. Полностью без эффектов полезная программа не обойдётся — результат надо куда-то вывести:

```
console.log(sum(4, 11)) // => 15
```

В приложениях эффекты обычно сосредоточены на краях (БД, API), а ядро — вычисления.

*Инкремент и декремент в JS — единственные базовые арифметические операции с побочным эффектом (меняют переменную). В Ruby и Python их нет; в JS стайлгайды часто советуют не использовать `++`/`--`.*

## Чистые функции

<!-- IMG (из Buildin, перезалить отдельно) -->
**Чистая** (pure) функция — детерминированная и без побочных эффектов; зависит только от аргументов.

Плюсы:

- Просто тестировать: подали вход — проверили выход.

- Безопасно вызывать повторно (асинхронность, потоки).

- Удобно комбинировать без переписывания всей программы.

В хорошей архитектуре эффекты изолируют, большая часть кода остаётся чистой.

Сейчас это может звучать абстрактно; понимание приходит с практикой и болью от «каши» эффектов. Тему будем возвращать, особенно в проектах.
