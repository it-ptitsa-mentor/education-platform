# Абстракция с помощью функций

Главная причина выделять функции — поднять уровень абстракции, а не просто убрать копипаст. Дублирование кода часто исчезает как побочный эффект. Вместо того чтобы в каждом месте писать свою сортировку, вызывают `sort()` — детали алгоритма спрятаны внутри.

Суть абстракции — отбросить несущественное и описать сложное проще. Это базовый навык мышления: ребёнок узнаёт «дерево» без знания породы. Умение выбирать границу абстракции приходит с опытом, обратной связью и рефлексией над тем, как код используется. Избыточные слои, наоборот, мешают: теряется смысл, растёт цена понимания. Абстракция — всегда компромисс: слишком низкий уровень даёт дублирование, слишком высокий — непроницаемую сложность (сравните с [теорией категорий](https://ru.wikipedia.org/wiki/%D0%A2%D0%B5%D0%BE%D1%80%D0%B8%D1%8F_%D0%BA%D0%B0%D1%82%D0%B5%D0%B3%D0%BE%D1%80%D0%B8%D0%B9)).

Функции высшего порядка поднимают абстракцию ещё выше: общий алгоритм остаётся одним, а различия выносят в колбеки. Вместо десяти почти одинаковых функций — одна, настраиваемая десятью способами.

Помните: [абстракции почти всегда «протекают»](https://www.joelonsoftware.com/2002/11/11/the-law-of-leaky-abstractions/).

## Пример дырявой абстракции

Типичная ошибка новичков при выделении абстракций. Задача в упрощении: по числу вывести *yes*, если оно чётное, и *no* иначе.

Первый вариант:

```
const check = (number) => {
  const result = number % 2 === 0 ? 'yes' : 'no'
  console.log(result)
}
```

Работает, но чётность не выделена — тестировать и читать сложнее. Логично вынести проверку:

```
const isEven = number => (number % 2 === 0) ? 'yes' : 'no'

const check = (number) => {
  const result = isEven(number)
  console.log(result)
}
```

Но этот код хуже первого: абстракция неверная. Чётность — математическое свойство, оно не знает про экран и строки *yes*/*no*. Имя `isEven` с префиксом *is* намекает на предикат — он должен возвращать только boolean. Правильнее так:

```
const isEven = number => (number % 2 === 0)
const check = (number) => {
  const result = isEven(number) ? 'yes' : 'no'
  console.log(result)
}
```

В боевом коде границы абстракций устроены сложнее.
