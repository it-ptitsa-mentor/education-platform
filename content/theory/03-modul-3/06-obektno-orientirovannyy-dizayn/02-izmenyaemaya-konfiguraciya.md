# Изменяемая конфигурация

Многие объекты в ООП — не абстракция данных, а **контейнер настроек** для повторяющихся операций: рендер Markdown в HTML, геолокация по IP и т.п. Опции передают в конструктор, хранят внутри и применяют при каждом вызове.

```
// timeout устанавливает ограничение в одну секунду на длительность запроса
const ipgeo = new IpGeo({ timeout: 1000 })
ipgeo.resolve('123.4.3.2')
```

Но иногда для одного запроса нужны другие опции. Есть три типичных ответа на эту ситуацию.

## Создание нового объекта

Самый прямой путь — создать ещё один экземпляр там, где нужны особые настройки. Просто, но с минусами: объект собирают не на этапе конфигурации системы, а в месте вызова, поэтому теряется полиморфизм (подмена реализации), дублируются общие опции, тестировать сложнее.

```
const ipgeo = new IpGeo({ timeout: 10 })
ipgeo.resolve('123.4.3.2')
```

## Сеттеры

Худший вариант — менять опции через сеттеры.

```
const ipgeo = new IpGeo({ timeout: 1000 })

// В одной части программы
ipgeo.resolve('123.4.3.2')

// В другой части программы
ipgeo.setOption('timeout', 10)
ipgeo.resolve('123.4.3.2')
```

Изменяемое состояние — один из главных источников трудноуловимых багов. Общий объект `ipgeo` используют разные части программы: правка в одном месте меняет поведение для всех последующих вызовов. С Markdown риск ещё выше — неверный вывод открывает [XSS](https://ru.wikipedia.org/wiki/%D0%9C%D0%B5%D0%B6%D1%81%D0%B0%D0%B9%D1%82%D0%BE%D0%B2%D1%8B%D0%B9_%D1%81%D0%BA%D1%80%D0%B8%D0%BF%D1%82%D0%B8%D0%BD%D0%B3):

```
const md = new Markdown({ sanitize: true })

// В одной части программы
md.render(markdown)

// В другой части программы отключаем санитайз
md.setOption('sanitize', false)
md.render(markdown2)
```

`sanitize` включает безопасный рендеринг. Если выключить, теги `<script>` из Markdown попадут в HTML как есть — допустимо для своего текста, недопустимо для пользовательского ввода. «Временно выключить и вернуть обратно» звучит разумно, но на практике забывают восстановить:

```
const md = new Markdown({ sanitize: true })

// В одной части программы
md.render(markdown)

// В другой части программы отключаем санитайз
md.setOption('sanitize', false)
md.render(markdown2)
md.setOption('sanitize', true)
```

Паттерн «поменял — сделал дело — откатил» почти всегда сигнал проблем в архитектуре; лучше перейти на безопасный вариант ниже.

## Новые опции на время запроса

Надёжный способ — передать временные опции аргументом метода. Они действуют только на этот вызов.

```
const md = new Markdown({ sanitize: true })

// В одной части программы
md.render(markdown)

// В другой части программы отключаем санитайз на время выполнения запроса
md.render(markdown2, { sanitize: false })
md.render(markdown3) // sanitize по прежнему равен true
```

Глобально `sanitize` включён; для `markdown2` его переопределили в вызове `render`, и следующие вызовы это не затрагивают.
