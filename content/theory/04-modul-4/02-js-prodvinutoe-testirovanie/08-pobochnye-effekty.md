# Побочные эффекты

Проще всего тестировать чистые функции: данные на вход, результат на выход, без сюрпризов, без состояния, без контакта с внешним миром.

```
import _ from 'lodash'

expect(_.last([1, 2, 3])).toBe(3)
```

Не весь код чистый. Без побочных эффектов не обходится ни одна реальная программа — результат нужно куда-то записать, отправить, сохранить.

Побочные эффекты сильно усложняют тесты: нужны более глубокие навыки и продуманная организация кода.

Примеры побочных эффектов:

- Случайные числа

- HTTP-запросы

- Отправка писем

- Взаимодействие с базой данных

- Взаимодействие с глобальными переменными

- Чтение и запись файлов

- Оперирование текущим временем

В чём сложность? Функция отправляет приветственное письмо:

```
if (sendGreetingEmail(user)) {
  // Вывести на сайте сообщение о том, что письмо было отправлено
};
```

Гипотетический тест:

```
expect(sendGreetingEmail(user)).toBe(true)
```

Проверяется только возврат `true`. Неясно, ушло ли письмо, какое именно и корректно ли оно.

Реальность ещё жёстче: настоящие письма в тестах отправлять нельзя. Это неэтично к людям. Даже на фейковые адреса мы цепляемся к внешней системе — тесты медленнее, чем у чистых функций. Внешняя среда недетерминирована: сеть падает, тесты «моргают» без видимой причины; при частой рассылке почтовик может заблокировать IP. Плюс риски безопасности.

И это только почта. У других побочных эффектов — свои ловушки и свои приёмы изоляции. В следующих уроках разберём распространённые случаи и способы с ними работать.
