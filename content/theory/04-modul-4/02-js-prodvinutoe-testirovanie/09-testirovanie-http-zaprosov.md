# Тестирование HTTP-запросов

Инверсия зависимостей работает не только с функциями, но и с объектами. Разберём её подробнее на HTTP и познакомимся со **заглушкой** (стаб, stub).

Допустим, функция смотрит приватные репозитории организации на GitHub и возвращает те, что являются форками:

```
// Библиотека для работы с GitHub API
import { Octokit } from '@octokit/rest'

const getPrivateForksNames = async (username) => {
  const client = new Octokit()
  // Клиент выполняет запрос на гитхаб и возвращает список приватных репозиториев указанной организации
  // Данные хранятся в свойстве data возвращаемого ответа
  const { data } = await client.repos
    .listForOrg({
      username,
      type: 'private',
    })
  // Оставляем только имена форков
  return data.filter(repo => repo.fork).map(repo => repo.name)
}
```

Что хотим проверить? Прежде всего — что функция возвращает массив имён приватных форков. Идеальный тест выглядел бы так:

```
test('getPrivateForksNames', async () => {
  const privateForks = await getPrivateForksNames('yandex')
  expect(privateForks).toEqual([/* массив имен, которые мы ожидаем увидеть */])
})
```

На практике не так просто: внутри — HTTP-запрос. Какие проблемы это даёт:

1. Нестабильная сеть может тормозить выполнение тестов и приводить к фантомным ошибкам. Тесты будут иногда проходить, иногда нет

1. У сервисов подобных *github.com* установлены ограничения на запросы в секунду, в час, день и так далее. Со 100% вероятностью тесты начнут упираться в эти лимиты. Более того, есть шанс, что машина с которой идут запросы, будет заблокирована

1. Реальные данные на GitHub не статичны, они могут и, скорее всего, будут меняться, что опять же приведет к ошибкам и необходимости править тесты

Здесь HTTP мешает проверить основную логику. Мы доверяем *github.com* и библиотеке *@octokit/rest* — проверять их работу не нужно (иначе можно сойти с ума, если не доверять никому).

Из прошлого урока известны способы обойти это — применим один из них.

## Инверсия зависимостей

Передадим клиент *Octokit* вторым аргументом — в тестах его можно подменить:

```
import { Octokit } from '@octokit/rest'

// Библиотека передается снаружи и ее можно подменить
const getPrivateForksNames = async (username, client = new Octokit()) => {
  // ...
}
```

Нужен фейковый клиент с тем же интерфейсом, но без сети, и заранее заданные данные для `listForOrg` — иначе не проверить поведение функции.

```
// Структура этого класса описывает только ту часть,
// которая необходима для вызова await client.repos.listForOrg(...)
class OctokitFake {
  // Здесь мы описываем желаемые данные, которые вернутся в тесте
  constructor(data) {
    this.data = data
  }

  repos = {
    listForOrg: () => {
      // Структура возврата должна совпадать с реальным клиентом
      // Только в этом случае получится прозрачно подменить реальный клиент на фейковый
      return Promise.resolve({ data: this.data }) // так как метод асинхронный
    },
  }
}
```

Тест с этим клиентом:

```
import OctokitFake from '/OctokitFake.js';

test('getPrivateForksNames', async () => {
  const data = /* ответ от GitHub, который мы хотим проверить */;
  const client = new OctokitFake(data);
  // Внутри выполняется запрос, который возвращает сформированные выше данные
  const username = /* имя пользователя на гитхабе */;
  const privateForks = await getPrivateForksNames(username, client);
  expect(privateForks).toEqual(/* что мы ожидаем основываясь на том, что вернул listForOrg */);
});
```

Такие подмены называют стабами (stub). Стаб заменяет реальный объект или функцию, убирая побочный эффект или делая поведение детерминированным. Стаб не проверяет вызовы — он лишь изолирует то, что мешает тестировать логику.

## Запрет HTTP-запросов

Другой способ — запретить HTTP в тестах. Дальше познакомимся с Nock и методом `nock.disableNetConnect()` — его стоит вызывать в начале файла с тестами. Он также показывает, куда ходят сторонние библиотеки. После отключения сети (если запросы не подменены) вывод может быть таким:

```
HttpError: request to https://api.github.com/orgs/yandex/repos?type=private failed, reason: Nock: Disallowed net connect fo
```
