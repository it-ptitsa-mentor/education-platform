# Введение

JavaScript изначально создавали для клиентской стороны — это повлияло на язык и на среды исполнения: браузеры, Node.js и другие. Браузер заточен под **событийную модель**: код не крутится непрерывно, а ждёт действий пользователя — клик, скролл, ввод. На одно событие может навешано много обработчиков; они не должны надолго блокировать интерфейс.

Поэтому движки JS асинхронны по умолчанию. Операции ввода-вывода (в том числе Ajax) не блокируют поток. Асинхронный код устроен иначе, чем синхронный: другая обработка ошибок, другие приёмы организации. То, что в синхронном варианте просто, в асинхронном усложняется — плата за отзывчивость.

Эволюция подходов в JavaScript: **колбеки** → **промисы** → **async/await**. В курсе разберём все три; в финале — современный код, близкий по читаемости к синхронному.

Основные темы данного курса:

- Асинхронный IO

- Обработка ошибок

- Event Loop

- Таймеры

- Промисы

- Async/Await

- EventEmitter
