В августе 2006 года Джон Резиг выпустил библиотеку **JQuery**. За короткое время она стала де-факто стандартом для интерактивности на сайтах:

```
// Знак $ — это функция JQuery, через которую все и делается
$(() => {
  // Эта функция выполнится на событии DOMContentLoaded
})
```

Успех во многом объясняется моментом появления: верстка уходила от таблиц, CSS набирал обороты, а в JQuery те же селекторы можно было использовать и для поведения.

В библиотеке рано появилась поддержка селекторов уровня CSS3 — раньше, чем во многих браузерах. Плюс слой совместимости между движками:

```
// Работает так же, как и document.querySelectorAll
// document.querySelectorAll('.section span')
// Возвращает специальную JQuery-коллекцию со своим набором методов
const spans = $('.section span')

// Кроме того, можно осуществлять селекцию внутри конкретного контейнера
// Добавляем его вторым аргументом
const container = $('.item-box')
const items = $('li', container) // находит все li внутри контейнера с классом item-box
```

JQuery популяризовала **ненавязчивый JavaScript** — обработчики не в атрибутах разметки, а в отдельном коде:

```
const buttons = $('button')
// Вешает обработчик click на все кнопки в этой коллекции
buttons.on('click', () => {
  alert('hey!')
})
```

Вместо:

```
<button onclick="alert('hey!')">
```

Сама библиотека — хороший пример предметно-ориентированного API: код ближе к формулировке задачи:

```
// Прячем все элементы по селектору container.main
$('container.main').hide()

// Удаляем элемент с id=address
$('#address').remove()
```

Долгое время JQuery была главным удобным способом делать AJAX и простую анимацию. Экосистема плагинов разрослась настолько, что часть разработчиков начинала с `$`, не углубляясь в чистый DOM.

## Манипуляции

`$` — единая точка входа. Строка внутри считается селектором; результат похож на `querySelectorAll()`, но возвращается jQuery-коллекция.

Операции применяются ко всей коллекции сразу:

```
// Выбираются все элементы h1
const headings = $('h1')
// Добавление класса всем заголовкам h1
headings.addClass('header')

// То же самое без jquery
const headings = document.querySelectorAll('h1')
headings.forEach(el => el.classList.add('header'))
```

Многие методы работают в двух режимах: с аргументом — запись, без — чтение:

```
// Если была найдена одна ссылка
const link = $('#home')
link.html('link to home') // установка textContent
link.attr('href', '/about') // установка атрибута
// Чтение атрибута
console.log(link.attr('href')) // => /about
```

## События

JQuery оборачивает `addEventListener`. Пример клика по кнопке:

```
$('button').on('click', () => {
  // Установка атрибута href на элемент с идентификатором w3c
  // attr может принимать вторым параметром функцию
  // В нее передается оригинальное значение атрибута
  $('#w3c').attr('href', (i, origValue) => `${origValue}/jquery`)
})
```

## AJAX

Запросы в JQuery появились до современных стандартов, поэтому API отличается от `fetch` и промисов:

```
// Цепочка вызовов работает по тому же принципу, что и промисы
// Разница только в деталях
const jqxhr = $.get('/api/v1/companies.json', (data) => {
  console.log('success')
})
  .done(() => {
    console.log('success')
  })
  .fail(() => {
    console.log('fail')
  })
  .always(() => {
    console.log('finished')
  })
```

## Анимация

Есть готовые эффекты и механизм анимации через изменение CSS-свойств:

```
<!-- Плавное появление элемента по клику -->
<div id="clickme">
  Click here
</div>
<img id="book" src="book.png" alt="" width="100" height="123">
<script>
  // Элемент #book загружается как скрытый, а по клику медленно появляется
  $('#clickme').on('click', () => {
    $('#book').show('slow', () => {
      // Действия после завершения анимации
    });
  });
</script>
```

## Перспективы

Раньше JQuery был отличным выбором; сейчас нативный DOM и стандарты покрывают большинство задач. Промисы, `fetch`, полифилы — всё это живёт вне модели JQuery.

Новые проекты редко подключают библиотеку, но легаси-код и вакансии ещё долго будут её упоминать. Сам DOM важнее: API JQuery проще освоить, когда вы уже понимаете дерево документа.
