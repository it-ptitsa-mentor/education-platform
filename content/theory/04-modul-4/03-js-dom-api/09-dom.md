# DOM

Чтобы уверенно писать JS для браузера, полезно понимать, как страница доходит до экрана. Упрощённая схема:

<!-- IMG (из Buildin, перезалить отдельно) -->
В общих чертах:

1. Браузер узнаёт адрес через [DNS](https://developer.mozilla.org/ru/docs/Glossary/DNS), затем запрашивает страницу у сервера:
```
GET /courses HTTP/1.1
HOST: ru.email.io
```

  В ответ приходит HTML.

1. Браузер разбирает разметку и строит **DOM-дерево**.

1. По дереву выполняется отрисовка — то, что вы видите:

<!-- IMG (из Buildin, перезалить отдельно) -->
Зачем дерево, если есть HTML? HTML — текст; манипулировать им напрямую неудобно. Проще построить объект с той же вложенностью, что и теги, и работать с ним. Это и есть **DOM-дерево**.

Слово «дерево» отражает вложенность тегов друг в друга.

Аналогия — AST в JavaScript: интерпретатор не работает с исходным текстом напрямую.

Браузер строит DOM внутри себя, но то же можно сделать из JS:

```
const html = `
  <body>
    <p>hello, <b>world</b>!</p>
  </body>
`

const parser = new DOMParser()

// Каждый HTML-тег становится узлом этого дерева
// Теги, вложенные в него, становятся дочерними узлами
// Для представления текста создаются специальные текстовые узлы
// Все элементы, представленные в HTML, попадают в DOM-дерево, включая пробелы и переводы строк
const doc = parser.parseFromString(html, 'text/html')

// Выводим содержимое тега <body>
console.log(doc.body.innerHTML)
```

Структура согласована между браузерами — отсюда приставка DOM.

**Document Object Model** — платформонезависимый API доступа к HTML: чтение и изменение содержимого, структуры и оформления.

Один код на все движки — иначе пришлось бы писать отдельно под каждый браузер.

Полная идентичность недостижима: стандарт обгоняют не все сразу. Новые методы DOM ждут, пока их подхватит большинство окружений, или подключают полифилы.

Текущая страница доступна как глобальный `document` с методами по спецификации.

Изменения сразу отражаются на экране:

```
// Этот заголовок заменит всю страницу (тег body)
document.body.innerHTML = '<h1>For You!</h1>'
```

## Восстановление

Битый HTML с незакрытыми тегами браузер не ругает — он «починит» разметку и что-то покажет:

```
// Этот HTML содержит ошибки, но браузер все равно отобразит его
const html = '</p>hello, buddy<div>'
```

Действуют правила восстановления. Плюс браузер может добавить узлы, которых не было в исходнике: `<tbody>` в таблицах, корневой `<body>` и т. п.

DOM — основа фронтенда: jQuery, React, Angular и другие библиотеки в итоге меняют дерево (или виртуальное представление над ним).

```
<h1>This is my territory</h1>
```
