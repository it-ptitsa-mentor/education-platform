# События документа

Мы уже знаем события, которые возникают от действий пользователя. Есть и события, связанные с жизненным циклом страницы в браузере:

- `beforeunload` – пользователь пытается уйти со страницы

- `load` – загружены все ресурсы (изображения, стили, скрипты и т. д.)

- `DOMContentLoaded` – DOM построен и готов к работе, без ожидания полной загрузки таблиц стилей, картинок и фреймов

Подробнее разберём последнее. Его часто используют в коде, который зависит от DOM.

Скрипт может выполниться раньше, чем дерево готово — тогда `querySelector` ничего не найдёт. Подписка на `DOMContentLoaded` откладывает инициализацию до безопасного момента.

Событие вешают на `document`:

```
<script>
  document.addEventListener('DOMContentLoaded', () => {
    // Если бы DOM не был готов, мы бы ничего не нашли
    const coll = document.querySelectorAll('.help');
    coll.forEach((el) => el.classList.add('hidden'));
  });
</script>
```

Скорость появления `DOMContentLoaded` сильно зависит от тегов `<script>`. По умолчанию скрипт в HTML блокирует разбор документа, пока не выполнится. Чем дольше работает такой блок, тем позже наступит событие.

Оптимизация загрузки скриптов — отдельная большая тема: много факторов и различий между браузерами. Здесь ограничимся иллюстрацией.

На скриншоте красная полоса — момент `load`, синяя — `DOMContentLoaded`. Внизу указано время от начала загрузки.

Браузер сначала скачивает HTML, находит ссылки на внешние ресурсы и тянет их параллельно, где это возможно.
