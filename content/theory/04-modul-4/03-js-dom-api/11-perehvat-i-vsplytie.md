# Перехват и всплытие

- [Погружение (Capturing)](https://buildin.ai/8e281dd0-5c73-4883-bb7d-8bbf8ddc6a68#b8175a07-bf9c-404b-a4ee-cbe24093af21)

- [Всплытие (Bubbling)](https://buildin.ai/8e281dd0-5c73-4883-bb7d-8bbf8ddc6a68#876bb535-fb14-4d5b-85b8-f01ba4b893e1)

- [W3C Модель](https://buildin.ai/8e281dd0-5c73-4883-bb7d-8bbf8ddc6a68#df2ea79b-d100-4edd-99db-5b37965a27c4)

Допустим, на нескольких элементах висит обработчик `click`. Снаружи — `div`, внутри две кнопки:

```
<div>
  <button id="send">Send</button>
  <button id="cancel">Cancel</button>
</div>
```

```
const div = document.querySelector('div')
const button1 = document.querySelector('#send')
const button2 = document.querySelector('#cancel')

div.addEventListener('click', () => alert('Div alert'))
button1.addEventListener('click', () => alert('Button Send alert'))
button2.addEventListener('click', () => alert('Button Cancel alert'))
```

Клик по области внешнего `div` (не по кнопке) сработает только на нём.

Клик по кнопке внутри `div` затронет и родителя — отработают оба обработчика.

Отсюда вопрос: в каком порядке? Обычно событие идёт от корня дерева к целевому узлу, затем обратно. Эти два прохода — **фазы** события.

## Погружение (Capturing)

Сначала событие спускается от корня к узлу, на котором произошло действие:

```
               | |
---------------| |---------------
| div          | |              |
|   -----------| |-----------   |
|   | button   \ /          |   |
|   -------------------------   |
|        Event CAPTURING        |
---------------------------------
```

На фазе погружения вызываются обработчики, зарегистрированные с третьим аргументом `true` в `addEventListener`:

```
div.addEventListener('click', () => alert('Div alert'), true)
button1.addEventListener('click', () => alert('Button Send alert'), true)
button2.addEventListener('click', () => alert('Button Cancel alert'), true)
```

При клике по `Send` событие идёт сверху вниз. Сначала сработает обработчик на `div`, затем на кнопке:

```
Div alert
Button Send alert
```

Обработчик `Cancel` не вызывается — клик был не на ней.

## Всплытие (Bubbling)

После погружения на `target` начинается всплытие вверх:

```
               / \
---------------| |---------------
| div          | |              |
|   -----------| |-----------   |
|   | button   | |          |   |
|   -------------------------   |
|        Event BUBBLING         |
---------------------------------
```

Если третий аргумент `addEventListener` не указан (или `false`), обработчик вешается на фазу всплытия:

```
div.addEventListener('click', () => alert('Div alert'))
button1.addEventListener('click', () => alert('Button Send alert'))
button2.addEventListener('click', () => alert('Button Cancel alert'))
```

Порядок обратный — сначала внутренний элемент, потом внешний:

```
Button Send alert
Div alert
```

Всплытие — базовая часть модели DOM. Без него сложно вешать один обработчик на контейнер вместо сотен дочерних узлов. Пример — контекстное меню.

По правому клику браузер генерирует `contextmenu`. Его можно перехватить в JS, чтобы:

- показать своё меню вместо системного;

- отменить стандартное поведение (`event.preventDefault()`);

- менять меню в зависимости от зоны страницы.

Другой пример — большая таблица в стиле Excel. Вешать `click` на каждую ячейку — много кода и лишняя нагрузка при росте таблицы. Проще один обработчик на `<table>`: при клике по любой кнопке внутри он всё равно сработает за счёт всплытия.

## W3C Модель

Большинство событий проходят обе фазы: погружение, затем всплытие до корня. Погружение используют редко; почти всегда достаточно всплытия.

В прошлом уроке мы видели `e.target` — самый глубокий узел, куда дошло событие. На всплытии `target` не меняется, поэтому по нему всегда ясно, где именно произошло действие.

`currentTarget` — элемент, на котором висит текущий обработчик. В зависимости от задачи нужен тот или другой:

Иногда всплытие нужно остановить:

- `event.stopPropagation()` — дальше по дереву событие не пойдёт, но остальные обработчики на текущем элементе ещё выполнятся;

- `event.stopImmediatePropagation()` — останавливает и всплытие, и остальные обработчики на этом узле.

Итого: **перехват** — когда обработчик на узле получает событие на своей фазе. Переходя от элемента к элементу, событие то спускается, то поднимается; на каждом шаге срабатывают подписанные обработчики, пока процесс не остановят описанными методами.
