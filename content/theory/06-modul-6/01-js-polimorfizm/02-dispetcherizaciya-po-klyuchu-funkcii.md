# Диспетчеризация по ключу (функции)

Диспетчеризация по ключу объекта сильнее, чем кажется на первый взгляд. Не всегда достаточно подставить разные данные — иногда нужен другой код.

Допустим, нужен массив фамилий пользователей. Для женщин берём девичью фамилию. Решение через `if`:

```
const lastNames = users.map((user) => {
  if (user.gender === 'female') {
    return user.maidenName
  }
  return user.lastName
})
```

Тот же смысл через объект, где значения — функции:

```
// Определяем key-value, где ключ — это пол, а значение — это функция,
// которая возвращает нужное значение для пола
const mapping = {
  female: user => user.maidenName,
  male: user => user.lastName,
}

// Выбираем нужную функцию по названию пола и вызываем ее
// Внутрь функции передается пользователь
const lastNames = users.map(user => mapping[user.gender](user))
```

В ряде случаев такой вариант удобнее: зависимости видны явно, особенно когда их много. Конфигурация данными позволяет вынести таблицу в другое место и расширять поведение без правок вызывающего кода.

Ещё пример — разбор данных по типу:

```
import yaml from 'js-yaml'

const mapping = {
  yml: yaml.load,
  json: JSON.parse,
}

const parse = (type, data) => mapping[type](data)
```
