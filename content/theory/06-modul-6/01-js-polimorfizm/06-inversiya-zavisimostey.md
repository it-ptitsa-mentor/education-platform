# Инверсия зависимостей

Когда внутри функции создаётся объект, появляется жёсткая связь с его классом. Проверка «насколько код связан»: мысленно перенесите функцию в другой проект — сколько зависимостей (и их зависимостей) утащит за собой? Если много — связанность высокая.

Для развязки есть [принцип инверсии зависимостей](https://ru.wikipedia.org/wiki/%D0%9F%D1%80%D0%B8%D0%BD%D1%86%D0%B8%D0%BF_%D0%B8%D0%BD%D0%B2%D0%B5%D1%80%D1%81%D0%B8%D0%B8_%D0%B7%D0%B0%D0%B2%D0%B8%D1%81%D0%B8%D0%BC%D0%BE%D1%81%D1%82%D0%B5%D0%B9) — [DIP](https://en.wikipedia.org/wiki/Dependency_inversion_principle) из SOLID. Формулировка:

- Модули верхнего уровня не должны зависеть от нижнего уровня. Оба зависят от абстракций.

- Абстракции не зависят от деталей. Детали зависят от абстракций.

В разных языках оттенки разные, но суть одна: не цепляться за конкретный класс. Создание объекта на месте использования без подмены — плохой знак. Правильнее инвертировать зависимость: получать готовые объекты снаружи, например параметром.

*DIP также про привязку к интерфейсам в сигнатурах, а не к классам — вернёмся к этому позже.*

Было:

```
const doSomething = () => {
  const logger = new Logger()
  // some code
}
```

Стало:

```
const doSomething = (logger) => {
  // some code
}
```

На докладах про DIP любят аналогию с Голливудом: «не звоните нам — мы сами перезвоним». Не тянуть классы напрямую, а принимать готовые объекты как зависимость.

Всегда ли так? Код, целиком построенный на инверсии, часто избыточно абстрактен. Серебряной пули нет: подмена нужна — делаем; нет — можно работать напрямую.

Рядом с DIP обычно звучит «инъекция зависимостей». DIP — про модульность, инъекция — про способы передать зависимости. Три варианта:

- Аргументы функций или методов — то, что мы уже использовали:
```
doSomethingUseful(new Logger())
```

- Конструктор, когда работаем с объектами:
```
const app = new Application(new Logger())
```

- Сеттеры — по возможности избегать: меняют объект и бьют по целостности:
```
const app = new Application()
app.setLogger(new Logger())
```

За громким термином — передача параметров. Термины помогают говорить короче, но легко уйти в «архитектурный космос».
