# Управление зависимостями

На курсе мы говорили об инициализации: создаются объекты, потом ими пользуется прикладной код. Это соединение с БД, логгер, кеш, шаблонизатор, интеграции и т.д.

В простых случаях, когда объектов мало и структуру задаём сами (без тяжёлого фреймворка), зависимости внедряют вручную:

```
const geolocationService = new GoogleMaps()
const storeService = new StoreService(geolocationService)
const app = new App({ storeService })
app.run()
```

При активной инверсии зависимостей инициализация похожа на матрёшку: верхние объекты получают нижние, слой за слоем собирается приложение.

С ростом числа сервисов сборка утомляет. Зависимость нужна глубоко в коде, а протащить её можно только через цепочку уровней. В крупных фреймворках создают и связывают сотни объектов — писать это руками нереально.

Для этого применяют два подхода (паттерна). Разберём оба.

## Локатор (Service Locator)

Простой вариант — service locator: объект-реестр всех зависимостей. Кому нужен сервис — берёт его у локатора.

Пример на Fastify: подключим Pug через локатор:

```
import fastify from 'fastify'
import pug from 'pug'

const app = fastify()

const locators = {
  renderer: templatePath => pug.renderFile(templatePath),
}

app.get('/hello', (req, res) => res.send(locators.renderer('index.pug')))
```

Подход прост, о нём много статей. Часто локатор называют антипаттерном: прикладной код знает о его существовании.

## Контейнер (DI Container)

Вершина эволюции инверсии — DI Container. Продвинутые контейнеры — целые фреймворки: инициализируют приложение, создают объекты, связывают их. В Java центральный пример — Spring: из него собирают и веб, и фоновые задачи.

Отличие от локатора: зависимости попадают в код прозрачно. Прикладной слой не знает про контейнер — видит только ожидаемые объекты. Инъекция — через конструктор, сеттеры или аргументы методов.

Снова Fastify и Pug, но через встроенный контейнер. Зависимость подключают через `register()`:

```
import fastify from 'fastify'
import view from '@fastify/view'
import pug from 'pug'

const app = fastify()

await app.register(view, { engine: { pug } })

app.get('/hello', (req, res) => res.view('index'))
```

Обработчик вызывает `view()` и не знает, откуда взялся рендерер.
