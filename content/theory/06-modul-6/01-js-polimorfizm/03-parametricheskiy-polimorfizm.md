# Параметрический полиморфизм

Слово «полиморфизм» в разных контекстах означает разное. В императивных языках чаще говорят о полиморфизме подтипов, в функциональных — о параметрическом. Здесь разберём второй.

*В уроке встречается TypeScript. Не обязательно понимать его полностью — цель в концепциях, а не в синтаксисе TS.*

В lodash есть `_.concat()`, склеивающая массивы:

```
_.concat([1], [2, 3, 1]) // [1, 2, 3, 1]
_.concat(['one'], ['two', 'three']) // ['one', 'two', 'three']
_.concat([true], [false, false, true]) // [true, false, false, true]
```

Функция работает с массивами любого типа элементов. Реализуем упрощённый вариант:

```
// Это немного урезанная версия функции concat, она работает только с двумя аргументами,
// каждый из которых — массив
// Функция создает новый массив, затем обходит по очереди переданные массивы
// и добавляет их значения во вновь созданный массив. Затем он возвращается наружу.
const concat = (coll1, coll2) => {
  const result = []
  coll1.forEach(value => result.push(value))
  coll2.forEach(value => result.push(value))
  return result
}
```

В этом коде с элементами массивов ничего не делают — только перекладывают. Значит, `concat` подходит для любых типов данных внутри массивов.

Для разработчиков только на динамических языках это естественно. В статических языках иначе.

```
const ages = [34, 18, 6, 60, 25]
const words = ['one', 'two', 'three']
```

В TypeScript у массива задаётся тип элементов: для первого — `number`, для второго — `string`. Элемент другого типа вызовет ошибку:

```
// В массив чисел добавляется строка
ages.push('hello') // Error
```

Функции над массивами тоже типизированы:

```
const concat = (arr1: number[], arr2: number[]): number[] => {
  // Создаем результирующий массив, длина которого равна сумме длин исходных массивов
  const result: number[] = []

  // Переносим в result все значения из первого массива
  for (let i = 0; i < arr1.length; i += 1) {
    result[i] = arr1[i]
  }

  // Переносим в result все значения из второго массива
  for (let j = 0; j < arr2.length; j += 1) {
    result[arr1.length + j] = arr2[j]
  }

  return result
}

// Объявление массива a
const a = [1, 2, 3, 4]
// Объявление массива b
const b = [4, 16, 1, 2, 3, 22]

// Сливаем массивы
const result = concat(a, b)

console.log(result) // => [1, 2, 3, 4, 4, 16, 1, 2, 3, 22]
```

Сигнатура `(arr1: number[], arr2: number[]): number[]` означает: только массивы чисел. Для строк и других типов эта функция не подойдёт — пришлось бы копировать один и тот же алгоритм для каждого типа.

Здесь помогает параметрический полиморфизм. Статические языки вводят конструкции для алгоритмов «без привязки к типу»: шаблоны (C++), дженерики (TypeScript, Java, C#):

```
const concat = <T>(arr1: T[], arr2: T[]): T[] => {
  const result: T[] = []

  for (let i = 0; i < arr1.length; i += 1) {
    result[i] = arr1[i]
  }

  for (let j = 0; j < arr2.length; j += 1) {
    result[arr1.length + j] = arr2[j]
  }

  return result
}

const a = [1, 2, 3, 4]
const b = [4, 16, 1, 2, 3, 22]

const result = concat(a, b)

// Вызов функции с типом number[]
console.log(result) // => [1, 2, 3, 4,  4, 16, 1, 2, 3, 22 ]

const words1 = ['one', 'two', 'three']
const words2 = ['four', 'five']

// Вызов функции с типом string[]
const wordsResult = concat(words1, words2)

console.log(wordsResult) // => [ 'one', 'two', 'three', 'four', 'five' ]
```

Тип *T* означает «любой тип элементов массива». `concat` ведёт себя как в JavaScript.

Параметрический полиморфизм даёт обобщённые алгоритмы для составных типов и сокращает дублирование. Иногда растёт сложность, но для типичных операций она умеренная — как в примере выше.

В динамических языках отдельные конструкции не нужны: коллекция может хранить что угодно, без дженериков в языке.

В литературе это часто называют обобщённым программированием.
