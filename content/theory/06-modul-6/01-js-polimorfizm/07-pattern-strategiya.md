# Паттерн Стратегия

Закрепим теорию на примере типичного полиморфизма подтипов.

Задача: посчитать стоимость туристической страховки при выезде за границу.

На сумму влияет много факторов. Часть меняет не только коэффициенты в формуле, но и саму формулу.

*В реальных страховках часто одна большая формула с подстановкой значений. Нам важна идея, а не детали отрасли.*

«В лоб» получится каша из вычислений и `if`:

```
// Примеры с потолка
if (age < 18) {
  let cost = salary * age

  if (country === 'uganda') {
    cost = cost / 2
  }
}
else if (age >= 18 && age < 24) {
  // ...
}
```

Можно упростить восприятие? Иногда да. Среди факторов ищем те, что задают глобальное ветвление — верхний `if`. Допустим, это возраст. Диапазоны:

- До 18

- От 18 до 24

- От 24 до 65

- Старше 65

Каждая группа задаёт свой алгоритм расчёта; группы независимы, хотя шаги могут совпадать.

В коде: класс на группу:

```
class LessThan18 {
  // Параметры, это те самые факторы, по которым строится вычисление
  calculate(params) {
    // тут считаем и возвращаем результат
  }
}

// Имя, конечно, так себе, в реальном коде стоит придумать что-нибудь говорящее
class MoreThan18AndLessThan24 {
  // Структура параметров должна 100% совпадать с остальными классами,
  // так как только в этом случае возможен полиморфизм
  calculate(params) {
    // тут считаем и возвращаем результат
  }
}

// Остальные классы
```

Мы разбили расчёт на независимые куски. Каждый такой класс — стратегия. Стратегия не хранит долгоживущее состояние: данные идут в метод, не в конструктор. По сути это функция в обёртке класса ради полиморфизма; то же можно сделать диспетчеризацией функций по ключу — код будет проще.

Как выбрать стратегию? Можно передать готовую снаружи (инверсия зависимостей):

```
calculateCost(strategy, params) {
  strategy.calculate(params);
}
```

Проблему мы сместили, не убрали: где-то останется `if` или диспетчеризация:

```
chooseCostInsuranceStrategy(user) {
  if (user.getAge() < 18) {
    return new LessThan18();
  } else if (/* ... */) {
    // some code
  }
}

strategy = chooseCostInsuranceStrategy(user);
strategy.calculate(params);
```

Со стратегией кода больше, но при диспетчеризации по объекту разница умеренная. Это плата за разделение, упрощающее расширение. Легко перегнуть и сделать хуже, чем было. Полиморфизм подтипов в JS многословен; расширять не всегда нужно — инверсию можно вводить по мере необходимости.
