# TypeScript

TypeScript в программе идёт вторым языком: полезен опыт JavaScript (типы, переменные, условия, циклы, функции, объекты, лямбды). Материал сжато повторяет базовый синтаксис и быстро переходит к типовым задачам TypeScript.

В этом разделе мы узнаем, что такое TypeScript, а также разберёте его особенности, установку и запуск.

## Что такое TypeScript

TypeScript — JavaScript плюс аннотации типов: статическая типизация против динамической в JavaScript. Сравним оба подхода.

В динамически типизированном JavaScript код выполняет интерпретатор построчно, без предварительной проверки типов:

```javascript
# node (Node.js) — интерпретатор JavaScript
# Код сразу запускается на исполнение
node index.js
```

Ошибка несовместимых типов (например, `BigInt` вместо обычного числа) проявится только при запуске:

```javascript
function sum(a, b) {
  return a + b
}

sum(10n, 5) // упс
// 10n — это число типа BigInt, при сложении с обычным числом возникнет ошибка в рантайме
// Uncaught TypeError: Cannot mix BigInt and other types, use explicit conversions
```

В статически типизированном TypeScript перед запуском действует компилятор. Плюсы:

- Нахождение некоторых видов ошибок ещё до запуска кода

- Более простой рефакторинг кода

- Полная поддержка возможностей редактора — автодополнения, навигации по коду и прочее

Перед исполнением TypeScript компилируют в JavaScript.

Во время компиляции проверяется, что программа **типобезопасна** — не содержит ошибок, подобных примеру выше. Если компилятор нашел несоответствие типов, то он останавливает компиляцию и выводит предупреждения о том, где типы не сходятся.

Тот же пример на TypeScript:

```javascript
function sum(a: number, b: number) {
  return a + b
}

sum(10n, 5)
// На этапе компиляции возникнет ошибка с таким текстом
// Argument of type 'bigint' is not assignable to parameter of type 'number'
```

Такой код не скомпилируется: компилятор укажет на несовместимость типов.

Если посмотреть на определение функции, то можно увидеть, что перед нами практически тот же JavaScript, за исключением описания типов входных переменных функции. Там указано, что у параметров `a` и `b` тип `number`. Примерно так же аннотируется типами и остальной код на TypeScript. Местами описания типов могут быть очень сложными, но сама идея остается прежней.

Однозначного ответа нет, какой подход лучше — статическая или динамическая типизация. Всё зависит от конкретной ситуации, а успешные проекты могут быть на разных языках.

## Какие особенности у TypeScript

TypeScript популярен, потому что:

- Его разработала компания Microsoft

- Он почти полностью совместим с JavaScript с точки зрения возможностей и типов — иначе говоря, это «JavaScript с описанием типов» или «надмножество языка JavaScript»

- Его компилятор поддерживает **транспиляцию** — умеет превращать TypeScript-код в JavaScript-код, удаляя определения типов

- У него строгая типизация

Последнюю особенность разберёте подробнее.

### Строгая типизация

Строгость типизации не равна статичности: язык может быть статическим, но нестрогим, и наоборот.

Например, в JavaScript мы можем сложить число со строкой. Когда язык встречает такую конструкцию, он автоматически выполнит преобразование типов: `10 + 'one'`. В TypeScript так не получится. Он выдаст ошибку:

> *The right-hand side of an arithmetic operation must be of type 'any', 'number', 'bigint' or an enum type*

Ruby и Python — динамические, но строгие; строгость обычно упрощает отладку, а не мешает.

## Как устанавливать и запускать TypeScript

Компилятор TypeScript поставляется как npm-пакет. Установка:

```javascript
# Должна быть установлена Node.js
# Внутри проекта, где будет использоваться TypeScript
npm install typescript
```

Затем нужно создать файлы с кодом на TypeScript. Они имеют расширение *ts*. Компилятор TypeScript доступен через утилиту `tsc`:

```javascript
npx tsc index.ts
```

На выходе получается файл с JavaScript-кодом, который уже можно запускать с помощью Node.js:

```javascript
node index.js
```

Также можно поставить пакет [ts-node](https://github.com/TypeStrong/ts-node), который выполняет одновременно компиляцию и запуск. Этот пакет предоставляет REPL для экспериментов с TypeScript.

Итог: TypeScript — JavaScript с типами; выбор между ними зависит от задачи. Вы установили компилятор и способ запуска.
