# Введение в HTML

## Что такое HTML

*HTML (HyperText Markup Language)* — язык разметки гипертекста. Это набор правил, по которым браузер отличает заголовок от списка, таблицу от картинки и так далее. Стандарт появился в 1993 году, чтобы унифицировать вывод текста на веб-страницах.

Посмотрите на страницу, которую читаете сейчас. Какой бы сложной она ни казалась визуально, её структура описана HTML. Суть формата почти не менялась десятилетиями.

Слово «язык» здесь — про **правила**, а не про программирование. HTML только размечает данные: сам он их не обрабатывает и не рисует. Отображением занимается браузер.

Исходный код страницы можно открыть сочетанием `Ctrl + U` (`Cmd + Option + U` на macOS).

Не пугайтесь объёма разметки. Со временем вы научитесь в ней ориентироваться.

Небольшой пример:

```
<section>
  <h1>Основы верстки</h1>

  <p>
    В профессии «Верстальщик» вы изучите множество популярных технологий,
    которые используют при создании верстки.
  </p>

  <p>
    Узнаете о работе с CSS, препроцессорами, немного попрограммируете. Изучите
    Bootstrap и научитесь проверять свой код с помощью специальных утилит.
  </p>
</section>
```

Сейчас непонятны `<section>`, `<h1>`, `<p>` — это нормально. Главное: за тегами стоит **текст**, ради которого страница и существует.

Контент важнее декора. Курсы научат вас стилизовать блоки, но если текст нечитаем, страница теряет смысл.

## Теги в HTML

Что означают `<section>`, `<h1>` и `<p>`? В разметке это **теги**. Ими оборачивают контент, который потом обрабатывает браузер.

Один из базовых тегов — параграф `<p>`. Он отделяет смысловые абзацы; визуально это обычно видно по отступам между блоками текста.

Чтобы вставить параграф, пишут открывающий `<p>` и закрывающий `</p>`, а между ними — текст абзаца.

```
<p>
  В профессии «Верстальщик» вы изучите множество популярных технологий, которые
  используют при создании верстки.
</p>

<p>
  Узнаете о работе с CSS, препроцессорами, немного попрограммируете. Изучите
  Bootstrap и научитесь проверять свой код с помощью специальных утилит.
</p>
```

Параграфов на странице может быть сколько угодно — почти все *HTML-теги* допускают повторное использование.

У `<p>` есть открывающая и закрывающая часть — такие теги называют *парными*.

Есть и *непарные* теги — о них вы узнаете позже, вместе с примерами применения. Не спешите :)

Ещё одна основа — *вложенность*. Снова посмотрите на пример:

```
<section>
  <h1>Основы верстки</h1>

  <p>
    В профессии «Верстальщик» вы изучите множество популярных технологий,
    которые используют при создании верстки.
  </p>

  <p>
    Узнаете о работе с CSS, препроцессорами, немного попрограммируете. Изучите
    Bootstrap и научитесь проверять свой код с помощью специальных утилит.
  </p>
</section>
```

Парный тег `<section>` оборачивает остальные теги и текст внутри них. На вложенности строятся большие разметки.

Иногда вложенность обязательна. Списки без неё не собрать:

```
<ul>
  <li>Первый элемент списка</li>
  <li>Второй элемент списка</li>
</ul>
```

Браузер выведет это так:

---

- Первый элемент списка

- Второй элемент списка

---

Без `<li>` браузер не поймёт, где элементы списка, а где обычный текст.

## Атрибуты

В HTML у тегов бывают **атрибуты** — дополнительные подсказки для браузера.

Ссылки задают тегом `<a></a>`, но одного текста внутри мало:

```
<a>Пойти изучать программирование</a>
```

Браузер не угадает адрес перехода :)

Нужно указать, куда вести ссылку. Для этого служит атрибут `href` со значением URL:

```
<a href="https://example.com">Пойти изучать программирование</a>
```

По клику пользователь попадёт по указанному адресу.

## Дополнительное задание

Воспользуйтесь онлайн-редактором HTML, например [https://htmlcodeeditor.com/](https://htmlcodeeditor.com/).

Вставьте в левую область:

```
<h1>Хорошие книги</h1>

<p>Советую прочитать эти книги:</p>

<ol>
  <li><b>Код. Тайный язык информатики</b> <i>(Чарльз Петцольд)</i></li>

  <li>
    <b>Не заставляйте меня думать</b>
    <i>(<a href="https://en.wikipedia.org/wiki/Steve_Krug">Steve Krug</a>)</i>
  </li>

  <li><b>Дизайн привычных вещей</b> <i>(Donald Norman)</i></li>
</ol>
```

Посмотрите результат. Удаляйте и меняйте теги — экспериментируйте, это полезно :)
