# Публикация в интернете

Рано или поздно проект выкладывают в сеть — чтобы показать результат, проверить на телефоне или отдать ссылку. Проекты «в стол» редко приносят пользу: их стоит публиковать.

Нужны **хостинг** (сервер с файлами) и **доменное имя** (адрес сайта). Как браузер по имени находит сервер, описано в [обзоре DNS на MDN](https://developer.mozilla.org/ru/docs/Glossary/DNS).

Здесь — бесплатный вариант на **GitHub** для простых страниц из HTML и CSS. Такие **статические сайты** не требуют серверного кода и базы данных: достаточно отдать файлы по запросу.

По сути сервер повторяет то, что делал ваш компьютер при локальном открытии файла:

- ОС читает файл с диска или из кэша

- Содержимое уходит в браузер

- Браузер строит страницу на экране

## Подготовка страницы

Сначала соберите минимальный сайт: один HTML и один CSS.

- Создайте отдельную папку проекта

- В ней — *index.html* и *style.css*

- Подключите стили и базовую разметку

Разметка и оформление — на ваш вкус. Ниже пример страницы-визитки:

**index.html**

*внутри тега body*

```
<main>
  <h1>Златопуст Локонс</h1>
  <h2>Создатель профессии «Верстальщик»</h2>

  <hr>

  <section>
    <h2>Ключевые навыки:</h2>
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </section>

  <section>
    <h2>О преподавателе</h2>
    <p>
      Много лет работал в аутсорсинговой компании. Развивал клиентские проекты,
      делал фичи для разных CMS. В основном верстал, готов передать весь опыт
      вам. Поработав с сотнями проектов, научился выкручиваться даже из самых
      безнадежных ситуаций. Поможет освоить верстку, чтобы она перестала
      казаться чем-то страшным.
    </p>
  </section>
</main>
```

**style.css**

```
body {
  font-family: sans-serif;
  line-height: 1.5;

  background-color: #f9f9f9;
  color: #333333;

  width: 980px;
  margin: 0 auto;
}

h1 {
  text-align: center;
  font-size: 36px;
  margin-bottom: 0;
}

main > h2 {
  text-align: center;
  font-size: 25px;
}
```

## GitHub Pages

**GitHub** — популярное хранилище кода. Для статики есть [GitHub Pages](https://pages.github.com/): бесплатная публикация из репозитория.

Полноценная работа с Git полезна, но для одной страницы можно обойтись веб-интерфейсом.

Шаги:

- Зарегистрируйтесь на [GitHub](https://github.com/), если ещё нет аккаунта.

- [Создайте репозиторий](https://github.com/new). Имя: **username.github.io**, где **username** — ваш логин.

<!-- IMG (из Buildin, перезалить отдельно) -->
- Загрузите *index.html* и *style.css*. В URL репозитория добавьте `/upload/` — например `https://github.com/username/username.github.io/upload/`. Выберите файлы и нажмите *Commit changes*.

<!-- IMG (из Buildin, перезалить отдельно) -->
- В *Settings* → *Pages* проверьте публикацию. Для репозитория `username.github.io` сайт часто включается автоматически; иначе в *Source* выберите ветку *main*.

<!-- IMG (из Buildin, перезалить отдельно) -->
Откройте ссылку, которую покажет GitHub — ваша страница в интернете.
