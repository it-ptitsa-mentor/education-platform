# Базовая структура HTML документа

Как театр начинается с вешалки, так и HTML-файл — с базового каркаса. В нём есть служебные теги, которые встречаются в любом документе и помогают браузеру правильно показать страницу.

Минимальный шаблон:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Моя первая страница</title>
  </head>
  <body></body>
</html>
```

Строк немного, но каждая несёт смысл. Разберём их по порядку.

## DOCTYPE

Первой строкой идёт `<!DOCTYPE>`. Это не тег и на странице не отображается. Она сообщает браузеру, по какому стандарту HTML написан файл. Сейчас везде HTML5:

```
<!DOCTYPE html>
```

В HTML5 объявление упростили. На старых сайтах вы можете увидеть длинные варианты — они влияли на режим совместимости. Сегодня достаточно короткой записи выше; старые DOCTYPE нужны лишь при поддержке очень древних браузеров.

## Парный тег html

`<html></html>` — корневой контейнер: внутри него лежит вся разметка, которую браузер обрабатывает как HTML.

Важен атрибут `lang` — язык страницы. По нему браузер корректнее обрабатывает символы разных алфавитов; в CSS по `lang` тоже можно задавать правила (например, типографику для конкретного языка).

Значения — привычные коды: `lang="ru"`, `lang="en"`, `lang="de"`.

## Парный тег head

`<head></head>` хранит служебную информацию: заголовок вкладки, описание, ключевые слова и другое. Это *метаинформация*. Сейчас она нужна не только браузеру, но и поисковикам при ранжировании.

Всё, что внутри `<head>`, на самой странице не показывается — туда не кладут видимый контент.

Разберём несколько тегов, которые пригодятся в любом проекте:

### Метаинформация

Непарный тег `<meta>` принимает разные атрибуты. Для старта важен `charset` — кодировка документа.

Кодировка — таблица, в которой каждому символу соответствует числовой код. Без явного указания у части пользователей текст может отображаться «кракозябрами». Универсальный выбор — `UTF-8`, в нём есть символы большинства языков:

```
<meta charset="UTF-8">
```

### Заголовок страницы

На вкладке браузера виден заголовок страницы. Его задают парным тегом `<title></title>`:

```
<title>Моя первая страница</title>
```

## Тело документа

После `<head>` идёт `<body></body>` — «тело» страницы. Здесь размещается всё, что пользователь видит на экране.

Возьмём фрагмент из прошлого урока и дополним его полным каркасом документа.

```
<header>
  <img src="/logo.png" alt="Логотип">
  <!-- Логотип сайта -->
  <nav id="menu">
    <!-- Меню -->
    <ul>
      <li><a href="/">Главная</a></li>
      <li><a href="/about">О нас</a></li>
      <li><a href="/contacts">Контакты</a></li>
    </ul>
  </nav>
</header>
```

Полная версия с обязательной структурой:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Моя первая страница</title>
  </head>
  <body>
    <header>
      <img src="/logo.png" alt="Логотип">
      <!-- Логотип сайта -->
      <nav id="menu">
        <!-- Меню -->
        <ul>
          <li><a href="/">Главная</a></li>
          <li><a href="/about">О нас</a></li>
          <li><a href="/contacts">Контакты</a></li>
        </ul>
      </nav>
    </header>
  </body>
</html>
```

Браузер может отрисовать HTML и без этого каркаса — подставит `<body>`, DOCTYPE и прочее. Но полагаться на автодополнение рискованно: лучше задавать структуру явно.

## Самостоятельная работа

Соберите базовую структуру документа и поэкспериментируйте с текстом в `<title>`. Попробуйте добавить эмодзи и посмотрите, как заголовок выглядит на вкладке.
