# Редакторы кода

Если до этого вы вёрстали только в онлайн-песочницах, пора поставить редактор кода и сохранять файлы на диске или на сервере.

Теоретически для HTML и CSS хватит обычного «Блокнота» или любого текстового редактора ОС — так верстали в 90‑х и начале 2000‑х. HTML и CSS выполняются в браузере: ваша задача — отдать ему корректные файлы.

<!-- IMG (из Buildin, перезалить отдельно) -->
Так можно и сейчас, но специализированные редакторы дают то, чего нет в простом текстовом окне:

- Подсветка HTML и CSS — проще отличать теги, селекторы, свойства и значения.

- Автодополнение — при вводе свойства или тега редактор подсказывает варианты, как вкладка *Styles* в *Chrome DevTools*.

- Проверка разметки — легко забыть закрыть тег; браузер иногда «догадывается», но не всегда угадывает ваш замысел.

## Visual Studio Code

Редакторов много; в курсе опираемся на популярный *Visual Studio Code* от *Microsoft*. Это универсальный инструмент: и вёрстка, и программирование на других языках. После настройки он станет основной средой для HTML, CSS и кода.

Скачать можно с [официального сайта](https://code.visualstudio.com/). Для HTML и CSS он уже удобен «из коробки».

Создайте файл (`Ctrl + N`) и сохраните с расширением `.html`, например `index.html`. Вставьте разметку и посмотрите на подсветку тегов:

```
<section>
  <h1>Основы верстки</h1>

  <p>
    В профессии «Верстальщик» вы изучите множество популярных технологий,
    которые используют при создании верстки.
  </p>

  <p>
    Узнаете о работе с CSS, препроцессорами, немного попрограммируете. Изучите
    Bootstrap и научитесь проверять свой код с помощью специальных утилит.
  </p>
</section>
```

Начните вводить новый тег — редактор предложит подходящие варианты.

<!-- IMG (из Buildin, перезалить отдельно) -->
Закрывающий тег `<section>` может подсветиться красным, пока разметка неполная — так VS Code сигнализирует об ошибке.

В пустом `.html`-файле введите `!` и нажмите `Enter` — появится каркас HTML-документа с основными мета-тегами.

<!-- IMG (из Buildin, перезалить отдельно) -->
Это делает встроенный плагин *Emmet* — о нём подробнее в одном из следующих уроков.

Создайте файл `.css`, например `style.css`. Редактор включит подсветку для CSS.

<!-- IMG (из Buildin, перезалить отдельно) -->
Селектор, имя свойства и значение обычно разных цветов — в больших файлах это сильно помогает.

Этого достаточно для старта. Позже вы подключите плагины и настроите редактор под себя.

## Другие редакторы

Выбор не ограничен VS Code. На рынке есть платные и бесплатные варианты. Среди бесплатных:

- *Sublime Text* — условно бесплатный, требует настройки, но лёгкий и стабильный.

- *Brackets* — заточен под HTML, CSS и *Javascript*; менее расширяемый, но для вёрстки достаточен.

Можно попробовать несколько редакторов и остановиться на удобном. Редактор — инструмент, он не заменит практику: привыкнув к одному, менять его без причины не обязательно.
