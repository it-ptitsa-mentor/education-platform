# Графические редакторы

Вёрстка опирается не только на CSS, но и на **макеты** — их нужно открыть, прочитать размеры и отступы. На рынке несколько распространённых инструментов: часть только под одну ОС, часть в браузере, часть — кроссплатформенные. Краткий обзор:

1. **Adobe Photoshop**. Долго был стандартом; сейчас уступает более узким UI-редакторам. Много лишнего для HTML-вёрстки и высокие требования к ресурсам. Официально — Windows и macOS; для Linux готовой версии нет.

1. **Gimp**. Часто используют в Linux как замену Photoshop; часть `.psd` открывается, но полной совместимости нет. Тоже не заточен под веб-интерфейсы.

1. **Sketch**. Редактор под макеты сайтов и приложений: удобные размеры, отступы, инспектор. Минус — только macOS, перенос на другие системы не планируется.

1. **Figma**. Работает в браузере (и в десктоп-клиенте); макет можно смотреть с любого устройства с интернетом. Ориентирован на интерфейсы; на учебной платформе и в практических заданиях часто дают макеты именно в Figma. Для личного использования есть бесплатный тариф.

Полный список редакторов шире. На работе может встретиться другой инструмент — это нормально. Интерфейсы похожи: освоив один, в другом разберётесь быстрее.

В уроке — базовые действия в **Figma**.

## Установка

Figma не обязательно устанавливать: достаточно [figma.com](https://www.figma.com/) и регистрации.

Есть десктоп-приложения для Windows и macOS — [страница загрузки](https://www.figma.com/downloads/). Для Linux — варианты сообщества, например [figma-linux на GitHub](https://github.com/Figma-Linux/figma-linux).

Выберите удобный для вас способ.

## Работа с макетом

Разберём интерфейс на примере учебного макета **Hero Section** из практического задания по CSS Flex (файл в Figma по ссылке ниже).

1. Зарегистрируйтесь в Figma — так проще смотреть свойства слоёв.

1. Откройте [макет Hero Section](https://www.figma.com/file/e81LEG70FWd7OGbFwjDVM3/CSS-Flex-Hero-Challenge).

Окно условно делится на четыре зоны:

### Шапка

Настройки файла, публичные ссылки, название. В своём макете здесь же появятся инструменты рисования.

<!-- IMG (из Buildin, перезалить отдельно) -->
### Слои

Слева — все объекты: текст, фигуры, изображения (**слои**). Порядок в списке задаёт наложение на холсте: слой выше перекрывает нижний.

<!-- IMG (из Buildin, перезалить отдельно) -->
Слои группируют — в Hero Section, например, группы *Left Side* и *Right Side*.

### Инспектор (Properties)

Панель свойств выбранного слоя.

<!-- IMG (из Buildin, перезалить отдельно) -->
Для текста «Фронтенд программист» типичные поля:

- **Layout**
  - Ширина и высота
  - `top` и `left` от угла макета — редко нужны при вёрстке, но иногда полезны

- **Content**
  - Текст внутри слоя; клик по контенту часто копирует строку

- **Typography**
  - Font → `font-family`
  - Weight → `font-weight`
  - Size → `font-size`
  - Line height → `line-height`

- **Colors**
  - Цвет заливки/текста; значение можно скопировать

### Окно просмотра макета

В центре — визуальный макет. Слои выбирают мышью; с зажатым `Alt` (или `option` на macOS) при наведении на соседний объект Figma показывает расстояние.

<!-- IMG (из Buildin, перезалить отдельно) -->
Например, для кнопки «Узнать больше» видно размер **127×40** px и отступ **16** px от текста выше.
