# Chrome DevTools

При вёрстке важно вовремя заметить ошибку и понять, как ведёт себя блок. С опытом растёт и объём диагностики: загрузка сети, вес страницы, картинки и скрипты — всё это помогает контролировать и оптимизировать сайт.

Раньше правки часто делали только в файлах и смотрели результат после сохранения. Сейчас в браузерах есть **инструменты разработчика** (веб-инспектор): можно менять HTML и CSS «на лету», не трогая исходники, и проверять вёрстку на разных ширинах экрана (подробнее — в курсе про адаптив).

В уроке — основы *Chrome DevTools* в *Google Chrome*. Логика похожа в Firefox, Safari и Edge; переключаться между браузерами обычно несложно.

<!-- IMG (из Buildin, перезалить отдельно) -->
Открыть панель:

1. Правый клик по странице → *Просмотреть код элемента* (*Inspect* в английской версии).

1. Горячие клавиши: `Ctrl + Shift + I` или `F12` (Windows/Linux), `Cmd + Opt + I` (macOS).

<!-- IMG (из Buildin, перезалить отдельно) -->
По умолчанию откроется вкладка *Elements* — главная для старта. Кратко про другие:

- *Elements* — дерево HTML и стили (разберём подробнее).

- *Console* — вывод и отладка *JavaScript*.

- *Sources* — загруженные файлы, в том числе с внешних доменов.

- *Network* — запросы, размеры, скорость, типы ресурсов.

Эти вкладки пригодятся и в работе; на первых порах чаще всего хватает *Elements*.

## Вкладка Elements

Условно две зоны: **HTML** и **CSS**.

<!-- IMG (из Buildin, перезалить отдельно) -->
Откройте инспектор на этой странице и раскройте дерево. Вложенность видна по отступам; при выборе узла внизу показывается цепочка предков с классами, например:

`html body.min-vh-100 div.mb-n5 div.row.no-gutters div.col-sm.overflow-hidden div.my-4`

Это по сути путь селектора CSS до выбранного элемента. Та же структура в разметке:

```
<html>
  <body class="min-vh-100">
    <div class="mb-n5">
      <div class="row no-gutters">
        <div class="col-sm overflow-hidden">
          <div class="my-4">А вот и наш выбранный блок :)</div>
        </div>
      </div>
    </div>
  </body>
</html>
```

Правый клик по узлу в дереве — меню действий.

<!-- IMG (из Buildin, перезалить отдельно) -->
Основное:

- *Add Attribute* — добавить атрибут.

- *Edit Attribute* — изменить атрибут.

- *Edit as HTML* — править фрагмент разметки: выбранный элемент и его **дочерние**; сам элемент для них **родитель**.

- *Delete Element* — убрать узел с экрана.

Изменения действуют только в открытой вкладке. После перезагрузки страницы они пропадут — удобно для быстрых экспериментов без правки файлов на диске.

Вторая зона — **CSS**.

<!-- IMG (из Buildin, перезалить отдельно) -->
На старте важны вкладки:

- *Styles* — просмотр, правка и добавление правил для выбранного элемента.

- *Computed* — все применённые свойства, наследование, значения по умолчанию браузера и наглядная **блочная модель**.

### Вкладка Styles

<!-- IMG (из Buildin, перезалить отдельно) -->
Вверху — правила, которые напрямую относятся к элементу (например `margin-bottom`, `margin-top`, `box-sizing`, `display`), с указанием селектора и файла со строкой — если CSS не собран в одну минифицированную строку.

Ниже блок *Inherited from ...* (*Наследованные стили от ...*) — что пришло от предков; для текущего элемента подсвечены релевантные свойства.

Значения можно менять кликом; редактор подсказывает допустимые варианты.

### Вкладка Computed

<!-- IMG (из Buildin, перезалить отдельно) -->
Сверху — схема блочной модели: `width`, `height`, `border`, `margin`, `padding` и итоговый размер.

Ниже — сводка каскада: какое свойство откуда взялось; по клику видны файл, строка и селектор.

Этого достаточно для начала. Возможностей у DevTools гораздо больше — ими пользуются на всех этапах разработки.
