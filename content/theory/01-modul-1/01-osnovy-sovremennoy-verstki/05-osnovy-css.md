# Основы CSS

HTML задаёт структуру. У многих тегов уже есть оформление по умолчанию: отступы у параграфов, подчёркивание ссылок, другой размер заголовков. Для современного дизайна этого мало — часть стилей не подходит, часть элементов нужно оформить отдельно.

За внешний вид отвечает **CSS** (*Cascading Style Sheets*, каскадные таблицы стилей). Браузер по CSS решает, как показать HTML.

## Стили CSS

Любое правило записывается по схеме: `свойство: значение;`.

Чтобы применить стиль, нужен **селектор** — он указывает, к каким элементам относятся правила. Пример разметки:

```
<p>
  IT Птица — учебная платформа с практическими курсами по программированию. Мы
  помогаем новичкам выйти на уровень профессиональной разработки, а опытным
  специалистам — осваивать новые темы и расти в профессии.
</p>
```

Изменим цвет и размер шрифта:

```
p {
  color: red;
  font-size: 20px;
}
```

Разбор записи:

1. `p` — **селектор по тегу**: все параграфы на странице.

1. `{ }` — граница блока правил для этого селектора.

1. Внутри — свойства в формате `свойство: значение;`. Список свойств большой; на практике постоянно смотрят документацию — так делают и опытные разработчики.

Полезные свойства для текста на старте:

- `font-size` — размер шрифта. Сначала удобны **пиксели** (`px`).

- `color` — цвет текста. Есть именованные цвета (`red`, `blue`…), hex и другие форматы. Список имён: [MDN: named colors](https://developer.mozilla.org/ru/docs/Web/CSS/named-color).

- `text-align` — выравнивание: `left`, `right`, `center`, `justify`.

## Подключение CSS

Три основных способа:

- отдельный `.css`-файл;

- тег `<style>` в HTML;

- атрибут `style` у элемента (**inline**).

### Использование отдельного CSS-файла

В проектах стилей много — держать их в HTML неудобно. Создают файл с расширением `.css` (имя любое).

Структура каталога, например:

`site/ ├── css/ │ ├── main.css ├── html/ │ ├── index.html`

*index.html*:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
  </head>
  <body></body>
</html>
```

Подключение через `<link>` в `<head>`:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
    <link rel="stylesheet" href="../css/main.css">
  </head>
  <body></body>
</html>
```

Путь `../css/main.css`:

1. `../` — на уровень вверх (из `html/` в `site/`);

1. `css/` — в папку со стилями;

1. `main.css` — имя файла.

Если HTML и CSS лежат рядом: `href="main.css"`. Стили пишут в `main.css` — браузер подхватит их при загрузке страницы.

### Использование тега `style`

CSS можно вложить в `<style>` внутри `<head>`:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>

    <style>
      p {
        color: red;
        font-size: 20px;
      }
    </style>
  </head>
  <body>
    <p>Параграф с размером шрифта 20 пикселей. Текст написан красным цветом.</p>
  </body>
</html>
```

Так иногда выносят критичные стили, чтобы они применились до загрузки внешнего файла. Основной способ в проектах — отдельный `.css`.

### Inline-стили

Атрибут `style` на теге:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
  </head>
  <body>
    <p style="color: red; font-size: 20px;">
      Параграф с размером шрифта 20 пикселей. Текст написан красным цветом.
    </p>
  </body>
</html>
```

Способ неудобен при многих правилах и дублировании; для повторяющихся элементов используют классы и внешний CSS.

## Классы и идентификаторы

Одинаковые теги часто нужно оформить по-разному. Два заголовка второго уровня:

```
<h2>Как стать программистом</h2>
<!-- Тут много текста -->

<h2>Какими навыками обладает программист</h2>
<!-- Тут много текста -->
```

Селектор по тегу `h2` не различит их. Помогают атрибуты `class` и `id`:

- `class` — можно повторять на странице, несколько классов через пробел;

- `id` — уникален на странице (один элемент с данным id).

```
<h2 class="title title-big">Как стать программистом</h2>
<!-- Тут много текста -->

<h2 class="another-title title-big">Какими навыками обладает программист</h2>
<!-- Тут много текста -->

<div id="help">Второго элемента с идентификатором help быть не должно</div>
<div id="another-help">Второго элемента с идентификатором another-help быть не должно</div>
```

## Селекторы

Селекторы определяют, **куда** попадут стили.

*«Не так важны стили, как то, к чему они применяются» © Народная мудрость*

Базовые типы:

1. **По тегу** — имя элемента (`p`, `h1`…).

1. **По классу** — `.имя-класса`.

1. **По id** — `#имя-id` (один раз на странице в разметке).

Пример:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
    <link rel="stylesheet" href="../css/main.css">
  </head>
  <body>
    <main class="content">
      <h1 id="main-title">Заголовок с идентификатором</h1>
      <p>Просто параграф без класса или идентификатора</p>
    </main>
  </body>
</html>
```

Типичный результат стилей:

- `.content` — фон у `<main>`;

- `#main-title` — цвет, размер и выравнивание заголовка;

- `p` — цвет текста параграфов.

### Вложенность селекторов

Часто нужно ограничить область действия правил:

```
<section class="news">
  <h2>Новости</h2>

  <article>
    <h2>Новость 1</h2>
    <p>Текст новости</p>
  </article>

  <article>
    <h2>Новость 2</h2>
    <p>Текст новости</p>
  </article>
</section>

<section class="stories">
  <h2>Интересные истории</h2>

  <article>
    <h2>История 1</h2>
    <p>Текст истории</p>
  </article>

  <article>
    <h2>История 2</h2>
    <p>Текст истории</p>
  </article>
</section>
```

Увеличить только заголовки внутри новостей:

```
.news article h2 {
  font-size: 32px;
}
```

Селектор читают **справа налево**: все `h2` внутри `article` внутри `.news`. Пробел между частями — **селектор потомков** (любая глубина вложенности).

---

**Важно:** не увлекайтесь длинными цепочками — обычно хватает 2–3 частей.

---

Правило `.news article h2` затронет **все** такие заголовки в `.news`, включая вложенные секции:

```
<section class="news">
  <h2>Новости</h2>

  <article>
    <h2>Новость 1</h2>
    <p>Текст новости</p>
  </article>

  <article>
    <h2>Новость 2</h2>
    <p>Текст новости</p>
    <section>
      <h2>Похожие новости</h2>

      <article>
        <h2>Похожая новость 1</h2>
        <p>Текст новости</p>
      </article>
    </section>
  </article>
</section>
```

Чтобы выбрать только прямых потомков, используйте **дочерний селектор** `>`:

```
.news > article > h2 {
  font-size: 24px;
}
```

Заголовки «Похожие новости» и «Похожая новость 1» не попадут под это правило.
