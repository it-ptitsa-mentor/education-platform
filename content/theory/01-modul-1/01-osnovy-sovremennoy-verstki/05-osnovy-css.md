# Основы CSS

HTML задаёт **структуру** страницы: заголовки, абзацы, ссылки, списки. Браузер уже рисует их с базовым оформлением — у параграфов есть отступы, ссылки подчёркнуты, у заголовков другой размер шрифта. Для реального сайта этого мало: нужны свои цвета, сетка, отступы и единый визуальный стиль.

За внешний вид отвечает **CSS** (Cascading Style Sheets — каскадные таблицы стилей). В этом уроке разберём, как записываются правила и как подключить стили к HTML. Про **каскад** — кто побеждает, когда правил несколько — поговорим в следующем уроке.

## Синтаксис правил

Любое CSS-правило строится по одной схеме: `свойство: значение;`.

Чтобы правило попало на нужный элемент, используют **селектор** — он говорит браузеру, *к каким* узлам применить стили. Пример разметки:

```
<p>
  IT Птица — практические курсы по программированию. Мы помогаем новичкам войти
  в профессию, а опытным разработчикам систематизировать знания и расти дальше.
</p>
```

Сделаем текст красным и крупнее:

```
p {
  color: red;
  font-size: 20px;
}
```

Разбор по частям:

1. **`p`** — селектор по тегу: все параграфы на странице.
2. **`{ }`** — блок объявлений: всё между скобками относится к выбранным элементам.
3. **Строки «свойство: значение»** — по одной на свойство, в конце точка с запятой.

Список всех свойств наизусть не учат — смотрят в [справочник MDN](https://developer.mozilla.org/ru/docs/Web/CSS/Reference) или шпаргалки. Начните с трёх частых для текста:

- **`font-size`** — размер шрифта. На старте удобны **пиксели** (`px`): число напрямую задаёт высоту букв на экране.
- **`color`** — цвет текста. Можно указать имя (`red`), шестнадцатеричный код (`#ff0000`) или функцию `rgb()`. Имена базовых цветов — [наглядный список](https://htmlcolorcodes.com/color-names/).
- **`text-align`** — выравнивание строк: `left`, `right`, `center`, `justify`.

## Как подключить CSS к странице

Три способа — от самого удобного в проектах до самого редкого.

### Отдельный файл `.css`

В больших проектах стили выносят в отдельные файлы, чтобы HTML не разрастался. Типичная структура:

`site/ ├── css/ │ ├── main.css ├── html/ │ ├── index.html`

Минимальный `index.html`:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
  </head>
  <body></body>
</html>
```

Подключение через `<link>` в `<head>`:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
    <link rel="stylesheet" href="../css/main.css">
  </head>
  <body></body>
</html>
```

Путь `../css/main.css` читается так:

1. `../` — подняться из папки `html/` в родительскую `site/`.
2. `css/` — зайти в каталог со стилями.
3. `main.css` — имя файла.

Если HTML и CSS лежат рядом, достаточно `href="main.css"`. После подключения правила из `main.css` действуют на всю страницу.

### Тег `<style>` в документе

Стили можно писать прямо в HTML — обычно внутри `<head>`:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>

    <style>
      p {
        color: red;
        font-size: 20px;
      }
    </style>
  </head>
  <body>
    <p>Параграф с размером шрифта 20 пикселей. Текст написан красным цветом.</p>
  </body>
</html>
```

Так иногда оформляют критичные стили «на первый экран», пока грузится основной файл. Для всего сайта это не основной подход.

### Inline-стили

Атрибут `style` на конкретном теге:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
  </head>
  <body>
    <p style="color: red; font-size: 20px;">
      Параграф с размером шрифта 20 пикселей. Текст написан красным цветом.
    </p>
  </body>
</html>
```

Минусы: дублирование, сложно поддерживать, нельзя одним правилом оформить все `<p>`. Используйте только для быстрых экспериментов или редких исключений.

## Классы и id

Два `<h2>` на одной странице — один селектор `h2` не различит их, если нужны разные стили. Помогают атрибуты **`class`** и **`id`**:

- **`class`** — можно повесить на сколько угодно элементов; у одного элемента — несколько классов через пробел.
- **`id`** — уникален на странице: второй такой же id быть не должен.

```
<h2 class="title title-big">Как стать программистом</h2>
<!-- Тут много текста -->

<h2 class="another-title title-big">Какими навыками обладает программист</h2>
<!-- Тут много текста -->

<div id="help">Блок с id help — второго такого на странице быть не должно</div>
<div id="another-help">Ещё один уникальный id</div>
```

В CSS к ним обращаются отдельными селекторами — см. ниже.

## Базовые селекторы

Селектор определяет **область применения** стилей. На старте достаточно трёх видов:

1. **По тегу** — имя элемента: `p`, `h1`, `a`.
2. **По классу** — точка и имя: `.content`, `.title-big`.
3. **По id** — решётка и имя: `#main-title`.

Пример страницы:

```
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>Основной файл index.html</title>
    <link rel="stylesheet" href="../css/main.css">
  </head>
  <body>
    <main class="content">
      <h1 id="main-title">Заголовок с идентификатором</h1>
      <p>Просто параграф без класса или идентификатора</p>
    </main>
  </body>
</html>
```

В `main.css` можно, например:

- `.content { background: black; }` — фон у `<main class="content">`;
- `#main-title { color: white; font-size: 20px; text-align: center; }` — только этот заголовок;
- `p { color: white; }` — все параграфы.

### Вложенность и комбинации

Часто нужно сузить выборку. Разметка с двумя секциями:

```
<section class="news">
  <h2>Новости</h2>

  <article>
    <h2>Новость 1</h2>
    <p>Текст новости</p>
  </article>

  <article>
    <h2>Новость 2</h2>
    <p>Текст новости</p>
  </article>
</section>

<section class="stories">
  <h2>Интересные истории</h2>

  <article>
    <h2>История 1</h2>
    <p>Текст истории</p>
  </article>

  <article>
    <h2>История 2</h2>
    <p>Текст истории</p>
  </article>
</section>
```

Чтобы увеличить только заголовки внутри новостей:

```
.news article h2 {
  font-size: 32px;
}
```

Селектор читают **справа налево**: «все `h2` внутри `article`, которые где-то внутри `.news`». Пробел между частями — **селектор потомков**: подойдут элементы на любой глубине вложенности.

---

**Совет:** не углубляйте цепочку больше чем на 2–3 уровня — так проще поддерживать стили.

---

Если внутри новости появится вложенный блок «Похожие новости», правило `.news article h2` затронет и его заголовки — они тоже потомки `.news`.

**Дочерний селектор** (`>`) ограничивает только **прямых** детей:

```
.news > article > h2 {
  font-size: 24px;
}
```

Заголовки во вложенных `<section>` внутри `<article>` уже не попадут под это правило.

### Комментарии в CSS

Как в многих языках, блок `/* ... */` не попадает в разбор:

```
/* Заголовки новостей */
.news article h2 {
  font-size: 32px;
}
```

## Самостоятельная работа

- Создайте `index.html` и `style.css`, подключите стили через `<link>`. Наберите любой текст и оформите его свойствами из урока.
- Повторите примеры из разделов про селекторы и подключение файлов — так быстрее запоминается синтаксис.
