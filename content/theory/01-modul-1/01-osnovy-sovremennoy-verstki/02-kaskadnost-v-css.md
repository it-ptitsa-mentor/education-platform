# Каскадность в CSS

В прошлом уроке разобрали вторую часть названия *каскадные таблицы стилей* — сами *таблицы стилей*: как записывать правила и применять их к HTML. Здесь — про слово *каскадные*.

Каскадность похожа на цепочку водопадов: вода переходит с уровня на уровень, пока не окажется в общем бассейне.

<!-- IMG (из Buildin, перезалить отдельно) -->
В CSS «вода» — это свойства. Они накладываются и могут перекрывать друг друга. **Каскадность** — способность CSS комбинировать правила из разных селекторов и дополнять их.

Пример. Параграф с классами `text-bold` и `text-dark`: первый задаёт жирное начертание, второй — тёмный цвет. Отдельно для тега `p` зададим `font-size: 24px`.

```
<p class="text-bold text-dark">
  Какой-то очень интересный текст об интересной вещи. Очень интересно.
</p>
```

В CSS:

```
p {
  font-size: 24px;
}

.text-bold {
  font-weight: bold;
}

.text-dark {
  color: #333333;
}
```

Текст будет тёмным, жирным и 24px: стили из классов и селектора по тегу **сложились**. Итог для элемента:

```
{
  color: #333333;
  font-weight: bold;
  font-size: 24px;
}
```

Разные селекторы дали один набор свойств — это и есть каскад.

## Приоритет селекторов

Типы селекторов имеют разный приоритет. При конфликте побеждает более «сильный». Параграф с классом `red` и id `blue`; для всех `p` — зелёный цвет:

```
p {
  color: green;
}

.red {
  color: red;
}

#blue {
  color: blue;
}
```

Проверьте в браузере, прежде чем читать дальше.

Текст будет **синим**. Не потому, что `#blue` записан последним — даже если перенести его в начало файла, цвет останется синим. Селектор по **id** сильнее класса и тега.

Условный порядок приоритета:

1. Селектор по идентификатору (`#blue`)

1. Селектор по классу (`.red`)

1. Селектор по тегу (`p`)

*1* — выше всего, *3* — ниже.

Ещё пример:

```
p {
  color: blue;
  font-weight: bold;
}

.important {
  color: purple;
  font-style: italic;
}

#intro {
  color: green;
}
```

Текст станет **курсивным, жирным и зелёным**:

- `p` — синий (потом перебит) и жирный
- `.important` — пурпурный (потом перебит) и курсив
- `#intro` — зелёный цвет

Итог:

```
{
  color: green;
  font-weight: bold;
  font-style: italic;
}
```

---

**Важно:** id перебивает классы, поэтому для стилей его используют редко — переопределить такие правила классами сложно. Это мешает интерактивным состояниям и смене оформления по классам.

## Переопределение свойств

Если у элемента несколько классов с противоречащими свойствами, в CSS действует правило **«кто ниже в файле — тот прав»**. Порядок классов в HTML не важен.

```
.alert {
  color: gray;
}

.alert-error {
  color: red;
}
```

Сначала серый, затем красный — победит красный. Даже при разметке

```
<p class="alert-error alert">Важное сообщение! Сообщение красного цвета</p>
```

цвет будет красным: CSS не зависит от порядка классов в HTML. Поменяйте порядок правил в файле — изменится и результат:

```
.alert-error {
  color: red;
}

.alert {
  color: gray;
}
```

## Вес селекторов

На практике селекторы сравнивают по **весу** — положение в файле не всегда решает. Пример:

```
<textarea class="form-input"></textarea>
```

Высота будет **50px**, если для `.form-input` задано `height: 50px`, а для `textarea` — другое значение: класс сильнее тега. Чтобы задать **200px** именно для `textarea` с этим классом, уточните селектор:

```
.form-input {
  height: 50px;
}

textarea.form-input {
  height: 200px;
}
```

Приоритеты **складываются** внутри одного селектора. Удобная модель — три цифры:

1. Число id в селекторе
1. Число классов
1. Число тегов

Для `<textarea class="form-input">`:

- `.form-input` → один класс → **010**
- `textarea` → один тег → **001**

010 > 001. После уточнения:

- `.form-input` → **010**
- `textarea.form-input` → класс + тег → **011**

011 > 010, поэтому применится `height: 200px`.

С опытом веса считают реже, но модель помогает понять, почему одно правило побеждает другое и зачем избегают громоздких селекторов.

## Самостоятельная работа

Создайте у себя файл *index.html* со следующим содержимым:

```
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Небольшая вёрстка</title>

  <style>
    div {
      width: 500px;
      height: 500px;

      background: #333333;
    }

    #main {
      color: white;
      width: 750px;
    }

    .text-white {
      color: white;
    }

    .alert {
      height: 350px;
      color: gray;
    }

    div {
      background: blue;
    }

    .alert-warning {
      background: #000000;
      color: yellow;
    }
  </style>
</head>
<body>
  <div id="main" class="text-white alert alert-warning">Какой-то текст</div>
</body>
</html>
```

Разберите, какие стили в итоге применятся к блоку.
