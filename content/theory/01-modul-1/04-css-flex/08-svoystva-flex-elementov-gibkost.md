# Свойства Flex-элементов. Гибкость

Кроме **flex-grow** из прошлого урока, гибкостью управляют **flex-shrink** и **flex-basis**.

Снова три блока в контейнере; на *CodePen* удобно менять значения и смотреть эффект.

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

Меняйте параметры в примерах — так быстрее запоминается поведение.

### flex-basis

**flex-basis** задаёт «стартовый» размер элемента по главной оси до распределения свободного места.

Это не прямая замена **width**/**height**: работает только у flex-элементов и только по главной оси. В отличие от **width**/**height**, задаёт *оптимальный* размер *до* укладки в контейнер.

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

У всех элементов **flex-basis: 200px**. Контейнер *600* px — три блока по 200 px влезают ровно.

Что если у одного элемента **flex-basis** такой, что суммарная ширина превысит контейнер? Поставим первому блоку *400* px:

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

Блоки сжались, чтобы остаться внутри — и с **flex-basis: 400px**, и с **200px**. Браузер пересчитывает пропорции; результат не всегда интуитивен.

### flex-shrink

**flex-shrink** управляет сжатием при нехватке места. **flex-grow** делит избыток ширины; **flex-shrink** — дефицит. По умолчанию у всех flex-элементов **flex-shrink: 1**.

Вернёмся к случаю, когда сумма **flex-basis** больше ширины контейнера. Сначала считаем *отрицательную ширину* — сколько не хватает:

**Отрицательная ширина** = *600* − (*400 + 200 + 200*) = *−200px*

Браузер компенсирует её, сужая элементы. *Общий вес*:

**Общий вес** = *1 × 400* + *1 × 200* + *1 × 200* = *800*.

Чтобы найти итоговую ширину элемента:

- умножить отрицательную ширину на **flex-shrink**;

- разделить **flex-basis** элемента на общий вес;

- перемножить результаты.

Для первого блока:

*(−200 × 1) × (400 / 800)* = *−100px* — столько вычтут из его ширины.

Для двух остальных по той же формуле получится *−50px* каждый.

**flex-shrink: 0** запрещает сжатие элемента.

### flex

**flex** — сокращённая запись **flex-grow**, **flex-shrink** и **flex-basis** в одной строке. Спецификация рекомендует именно её. Порядок значений:

- **flex-grow**

- **flex-shrink**

- **flex-basis**
