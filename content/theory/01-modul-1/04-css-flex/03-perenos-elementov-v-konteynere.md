# Перенос элементов в контейнере

По умолчанию flex-элементы стараются уместиться в одну строку без переноса. Они занимают только нужную ширину — напоминает поведение строчных элементов.

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

Что будет, если места не хватит? Пока не задано иное, контейнер держит всё в одной строке — элементы могут выходить за его границы. Такое поведение не всегда ожидаемо, но полезно, например, при вёрстке слайдеров.

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

Добавим отступы у дочерних элементов, чтобы видеть реальную ширину контейнера. Чаще нужен перенос на новую строку — для этого есть **flex-wrap**.

**flex-wrap** принимает одно из трёх значений:

- **nowrap**. Поведение по умолчанию: одна строка даже при нехватке места.

- **wrap**. При нехватке ширины элементы переносятся на следующую строку.

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

- **wrap-reverse**. Как **wrap**, но новые строки добавляются в обратном порядке по главной оси — строка с *1* и *2* окажется ниже, чем при обычном **wrap**.

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

**flex-wrap** удобен в адаптивной вёрстке: на каркасе сайта он помогает автоматически переносить блоки на узких экранах.
