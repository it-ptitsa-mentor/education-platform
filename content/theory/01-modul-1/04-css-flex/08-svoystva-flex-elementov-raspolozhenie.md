# Свойства Flex-элементов. Расположение

Помимо гибкости, у каждого flex-элемента можно задать положение относительно соседей. В этом уроке — **order** и **align-self**.

### order

**order** задаёт порядок по главной оси. Удобно, например, менять порядок блоков на мобильных без перестановки в HTML.

Значение — любое число: чем меньше, тем раньше элемент в потоке. Поменяем местами первый и третий:

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

Отрицательное **order** не «ломает» раскладку: такие элементы идут раньше с положительным **order**.

Если **order** указан не у всех, элементы без свойства выводятся по порядку в разметке — и *перед* теми, у кого **order** положительный:

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

При положительных **order** элементы без свойства идут первыми. С отрицательным **order** элемент окажется *до* тех, у кого свойство не задано: по умолчанию **order: 0**.

Элементы без **order** в примере чуть темнее — сопоставьте классы `.bg-red-one`, `.bg-green-two`, `.bg-blue-three` с положением на экране. Если нужен полный контроль, задайте **order** всем детям контейнера.

### align-self

**align-self** похож на **align-items**, но действует на один элемент, а не на весь контейнер — переопределяет выравнивание по поперечной оси для выбранного блока.

Основные значения:

- **center**

- **flex-start**

- **flex-end**

- **baseline**

<!-- CodePen: интерактивный пример (перезалить отдельно) -->

В примере добавлены классы:

- **align-self-center**

- **align-self-start**

- **align-self-end**

- **align-self-baseline**

Меняйте классы у элементов и смотрите, как смещается выравнивание.
