# Наложение элементов

В уроке про плавающие элементы вы уже видели, что блоки могут перекрывать друг друга. То же бывает при позиционировании вне нормального потока. Порядок наложения настраивают свойством `z-index` — из названия видно связь с осью `z`.

Ось `z` определяет, что ближе к пользователю, а что уходит «назад». Поставим трём блокам `position: absolute`:

[CodePen](https://codepen.io/)

Логичный вопрос: «Где два других блока?». Они под последним — `.block-three`. С абсолютным позиционированием все три оказались в одной точке. Сначала в угол лёг `.block-one`, затем `.block-two` накрыл его (браузер поднял второй по оси `z`), затем то же с `.block-three` — он оказался сверху и закрыл остальных.

Блоки не исчезают из DOM. Это видно, если задать разную ширину и высоту:

[CodePen](https://codepen.io/)

`z-index` задаёт слой: чем больше число, тем выше элемент по оси `z`. Работает только вместе с `position`, равным одному из значений:

- `absolute`

- `relative`

- `fixed`

- `sticky`

Как браузер складывает слои по умолчанию:

1. Внизу — корневой элемент `html`, поверх него всё остальное.

1. Далее элементы в нормальном потоке — в порядке разметки, с учётом блочного и строчного поведения.

1. В конце — элементы с `position`, тоже в порядке появления в HTML (как в примерах выше).

Поменяем порядок в последнем примере: у `.block-two` поставим `z-index` больше, чем у `.block-three`. Третий блок «пропадёт» из виду, если он меньше по размеру:

[CodePen](https://codepen.io/)

`z-index` может быть отрицательным, нулевым или положительным — отрицательные значения уходят «ниже» положительных. В проектах удобно увеличивать `z-index` с шагом *100*, чтобы не путаться в цифрах:

```
z-index: 34234;
z-index: 43233;
z-index: 34324;
```

По таким числам сложно быстро понять порядок — приходится сравнивать каждую цифру. Сравните с вариантом:

```
z-index: 34200;
z-index: 43200;
z-index: 34300;
```

Если слои в диапазоне меньше *100*, этот приём можно не использовать.
