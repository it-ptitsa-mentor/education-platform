# Плавающие элементы

Разобравшись с `position`, переходим к **плавающему потоку**. Его отличие — обтекание: как в журнальной вёрстке, когда картинка стоит слева или справа от текста, а строки огибают её.

[CodePen](https://codepen.io/)

Чтобы включить плавающий поток, используют `float` с двумя основными значениями:

- `left`

- `right`

По названию ясно: обтекание слева или справа. В примере выше замените `float: left` на `float: right` и поменяйте отступы — `margin-right` на `margin-left`, — чтобы увидеть разницу.

При `float` элемент тоже выходит из нормального потока — как при `absolute`. Почему соседний блок не наезжает? Наезжает, но текст «знает» о плавающем соседе, и браузер сдвигает строки к его границе.

Эффект заметнее с `opacity` и внутренним отступом у блока:

[CodePen](https://codepen.io/)

Раньше `float` массово применяли для колоночных макетов — хотя правило для этого не задумывалось, и вёрстка на нём была неудобной. С появлением `flex` и `grid` для сеток `float` почти не нужен; остаётся классическая задача — обтекание текста вокруг иллюстраций.

Ещё деталь: с `float` любой элемент становится блочным. Для строчных, например `span`, после `float` можно задавать ширину и высоту без отдельного `display`.

[CodePen](https://codepen.io/)

## Очистка плавающего потока

На макетах `float` часто создавал проблемы. Бывает, что элементы плавают, а нужно запретить обтекание или «сбросить» поток.

Пример: статья и несколько боковых плашек с `float`:

[CodePen](https://codepen.io/)

Разберите разметку и посмотрите, где оказываются блоки.

Чтобы плашки не заходили в текст, у обтекающего элемента ставят `clear` со значениями:

- `left` — не обтекать слева

- `right` — не обтекать справа

- `both` — не обтекать ни слева, ни справа

Запретим обтекание статьи слева — у `<main>` зададим `clear: left`:

```
main {
  clear: left;
}
```

[CodePen](https://codepen.io/)

То же с `clear: right` для правой плашки не сработает:

```
main {
  clear: right;
}
```

[CodePen](https://codepen.io/)

Потому что плавающий блок лежит *внутри* статьи — обтекается не весь `<main>`, а его часть. `clear` тогда ставят у того, кто обтекает плавающий элемент — в примере у параграфа:

```
<p class="clear-right">Также у PHP недавно появились альтернативные рантаймы (среды выполнения), типа RoadRunner. Они позволяют срезать расходы ресурсов компаний на инициализацию приложения — на настройку и запуск всех необходимых библиотек и фреймворков.</p>
```

[CodePen](https://codepen.io/)
