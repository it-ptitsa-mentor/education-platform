# Фиксированное позиционирование

Позиционирование не ограничивается относительным и абсолютным — для большинства задач этой пары хватает. Но бывают сценарии, где одного вида недостаточно.

Например, нужна шапка, которая остаётся на экране при прокрутке. Ни `relative`, ни `absolute` не подходят: блок останется привязанным к месту в документе и уедет вместе с контентом.

Здесь помогает *фиксированное позиционирование*. Оно близко к абсолютному:

- элемент выходит из нормального потока, его место занимают соседи;

- положение задаётся через `top`, `right`, `bottom`, `left`.

Отличие в том, что **точка отсчёта — видимая область окна браузера**. Если задать `top: 0` и `left: 0`, элемент останется в левом верхнем углу экрана и при прокрутке страницы.

```
<style>
  .fixed {
    position: fixed;
    top: 0;
    left: 0;
  }
</style>

<div class="fixed">Блок с фиксированным расположением. Его координаты относительно видимой области браузера всегда будут одни и те же.</div>
```

[CodePen](https://codepen.io/)

Прокрутите текст — левая колонка останется на месте. Так часто делают меню, которое всегда видно пользователю.
