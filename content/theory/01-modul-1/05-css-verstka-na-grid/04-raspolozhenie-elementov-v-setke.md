# Расположение элементов в сетке

В прошлом уроке мы собрали 12-колоночную сетку, где элементы шли друг за другом и занимали по одной колонке. В реальных проектах так бывает редко: у блоков своё место в макете, и они редко выстраиваются в одну линию.

В одном из первых уроков мы говорили, что у каждой *Grid-линии* есть порядковый номер — при счёте слева направо и справа налево (в последнем случае номера отрицательные). Посмотрим на обычную сетку:

<!-- IMG (из Buildin, перезалить отдельно) -->
По вертикали здесь 5 *Grid-линий*. У крайней левой два номера: *1* и *-5*. По горизонтали 4 линии; у нижней — *4* и *-1*.

Эти номера привязывают элементы к конкретным линиям. С Grid мы реже оперируем шириной и высотой блока напрямую — размеры задают колонки и ряды.

Вернёмся к примеру с 12 колонками и тремя элементами:

[codepen](https://codepen.io/pen/BajRRWm)

Расположим блоки так:

- Первый — на все 12 колонок с линии 1, по вертикали 2 ряда.

- Второй — ниже первого, 4 колонки с первой.

- Третий — оставшиеся 8 колонок рядом со вторым.

Получится типичный двухколоночный макет. Для привязки к линиям в *CSS Grid Layout* есть четыре свойства:

- `grid-column-start` — начальная вертикальная *Grid-линия*.

- `grid-column-end` — конечная вертикальная линия. Значение указывает, **до** какой колонки тянется элемент; сама эта колонка не входит. Чтобы включить последнюю колонку в 12-колоночном макете, укажите 13.

- `grid-row-start` — начальная горизонтальная линия.

- `grid-row-end` — конечная горизонтальная линия; логика та же — элемент тянется **до** указанного ряда, не включая его.

Так задаются и старт, и конец — отсюда ширина и высота блока. Пропишем свойства для трёх секций с семантической разметкой:

[codepen](https://codepen.io/pen/NWxvqdg)

Для шапки указан `grid-column-start` — кажется избыточным, ведь блок и так начинается с первой колонки. Но это важно, когда задана конечная линия: без `grid-column-start` элемент может «расти» справа налево.

[codepen](https://codepen.io/pen/XWXavrd)

Есть сокращения: `grid-column` — начало и конец по горизонтали через слэш; `grid-row` — то же по вертикали.

Тот же пример короче:

```
.grid-header {
  grid-column: 1 / 13;
  grid-row: 1 / 3;
}

.grid-aside {
  grid-column: 1 / 5;
  grid-row: 3 / 13;
}

.grid-main {
  grid-column: 5 / 13;
  grid-row: 3 / 13
}
```

Удобный приём — отрицательные номера линий. У каждой линии два значения, поэтому конечную можно задать как *-1* и растянуть блок до края без подсчёта колонок.

## Ключевое слово span

В макетах удобнее знать стартовую линию и **сколько** колонок занять, а не индекс конца. Раньше изученными свойствами это не сделать.

В стандарте есть ключевое слово **span** — сколько ячеек охватить: `span <количество>`.

В двухколоночном макете шапка уже тянется до *-1*. Для колонок ниже **span** упрощает запись:

```
.grid-header {
  grid-column: 1 / -1;
  grid-row: 1 / span 2; /* элемент на 2 строки */
}

.grid-aside {
  grid-column: 1 / span 4; /* 4 колонки по горизонтали */
  grid-row: 3 / -1;
}

.grid-main {
  grid-column: 5 / -1;
  grid-row: 3 / -1
}
```

[codepen](https://codepen.io/pen/rNxgyMJ)

Можно указать только **span** — браузер займёт указанное число колонок или строк. Например, ширина в 2 колонки:

```
.col-2 {
  grid-column: span 2;
}
```

Так удобно собирать «фреймворк» из классов по числу занятых колонок.

[codepen](https://codepen.io/pen/ExPzWwd)

## Именование линий

Помимо индексов линиям можно дать имена — так проще читать *CSS* и не держать в голове номера на сложных макетах. Синтаксис в `grid-template-columns` и `grid-template-rows`: *[имя линии] ширина полосы после линии*. Имена можно добавлять и в готовый шаблон.

В двухколоночном макете назовём стартовые линии областей — четыре имени:

1. *header-top* — верх шапки.

1. *content-begin* — начало основного контента (*aside* и *main*), верхняя граница шапки.

1. *sidebar-begin* — левая граница *aside*.

1. *main-begin* — старт *main*, правая граница *aside*.

Пример записи:

```
.grid-12 {
  display: grid;
  grid-template-columns: [sidebar-begin] 1fr repeat(3, 1fr) [main-begin] 1fr repeat(7, 1fr);
  grid-template-rows: [header-top] 1fr 1fr [content-begin] 1fr repeat(9, 1fr);

  height: 100vh;
}
```

Строка `grid-template-columns: [sidebar-begin] 1fr repeat(3, 1fr) [main-begin] 1fr repeat(7, 1fr);` — четыре части:

- `[sidebar-begin] 1fr` — именованная линия и полоса 1fr.

- `repeat(3, 1fr)` — три полосы по 1fr.

- `[main-begin] 1fr` — линия *main-begin* и полоса 1fr.

- `repeat(7, 1fr)` — семь полос по 1fr.

Имена подставляются в `grid-column-start`, `grid-column-end`, `grid-row-start`, `grid-row-end`.

[codepen](https://codepen.io/pen/ZEQJgyp)

**Важно:** именованные линии не отменяют индексы — можно обращаться и по номеру (в том числе отрицательному), и по имени.

## Создание именованных Grid-областей

Кроме линий можно именовать целые области — несколько полос, ограниченных общими линиями. Это один из сильных инструментов *CSS*: имена задают и визуальную схему сетки. Минус — запись может быть громоздкой. Разберём на практике.

Для областей служит `grid-template-areas`. Значение — строки в двойных кавычках: сколько колонок занимает область, столько раз повторяется имя в строке, части разделяются пробелом. Сетка из 6 колонок, двухколоночный макет: шапка на всю ширину — шесть раз *header*. Область связывается с элементом через `grid-area`.

```
.grid-6 {
  display: grid;
  grid-template-areas: "header header header header header header";
}

.grid-header {
  grid-area: header;
}
```

У `.grid-6` нет `grid-template-columns` и `grid-template-rows` — их роль временно взяло `grid-template-areas`: число имён в строке задаёт колонки по 1fr. При необходимости ширину полос уточняют отдельными `grid-template-columns` и `grid-template-rows`.

Следующий ряд — *aside* и *main*:

```
.grid-6 {
  display: grid;
  grid-template-areas: "header header header header header header" "aside aside main main main main";
}

.grid-header {
  grid-area: header;
}

.grid-aside {
  grid-area: aside;
}

.grid-main {
  grid-area: main;
}
```

**Важно:** в каждой строке одинаковое число колонок. Иначе браузер проигнорирует `grid-template-areas`.

Дополнительные ряды добавляют при нестандартной высоте ячеек; иначе высоту задаёт контент — как в обычной вёрстке. Здесь получилась сетка 6×2.

[codepen](https://codepen.io/pen/QWyqLaM)

Для читаемости строки в `grid-template-areas` переносят; пробелы внутри значений не мешают. Колонки можно выровнять друг под другом:

```
.grid-6 {
  display: grid;
  grid-template-areas: "header header header header header header"
                       "aside  aside  main   main   main   main";
}
```

Иногда нужна пустая ячейка, а не область. Удалить «лишнюю» колонку нельзя — вместо имени ставят точку `.`. Шапка на 5 колонок из 6:

```
.grid-6 {
  display: grid;
  grid-template-areas: "header header header header header ."
                       "aside  aside  main   main   main   main";
}
```

[codepen](https://codepen.io/pen/qBbPBdE)

## Самостоятельное задание

Соберите трёхколоночный макет:

- Шапка.

- Левый сайдбар.

- Основной контент.

- Правый сайдбар.

- Футер.

Сначала через *Grid-линии*, затем через *Grid-области*. Число колонок выберите сами (не меньше трёх).
