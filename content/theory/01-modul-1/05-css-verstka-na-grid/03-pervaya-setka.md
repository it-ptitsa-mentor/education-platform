# Первая сетка

Разобрав терминологию, соберём первую сетку: 12 колонок и 12 строк.

Как и в *Flex*, нужен контейнер. Создадим секцию с классом *grid-12*:

```
<section class="grid-12"></section>
```

Чтобы сделать секцию grid-контейнером, задаём `display: grid`. Сразу визуально мало что изменится — добавим три дочерних блока с разным оформлением и на их примере настроим сетку.

[codepen](https://codepen.io/pen/xxZqBVr)

Без явного числа колонок и строк получили три строки и одну колонку. Если нужна именно такая структура, возможно, grid избыточен — браузеру не обязательно строить сетку.

Альтернатива — `display: inline-grid`: внутри всё как у `grid`, но контейнер ведёт себя как строчный элемент и занимает только нужную ширину; соседи могут обтекать его при наличии места.

[codepen](https://codepen.io/pen/wvMdJJG)

## Создание сетки

Создадим сетку 12×12. За колонки отвечает `grid-template-columns`, за строки — `grid-template-rows`. Свойства принимают разные значения, в том числе:

- Любые *CSS-единицы*: *px*, *em*, *rem* и т.д.

- `min-content` — ширина колонки по минимально возможному размеру контента.

- `max-content` — по максимально возможному размеру контента.

- `minmax(min, max)` — функция с нижней и верхней границей; браузер подбирает ширину внутри диапазона.

- `auto` — колонки подстраиваются так, чтобы самый крупный элемент поместился.

Ещё несколько важных значений разберём ниже.

Число колонок и строк задаётся не отдельным свойством, а **количеством значений**, разделённых пробелом: каждое значение — одна полоса. Для 12×12 можно перечислить по 12 размеров в каждом свойстве.

Соберём ровную сетку с ячейками 20×20 px. Разметка:

```
<section class="grid-12">
    <div class="grid-element bg-gray"></div>
    <div class="grid-element bg-red"></div>
    <div class="grid-element bg-blue"></div>
</section>
```

Высоту контейнера убрали — она выводится из размеров полос; отдельные width/height контейнера нужны только при особых требованиях.

[codepen](https://codepen.io/pen/WNrjpqX)

До `grid-template-columns` и `grid-template-rows` элементы тянулись на всю ширину. Теперь каждый занимает одну ячейку 20×20 px — это видно в *веб-инспекторе* (*Chrome DevTools*) при наведении на блок.

<!-- IMG (из Buildin, перезалить отдельно) -->
В `grid-template-columns` и `grid-template-rows` можно смешивать значения — для каждой полосы своё:

```
.grid-12 {
  display: grid;
  grid-template-columns: 20px 3em minmax(20px, 10em) auto 20px 8% 20px 20px 20px 10em 20px 20px;
  grid-template-rows: 20px 20px 20px 20px 20px 20px 20px 20px 20px 20px 20px 20px;
}
```

[codepen](https://codepen.io/pen/eYJWWJx)

Обратите внимание на размеры полос в этом варианте.

### Repeat

Перечислять одинаковые колонки вручную утомительно и легко ошибиться. Есть функция `repeat` с двумя аргументами:

1. Сколько раз повторить.

1. Какое значение повторять.

12-колоночная сетка короче:

```
.grid-12 {
  display: grid;
  grid-template-rows: repeat(12, 20px);
  grid-template-columns: repeat(12, 20px);
}
```

[codepen](https://codepen.io/pen/BajRRWm)

Картина та же, кода меньше — удобнее читать :)

`repeat` — одно из значений в `grid-template-columns` и `grid-template-rows`; её можно вызывать несколько раз и комбинировать с другими размерами. Первые 6 полос по 20 px, остальные по 30:

```
.grid-12 {
  display: grid;
  grid-template-rows: repeat(6, 20px) repeat(6, 30px);
  grid-template-columns: repeat(6, 20px) repeat(6, 30px);
}
```

В *веб-инспекторе* сетка выглядит так:

<!-- IMG (из Buildin, перезалить отдельно) -->
Вместо фиксированного числа повторений есть:

- `auto-fill` — браузер повторяет колонки столько раз, сколько влезает в ширину контейнера.

- `auto-fit` — почти как `auto-fill`, но пустые колонки сжимаются до нуля; освободившееся место забирают заполненные.

[codepen](https://codepen.io/pen/QWyvvxJ)

Откройте сетку в *веб-инспекторе*: с `auto-fill` получается *16×5*, с `auto-fit` — *3×1*: браузер убрал пустые колонки без элементов.

### Фракция

Речь не о политике. При проектировании *CSS Grid Layout* хотели упростить адаптивность — сегодня страницу нужно подстраивать под разные экраны. Относительные единицы помогают, но при смене числа полос приходится пересчитывать все размеры.

В стандарте появилась единица **фракция** (fraction unit): доля свободного пространства, которую должна занять полоса.

Принцип похож на *flex-grow* из курса [CSS: Flex](https://buildin.ai/ae8f00f2-1bf1-41de-8046-ae92d79ae787): указываем, сколько «долей» занимает ячейка относительно соседей.

Сетка на всю ширину и высоту экрана, 12 колонок и 12 рядов, размер полос зависит от *viewport* — с *fr* это просто:

```
.grid-12 {
  display: grid;
  grid-template-rows: repeat(12, 1fr);
  grid-template-columns: repeat(12, 1fr);

  width: 100vw;
  height: 100vh;
}
```
