# Работа с сеткой

В уроке по терминологии мы смотрели блог T-Журнала: помимо полос, по которым выстроен контент, в сетке есть отступы между ними.

Зачем они? Часто контент в ячейке тянется на всю ширину; промежутки между колонками заменяют *margin*, но с важным отличием: *Grid* ставит отступы только там, где нужно — не приходится снимать лишние *margin* у первой или последней колонки.

Эти отступы **добавляются к ширине полос**, внутри самого элемента отступов не появляется.

Перейдём к практике. Возьмём типичный двухколоночный макет из прошлых уроков. Для наглядности зададим ширину каждой колонки 100 пикселей и опишем *Grid-области*:

[codepen](https://codepen.io/pen/LYGOPVB)

Обратите внимание, как сочетаются `grid-template-areas` и `grid-template-columns` / `grid-template-rows` — они не заменяют друг друга, а работают вместе.

Макет из 6 колонок. Между сайдбаром и основным контентом пока нет зазора — тексты из разных блоков визуально слипаются, читаемость страдает.

[codepen](https://codepen.io/pen/jOWaNrM)

Отступы между *Grid-полосами* задают `grid-column-gap` и `grid-row-gap` — между колонками и между рядами. Значение — любая *CSS*-единица: пиксели, проценты и т.д.

Добавим между колонками 30 пикселей на контейнере:

[codepen](https://codepen.io/pen/mdVBNGz)

В *Chrome DevTools* картина такая:

---

Свойства `grid-column-gap` и `grid-row-gap` постепенно уходят из употребления: в спецификации те же отступы появились у *Flex*, чтобы отказаться от отрицательных *margin* в макетах. Для обоих модулей ввели универсальные `column-gap` и `row-gap` без приставки *grid-*.

---

Для краткости есть `grid-gap` (или просто `gap`) — отступы по рядам и колонкам в одном свойстве. `grid-gap` принимает два значения:

- Отступ между рядами.

- Отступ между колонками.

Одно значение задаёт одинаковый отступ и для колонок, и для рядов.

## Позиционирование полос

В *CSS Grid Layout* два свойства выравнивают полосы внутри контейнера: `justify-content` и `align-content`. Знакомо тем, кто проходил *Flex*: модули *CSS Flexible Box Layout* и *CSS Grid Layout* разделяют одни и те же имена и значения. В примере — три полосы разного цвета; контейнер серый, чтобы видеть его ширину.

[codepen](https://codepen.io/pen/ExPbPRW)

### justify-content

`justify-content` выравнивает *Grid-полосы* по горизонтали — колонки внутри контейнера. Как во *Flex*, возможны значения:

**start** — обычное поведение: колонки от левого края друг за другом. Дополнительные примеры здесь излишни.

**end** — колонки смещаются к правому краю.

[codepen](https://codepen.io/pen/gOPXPZq)

**center** — колонки по центру контейнера.

[codepen](https://codepen.io/pen/KKVyVEd)

**stretch** — значение по умолчанию, когда у колонок нет фиксированной ширины. В примере колонки с `auto`; тёмная рамка — дочерние элементы 40×40 px.

Поиграйте с `justify-content` в примере и сравните с тем, что уже видели во *Flex*.

[codepen](https://codepen.io/pen/wvMPMOL)

**space-around** — как во *Flex*: равные промежутки между колонками и половина такого расстояния по краям.

[codepen](https://codepen.io/pen/zYrPqdW)

**space-evenly** — как *space-around*, но отступы по краям равны расстоянию между колонками, а не половине.

[codepen](https://codepen.io/pen/BajmKmo)

**space-between** — одинаковые промежутки между колонками, без отступов от краёв; крайние прижаты к границам. Два элемента окажутся по краям.

[codepen](https://codepen.io/pen/BajmKYj)

### align-content

`align-content` похоже на `justify-content`, но выравнивает **ряды** по вертикали, а не колонки по горизонтали. В одном примере *codepen* показаны все значения свойства:
