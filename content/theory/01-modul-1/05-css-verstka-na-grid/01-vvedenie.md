# Введение

После позиционирования и Flex вы наверняка заметили: даже с большим набором инструментов сверстать макет «в лоб» не всегда просто. Flex не закрыл главную задачу — быстро собрать каркас страницы. Приходится плодить вложенность: Flex по сути одномерен, удобно управлять контентом либо по оси **x**, либо по **y**. Для контента это отлично, для общей структуры — слабовато.

Flex изменил подход к вёрстке и избавил от неудобного позиционирования и бесконечной борьбы с float. Но остался вопрос: что делать со сложными шаблонами? Число *flex*-контейнеров росло вместе с количеством нестандартных блоков, а управлять такой конструкцией становилось всё труднее. Без единой сетки страницы, описанной в одном месте, структура получалась хрупкой и ломалась быстрее, чем успеваешь моргнуть.

Для каркаса страницы появился CSS Grid Layout — просто Grid. В отличие от Flex, это двумерная система раскладки.

**Важно:** Grid не заменяет Flex и не задумывался как замена. Это два разных модуля, которые хорошо дополняют друг друга.
